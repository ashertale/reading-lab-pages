單向門與雙向門

One-Way Door / Two-Way Door

不是所有決策都值得用同樣的慎重度對待；可逆決策若被過度拖慢會浪費時機，不可逆決策若被當成可隨便回頭，代價又會很高。

Read For

看懂什麼決策該快、什麼決策該慢，避免把所有選擇都塞進同一種審批節奏。

Orientation

真正拖慢團隊的，常不是做錯決策，而是把可快的決策也做成不可逆決策的樣子

這個工程概念會一直被拿出來講，是因為 可逆決策若被過度升格，組織會失去速度；不可逆決策若被低估，錯誤又會以更貴的形式回來。

Core Tension

可逆決策若被過度升格，組織會失去速度；不可逆決策若被低估，錯誤又會以更貴的形式回來。

01 先判斷回頭成本

能快速試錯與回滾的決策，本來就不該享受和鎖死架構一樣的審批待遇。

02 再讓流程跟著決策性質走

可逆決策要重速度，不可逆決策要重資訊密度與反證。

03 最後為可逆性刻意設計

很多決策之所以變成單向門，不是本質如此，而是前期沒有為回頭留結構。

Setup

把問題放回一個小型系統

單向門與雙向門的價值，在於把「先做再說」和「這一步其實很難回頭」分出不同重量。

最小劇情

團隊為了一個可快速回滾的 UI 試驗跑完整月審批流程，卻對一個難以逆轉的資料模型鎖定只用一天拍板。

真正難點

難點在於很多組織用相同流程處理不同性質的決策，於是既慢又不準。

把不同決策混進同一套流程
→
可逆決策被拖慢、不可逆決策被低估
→
速度與風險同時失衡
Key Point

單向門與雙向門最實用的地方，是讓你先判斷決策的可逆性，再決定該用多快、多寬鬆、多少人參與的流程。

Lenses

把設計張力拆成幾條線來看

門是不是難回頭，不是看決策當下有多大聲，而是事後要不要為它付出高昂撤回成本。

可逆性

一個決策能不能低成本回頭，決定了它值不值得快試。

節拍匹配

決策流程若和決策性質不匹配，組織就會在錯的地方慢、錯的地方快。

結構設計

有些雙向門是被設計出來的，例如 feature flag、試點和可替換邊界。

判準 當下看起來像什麼 真正代價
可逆決策走重流程 看起來很穩 常把試驗速度和學習節奏一起拖死
不可逆決策走快路 短期很省事 一旦錯，代價常遠超過當初省下的時間
流程按可逆性分流 管理較細 卻更能同時保住速度與風險控制

Psychology

為什麼團隊容易在這裡失手

團隊常把一時好推進的決定誤當成可逆，只因為真正的回頭成本還沒被寫上白板。

形式公平

很多組織喜歡所有決策走同樣流程，因為看起來最一致。

速度誤配

一旦習慣快決，連難逆轉決策也會被同一種節奏拖走。

回頭幻覺

人常高估自己未來回頭重做的容易度，所以低估單向門風險。

Applications

把它帶回架構、流程與介面

回到架構選型、產品實驗與流程變更，這個框架很適合幫你重排決策節奏。

產品試驗

可快速回滾的體驗變更應該用小步快跑，而不是走完整月治理。

架構選型

一旦會綁死資料模型或供應商，決策就該升格成單向門處理。

團隊節拍

把決策先分類成可逆與難逆，會比先討論誰權限高更有效。

  • 產品試驗: 對可回滾變更採小步快跑，不要走重流程。
  • 架構變更: 對鎖死型決策提高資訊密度和反證要求。

Misreadings

這個概念最常被怎樣讀歪

若一開始就把 單向門與雙向門 聽成「所以所有決策都該快」，後面連「不是。只有可逆決策值得在較低資訊密度下快速前進。」這層校正都很難真正進場。

所以所有決策都該快 校正

不是。只有可逆決策值得在較低資訊密度下快速前進。

只要說成雙向門就能隨便試 校正

如果回滾成本其實很高，假裝它可逆只會讓風險失真。

這只是管理口號 校正

任何牽涉試錯、回滾與鎖定成本的系統都能直接受用。

Questions

把它變成下一次重構前的追問

這個決策如果做錯，回頭到底要花多少時間、資金和信任成本

先量出來，流程該快還是慢會清楚很多。

我們能不能先改設計，把單向門變成雙向門，再來追求速度

很多速度問題其實是結構設計問題。
