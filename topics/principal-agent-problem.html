委託代理問題

Principal-Agent Problem

當一方替另一方做決策，但兩者目標、風險承擔與資訊都不一致時，制度就會開始漏出代理成本。

Orientation

目標一分開，合作就開始出現視差

委託代理問題不是對人性悲觀，而是提醒你：只要決策權與後果承擔分開，系統就需要處理目標偏移與資訊不對稱。

Core Tension

委託人希望代理人替自己最大化結果，但代理人同時也在最大化自己的方便、風險控制與回報。兩者不會自動重合。

01 先找誰替誰做事

只要存在代行關係，就要假設兩邊的成本函數可能不同。

02 再看誰知道得更多

資訊不對稱讓委託人很難直接驗證代理人是否真的朝共同目標前進。

03 最後設計對齊機制

激勵、監督、透明度與退出權，都是降低代理成本的工具。

Setup

把問題縮到最小

這個問題之所以普遍，是因為現代組織幾乎處處需要授權；而一旦授權，偏移就不是例外，而是預設情況。

最小劇情

公司希望供應商替自己壓低維修成本並確保品質，但供應商最在意的是交期、利潤與風險暴露，於是雙方開始在看不見的地方調整行為。

真正難點

只要委託人看不到代理人的實際努力與局部權衡，就很容易把偏移誤解成態度問題，而不是結構問題。

把目標委託出去
→
資訊與後果分離

代理成本開始累積
Key Point

委託代理問題的重點不是抓壞人，而是承認：只靠口頭共識，通常不足以讓多方在長期裡持續對齊。

Lenses

三個看代理成本的角度

當合作開始變形時，可以從激勵、可觀測性與退出機制三面一起拆。

激勵不對齊

代理人被獎勵的東西，可能和委託人真正想要的結果不同。

可觀測性不足

委託人看得到產出，卻看不到過程與被犧牲的隱性品質。

更換成本過高

當退出或替換困難時，代理人更容易累積談判優勢與行為空間。

看法 看起來像 真正風險
只看表面產出 合作似乎正常 隱性品質、長期風險與替代成本被藏起來。
增加透明度 管理變繁瑣 成本變高，但能讓偏移更早被看見。
調整激勵與退出權 制度比較硬 前期設計辛苦，但能持續壓低代理成本。

Psychology

為什麼資訊落差會慢慢改寫彼此的選擇

人很容易把代理偏移讀成人格問題，因為那比承認制度本身沒有設計好來得直覺。

委託人會高估口頭對齊

只要大家都說目標一致，就容易以為實際行為也會自然一致。

代理人會優先優化可見目標

在壓力下，人通常先顧自己的評價與風險，而不是抽象的共同利益。

重點在於把合作失真從道德敘事拉回制度設計。

Applications

把代理失真帶回 vendor 與內部治理

工程 outsourcing、內部平台、管理匯報與供應鏈合作，都能用這個框架看見隱性成本如何產生。

委外開發

若供應商的 KPI 偏重交付件數，而你真正要的是可維護性與風險透明，偏移幾乎一定會發生。

內部平台團隊

平台團隊替產品團隊做基礎建設時，若成功指標只看 adoption，穩定性與例外支援可能被低估。

  • 遇到協作摩擦時，先畫出誰替誰做決策、誰承擔後果、誰握有關鍵資訊。
  • 若想降低代理成本，不只要對齊獎勵，也要降低替換成本與資訊不透明。

Misreadings

別把委託代理問題講成誰比較壞

這個框架最常被誤解成『大家都不可信』，那會錯過它真正有用的部分。

所以只能完全不授權 校正

不是。授權是必要的，關鍵在於承認授權需要制度支撐。

只要多監督就行 校正

監督重要，但若激勵與退出機制不動，監督很容易變成高成本的補丁。

Questions

看合作設計時，先問哪兩個對齊條件

如果你正在設計合作制度，下面兩個問題特別值得先問。

代理人真正被獎勵什麼

現在的制度是否在鼓勵代理人優化委託人的目標，還是只優化可見但不完整的替代指標？

委託人看不到什麼

有哪些重要風險、品質或努力成本，正因資訊不對稱而無法被日常追蹤？
