折衷效應

Compromise Effect

選項一多，中間方案常自帶安全感；它看起來不像冒進，也不像省過頭，因此很容易被誤認成最合理。

Read For

分辨「這個方案位在中間」和「這個方案真的符合需求」，不要讓位置替證據工作。

Orientation

中間位置會讓方案看起來已經做過權衡，即使真正的權衡還沒開始

折衷效應不是說折衷不好，而是提醒你：位置感會偷走需求檢查，讓人把不極端誤認成合理。

Core Tension

中間方案能降低後悔與辯護成本；但若它只是因為被兩端襯托才成立，決策就被選項排列牽著走。

01 先看集合怎麼被排出來

中間選項的吸引力取決於旁邊放了什麼；改變陪襯，偏好也可能跟著移動。

02 再問需求到底在哪一端

真正需求可能需要極簡、極穩或極高性能；中間不代表剛好，可能只是看起來不尷尬。

03 最後做獨立評估

把其他選項暫時拿掉，要求中間方案自己說明價值、風險與放棄的東西。

Setup

三個方案並排時，人會很自然地把中間那張牌拿起來

中間方案讓人覺得自己沒有被便宜誘惑，也沒有被昂貴誘惑；它提供的是一種容易辯護的選擇姿態。

最小劇情

定價頁有 Basic、Pro、Enterprise。Basic 少了幾個關鍵功能，Enterprise 顯得太貴，Pro 因此像自然選擇；但使用者可能只需要 Basic 加一個功能。

真正難點

中間方案的安全感來自比較，不一定來自需求。你要看它是否在沒有兩端陪襯時仍然站得住。

兩端製造不安
→
中間取得辯護優勢
→
需求檢查被壓縮
Key Point

折衷真正可靠時，應該能說出它保留了什麼、放棄了什麼，而不是只說它沒有那麼極端。

Lenses

從三個角度拆開中間方案的吸引力

中間看似穩妥，但它可能同時包含選項設計、責任規避與真實需求三種不同來源。

選項設計

若某方案只在特定排列下迷人，換一組比較對象就失去吸引力，它更像被架構出來的中間，而不是本身強。

責任規避

選中間很容易對上級或客戶解釋：沒有太省，也沒有太花。這種好解釋不等於好結果。

需求證據

把需求列成不可退讓、可交換與可延後三欄；若中間方案沒有命中不可退讓欄，它的位置再漂亮也沒用。

Psychology

人不是只在選功能，也在選一個比較不容易後悔的故事

折衷吸引人的原因之一，是它幫我們避開兩種後悔：買太少和買太多。

中間讓人像有節制

選最低價怕被說短視，選最高價怕被說浪費；中間方案剛好讓人看起來有控制力。

極端需要更多證明

選極簡或極高規格都需要明確理由。當理由還沒準備好，中間自然變成最低摩擦路徑。

比較取代了目標

選項越多，人越容易在相對位置裡找答案，反而忘記原本要解的是哪個具體問題。

折衷可以是成熟判斷，也可以是逃避判斷；差別在於它有沒有經過需求與代價的獨立檢查。

Applications

中間方案常出現在定價、架構與 roadmap 會議裡

只要選項是人設計出來的，中間位置就可能成為一種隱形推力。

產品定價

若 Pro 方案總是被選，先檢查 Basic 與 Enterprise 是否只是陪襯，再看 Pro 是否真的對應主力使用情境。

架構評審

「折衷架構」可能同時保留兩邊缺點。請明確列出它犧牲的性能、可維護性與 rollout 複雜度。

Roadmap 排序

中等風險、中等收益的項目常讓團隊舒服，但真正需要的也許是少數高風險高價值下注，或先清掉低風險基礎工作。

Misreadings

折衷不是原罪，無證據的折衷才是問題

這個概念要你檢查中間位置的來源，而不是把所有中間方案都打成操控。

以為中間一定錯 校正

很多成熟決策確實是折衷；判準是它是否回應需求，而不是它在排序中的位置。

把舒服感當成安全 校正

容易辯護的選擇不一定風險低。舒服感可能只是因為責任分散，不是因為結果更好。

用陪襯方案操控選擇 校正

設計選項時可以協助比較，但若故意放入不合理端點，就不是幫助決策，而是在製造偏好。

Questions

讓中間方案離開陪襯，自己站一次

以下問題適合放進採購表、pricing review 或架構決策紀錄。

拿掉兩端還想選嗎

如果只留下這個方案本身，它是否仍能用需求、成本與風險說服我？

它折掉的是什麼

這個折衷犧牲的是無關緊要的極端，還是剛好犧牲了真正重要的需求？

延伸路線

可接著讀 decoy effect，對照陪襯選項如何更直接地改變選擇。
