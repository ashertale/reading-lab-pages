不作為偏誤

Omission Bias

人常覺得因為自己主動做而造成的傷害，比因為自己沒做而發生的同等傷害更難承受，於是決策會偏向『至少不是我動手造成的』。

Read For

這一頁要把『什麼都不做比較安全』和『不做其實也是會造成後果的決策』拆開，因為很多保守選項只是把責任感藏進不作為裡。

主題類型
道德偏誤 / 行動不對稱
核心問題
Harms from action are judged more harshly than equivalent harms from inaction
主要判準
主動性、責任感、傷害可見度、道德評價
讀完產出
能分辨哪些風險評估是在比較後果，哪些其實是在比較自己對作為的心理負擔

Orientation

不做看起來像中立，卻常只是把後果包成比較不刺眼的版本

不作為偏誤提醒你：人對同樣結果的道德感受，會因為它來自作為還是不作為而不對稱。

Core Tension

謹慎避免亂動很重要，但若只因『主動做錯比較像我的錯』而拒絕必要介入，最終可能讓可避免的傷害留在系統裡。

01 先比較後果，不只比較感受

行動與不行動都可能造成傷害，不能只因責任感不同就當成兩回事。

02 再拆道德重量與實際風險

不作為的舒適感不代表它的後果比較輕。

03 最後誠實承認不作為也是選擇

如果你知道可能後果仍選擇不動，那就是一種主動決策。

Setup

從一個做也痛、不做也痛的風險選擇開始

不作為偏誤在醫療、風險治理、產品下架與事故處理裡都很常見。

最小劇情

你知道某功能有安全隱患，修補會暫時讓部分客戶流程中斷；若不修，未來可能讓更多人暴露風險。

真正難點

很多人不是沒看見後果，而是主動出手造成眼前損失的心理重量比較難背。

作為與不作為都帶後果
→
主動造成的傷害更刺眼
→
決策偏向保留現狀或延後介入
Key Point

不作為偏誤最麻煩的地方，是它會把『責任感較輕』偽裝成『風險較小』。

Lenses

四個看行動不對稱的角度

真正該比的是不同選項造成的總後果，而不是哪種形式比較不刺手。

後果對稱性

如果結果接近，自己是否只因主動性不同就改變偏好？

時間距離

作為造成的傷害常立刻可見，不作為的傷害則比較延後。

責任可見性

主動行為比較像可被指認的責任，不作為則容易被背景化。

制度期待

系統是否默默獎勵『先別動』，即使長期更危險？

情境 不作為偏誤看起來像 真正該問什麼
安全修補 先別 patch 以免影響現場 不 patch 的累積風險是否其實更大
醫療介入 不治療比較不像加害 不介入造成的後果是否真的較小
產品下架 先不要動使用者流程 延後下架是否在放大更高代價風險

Psychology

為什麼不作為常被感覺比較乾淨

因為不作為的因果鏈比較分散、比較慢、比較不像是『我的手造成』。

主動行為更像責任

出手造成的結果比較容易被歸到自己身上。

延後傷害比較不刺眼

不作為的後果常在時間上被拉開。

不動像在維持中立

即使現實上它同樣是一種立場，心理上卻比較像沒選。

不作為偏誤不是鼓勵亂動，而是要求把『不動』也當成要負責的選項來看。

Applications

把後果對稱帶回風險處理與變更治理

對工程、產品與安全團隊來說，很多『再等等看』若沒有對稱評估，其實是在不自覺地偏袒不作為。

漏洞修補

怕 patch 影響穩定而延後更新，若沒有同時量化暴露風險，就容易被不作為偏誤帶著走。

功能下架

不想主動拿掉舊功能，但繼續保留也可能讓更多人依賴更差的路徑。

事故升級通報

擔心主動升級引發緊張，可能反而延誤真正需要的協調。

  • 遇到作為/不作為兩難時，把兩邊後果放同一張表，比較的是總後果，不是刺手程度。
  • 若團隊對主動改動特別苛刻，對延後不動卻沒有同等檢討，不作為偏誤通常就有空間長大。

Misreadings

三個常見誤讀

不作為偏誤不是說應該總是主動出手，而是提醒不作為不能自動享有道德折扣。

不動就是比較保守 校正

保守不等於風險小，只是責任感不同。

只在倫理兩難才有 校正

工程修補、產品決策與管理升級都很常見。

看見這偏誤就該激進行動 校正

不是。重點是做對稱後果比較，而不是偏向另一端。

Questions

我是在評估後果，還是在逃避主動承擔的刺痛

下面兩個問題很適合放在高風險決策前。

若把這兩種後果都當成『我的決策造成』，我還會偏好不作為嗎

這能幫你把責任感差異拿掉。

現在不做的風險，是否被我像做了會出錯那樣認真量化

若沒有，比較本身就不對稱。
