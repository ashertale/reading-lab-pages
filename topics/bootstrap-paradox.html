Bootstrap Paradox

自舉悖論讓一件物品、資訊或想法在時間迴圈中存在，卻找不到最初創造它的人。

Orientation

東西一路被傳下去，但沒有人第一次寫下它

自舉悖論的壓力不在時間旅行本身，而在一個資訊或物件只靠循環維持，沒有外部創造時，我們還能不能說它有來源。

Core Tension

日常因果需要起點：某人寫書、發明公式、打造物件。時間迴圈卻可能讓結果成為自己的原因。

01 追蹤物件身份

同一本書、同一段程式或同一份資料是否在循環中保持同一身份？

02 追蹤資訊來源

資訊是被創造、複製、修正，還是只是在時間中傳遞？

03 追蹤熵與磨損

物件若物理存在，循環中老化、損耗和修復也要交代。

Setup

未來的你把一本書帶回過去，年輕的你照著它出版

最小場景是同一本書在時間中循環，卻沒有任何人原創內容。

最小劇情

你從未來取得一本小說，回到過去交給年輕作者。作者照抄出版，這本書後來又成為你帶回過去的那一本。

真正難點

書的每一次出現都有前一個原因，但整個循環沒有回答文本第一次從哪裡來。

未來物件回到過去
→
過去事件產生未來物件
→
來源問題被循環吞掉
Key Point

自舉悖論逼你分清『每一步都有原因』和『整體有外部起源』不是同一件事。

Lenses

把來源問題拆成身份、資訊與物理承載

不同版本的自舉悖論卡在不同地方：有些卡資訊來源，有些卡物件持續存在。

資訊閉環

內容是否只是被複製，沒有創作者？

物件閉環

物理物件如何避免無限老化或無來源材料問題？

版本漂移

循環每次是否完全一樣，還是微小差異會累積成另一條歷史？

Psychology

人習慣用線性故事找第一因

自舉悖論不舒服，是因為它保留局部因果，卻拿走我們想要的起點。

起源直覺很強

看到穩定物件或資訊，人自然會問誰先創造。

循環看似作弊

每一步都能說明，整體卻像把來源藏起來。

故事語法偏好開頭

敘事需要開端，但因果閉環可能沒有敘事式開端。

遇到自我維持的流程時，別只問下一步來自哪裡，也問整體是否需要外部注入。

Applications

在系統裡檢查自我複製的來源資料

時間旅行是想像實驗，但來源閉環在資料、模型與文件流程中很常見。

資料管線

報表若被下游修正後又回灌上游，必須標記哪個欄位是原始觀測。

模型訓練

模型輸出被當成訓練資料時，要追蹤 synthetic data 的比例和來源。

文件流程

規格文件互相引用時，要保留權威來源，避免引用循環變成事實來源。

Misreadings

不是所有循環都是自舉悖論

回饋迴路可以有外部輸入；自舉悖論特別在意來源是否完全被循環取代。

把循環都當矛盾 校正

有些循環只是穩態回饋，不一定抹掉起源。

只討論時間旅行 校正

資料 provenance 和模型回灌也會出現來源閉環。

忽略物理承載 校正

資訊閉環和物件閉環要分開處理。

Questions

問這份資訊第一次從哪裡來

這個內容是否有外部原始來源？

如果每個來源都指回另一份副本，就需要找權威起點。

循環中是否有版本漂移或品質衰減？

追蹤複製、修正、壓縮和回灌如何改變原始內容。

延伸路線

可接著讀 causal loop，理解因果閉環如何改變起點與結果的關係。
