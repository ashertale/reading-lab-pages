Orientation

最麻煩的不是做錯，而是不知道自己在做錯

鄧寧-克魯格效應抓到的是 metacognition 的落差：你需要某些能力，才能評估自己是否具備那些能力。

Core Tension

一個人可能因為知識不足而犯錯，又因為同樣的知識不足，看不見自己錯在哪；這讓自信與實力短期內脫鉤。

01 先分能力與校準

做得好與知道自己做得好，是兩件相關但不同的能力。

02 再看回饋系統

若環境沒有即時、可信的回饋，錯誤自評會被維持更久。

03 最後設計對照

真實案例、外部 review 與明確標準，比感覺更能修正過度自信。

Setup

先看一個很熟悉的學習階段

這個效應好用，不是因為它能拿來嘲笑新手，而是因為每個人都會在某些領域碰到它。

最小劇情

剛摸到某個新工具的人，看了幾篇文章就覺得架構已經很清楚，開始大刀重構，卻沒看到那些真正會在量產時出現的邊界。

真正難點

他不是故意自大，而是還沒累積足夠失敗樣本與判準來校正自己。

理解還很淺
→
自評工具也跟著淺
→
信心暫時高於能力
Key Point

這個效應最值得帶回工作現場的，不是拿來貼人標籤，而是提醒你為什麼好回饋系統如此重要。

Lenses

四個看校準失準的角度

當你懷疑自己或他人落入這種狀態時，可以先看這四件事。

知識碎片化

知道幾個名詞、案例與工具，會讓人過早感到自己已掌握全貌。

缺少負回饋

如果錯誤不會立刻出事，過高自評就更容易存活。

標準不清

沒有明確品質標準時，人會拿自己的感覺當尺。

比較樣本太窄

只看自己身邊少量例子，會讓自評建立在錯的基準上。

狀態 看起來像 更可靠的判斷方式
剛入門很有信心 好像學很快 看能否處理邊界條件與反例
反覆低估複雜度 只是樂觀 檢查是否真的能說出失敗模式
拒絕 review 有主見 看是否能根據證據調整立場
只會用成功案例證明自己 經驗豐富 加入失敗樣本與外部 benchmark

Psychology

為什麼不懂有時比半懂更危險

因為半懂會產生一種危險的流暢感：你知道夠多能講話，卻還不知道自己漏了什麼。

流暢感會偽裝成掌握感

能把概念講順，不等於能處理真實系統中的反例與 tradeoff。

人傾向保護自我形象

承認『我其實還看不懂』需要心理空間與安全感。

回饋常比想像中難吞

若團隊文化把錯看成羞辱，自評修正就更慢。

把這個效應用來理解回饋與學習，比用來嘲諷他人更有價值。

Applications

把校準問題帶回 mentoring 與 review

工程現場裡最常見的版本，不是公開自大，而是對複雜度、風險與完成度的系統性錯估。

估時與承諾

剛接觸某領域的人最容易說『這個兩天可以』，因為還不知道真正會卡住的地方。

Code review

看起來很完整的 patch，可能只是作者還沒看見 failure mode，所以覺得自己已經 cover 到了。

帶新人

真正好的 mentoring 不只是教做法，還要教『如何知道自己其實還不知道』。

  • 給新手的不該只有答案，還要有判斷品質的 rubric 與反例集合。
  • 回饋若只說對錯，往往不夠；更有用的是指出哪個判準缺席，讓自我校準工具長出來。

Misreadings

三種常見濫用

這個效應很容易被拿來當網路嘲諷梗，但那其實是把它用壞了。

只要自信高就是 Dunning-Kruger 校正

不對。核心不是自信本身，而是能力不足與自評失準一起出現。

這效應只發生在別人身上 校正

每個人都可能在陌生領域落入相同結構。

解法就是打擊信心 校正

真正有效的是提供清楚標準、外部回饋與可比較案例，而不是羞辱。

Questions

我是真的懂，還是只會講得很像懂

如果遇到反例，我能不能說出自己會怎麼改判

真正理解通常包含知道理論在哪裡會失效。

我的信心建立在什麼回饋上

是實際成果、外部 review，還是只是我目前覺得很順？
