虛假共識效應

False Consensus Effect

人很容易把自己的想法、偏好與常識感，誤當成大多數人也差不多會這樣看，於是低估歧異、溝通與對齊成本。

Read For

這一頁要把『這不是大家都會這樣想嗎』和『其實只是我周圍的人都這樣想』拆開，因為很多誤解都從過度投射開始。

主題類型
社會認知 / 觀點投射
核心問題
We overestimate how much others share our beliefs and preferences
主要判準
共享假設、觀點差異、樣本偏差、對齊成本
讀完產出

Orientation

覺得自己很普通，常讓我們低估別人其實並不站在同一張地圖上

虛假共識效應讓人把自己的世界模型看得比實際更普遍，於是很多必要說明被直接跳過。

Core Tension

假設他人和自己差不多能節省溝通成本，但若差異其實很大，這種節省最後只會變成更高的返工與誤判。

01 先懷疑自己的常識感

只因某事對你和你周圍很自然，不代表它對更大群體也自然。

02 再找真實分布

在沒有資料前，『大家應該都會』常只是樣本偏狹的幻覺。

03 最後把關鍵前提說出來

越覺得不言自明的東西，越可能最需要驗證。

Setup

從一句『這大家都懂吧』開始

虛假共識效應在產品設計、文件撰寫與團隊決策中尤其容易偷渡。

最小劇情

設計者覺得某個設定流程很直觀，因為自己和同溫層同事都這樣想；結果真實使用者第一步就卡住。

真正難點

真正被高估的常不是別人的理解力，而是自己觀點的普遍性。

把個人偏好當常態
→
省略對齊與驗證
→
差異在執行時才爆出來
Key Point

虛假共識效應最值得警覺的地方，是它會讓你連需要說明或驗證都沒意識到。

Lenses

四個看觀點投射的角度

當你覺得某件事太 obvious，不妨先問那個 obvious 是不是只活在你的局部樣本裡。

樣本來源

你的直覺來自多元人群，還是來自很像你的一小群人？

背景差異

使用者、跨部門或新成員的起點是否和你真的接近？

語意隱含

有哪些前提你省略了，只因你預設大家都知道？

驗證機制

你有沒有用觀察、資料或試用去檢查這份『共識』是真的？

場景 虛假共識看起來像 後續代價
產品設計 這操作很直觀啊 真實使用者大量迷路
團隊決策 大家應該都支持 沉默被誤當共識，執行期阻力才浮現
文件撰寫 這步不用特別解釋 讀者在作者看不見的地方反覆卡住

Psychology

為什麼我們這麼容易把自己當成平均值

因為自己的觀點永遠最熟悉，而周圍常又聚集了和自己相似的人。

同溫層會強化普遍感

如果每天接觸的人背景相近，就更容易把局部共識誤認成普遍共識。

自我觀點最容易取用

大腦在推測別人時，最便宜的做法就是先拿自己當樣本。

沉默很容易被讀成同意

沒有人當場反對，不代表大家真的理解或認同。

虛假共識效應不是叫你懷疑一切，而是提醒『我的直覺很常只是我的直覺』。

Applications

把真實分布帶回設計與對齊

對產品與工程團隊來說，越是在高手密度高的環境，越要警惕把自己當成典型使用者。

內部工具

作者熟得不能再熟的流程，常正好是新人最需要說明的地方。

跨部門合作

工程覺得理所當然的名詞，對 PM 或營運可能完全不是同一套地圖。

架構決策

你覺得一個 tradeoff 顯而易見，不代表其他角色也在用同樣優先序。

  • 只要你心裡浮現『大家應該都知道』，那通常就是最值得驗證的一句話。
  • 面向不同角色寫說明時，先假設共識不存在，通常比先假設共識存在更安全。

Misreadings

三個常見誤讀

虛假共識效應不是說大家永遠都不同，而是說你不能只靠自己的普遍感下注。

只要多數人同意就沒事 校正

問題常在你還沒真的驗證多數，就已經把它當既成事實。

這只會發生在社會議題 校正

產品、工具、流程與文件都非常常見。

知道自己有偏誤就不會投射 校正

投射仍很自然，只是你可以多補一步驗證。

Questions

這真的是共識，還是只是我周圍樣本的回音

下面兩個問題很適合放在設計與決策前。

如果把我自己和我最常合作的那群人拿掉，其他人還會這樣想嗎

這能幫你檢查樣本是否太窄。

我現在依賴的共有前提，有沒有被實際測過

沒有測過的共識，很多只是投射。
