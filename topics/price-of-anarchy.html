無政府代價

Price of Anarchy

每個參與者都照自己的局部成本行動，並不保證系統只是『沒那麼理想』；在某些網路裡，全員理性甚至會把整體成本明顯推高。

Read For

這頁想修正一個很常見的工程直覺：只要每個 agent 都在做對自己最好的選擇，整體結果應該不會差太多。無政府代價告訴你，這個保證根本不存在。

Orientation

每個人都走自己眼中最便宜的路，系統總帳仍可能更貴

無政府代價的尖銳之處，在於它不靠惡意角色。所有參與者都只是正常地追求自己的成本最小，整體卻還是能被擠到很差的位置。

Core Tension

局部最佳回應會形成均衡，但均衡只保證沒有個體想單獨改路，不保證整個系統已經接近最好的全域配置。

01 先分開均衡與最適

一個狀態可以很穩，因為沒人想單獨離開；但這種穩定和系統總成本低不一定是同一回事。

02 再找外部性去哪裡了

每個人做選擇時若沒有把自己加給別人的擁塞、等待或搶占成本算進去，整體就會偏離全局最適。

03 最後問能否改規則而不是罵參與者

很多時候問題不在『大家太自私』，而在協議、路由或獎勵設計沒有讓局部決策看見全局代價。

Setup

從同一條快車道塞爆開始

你不需要大型交通網才能理解它；一個共享佇列就很夠用。

最小場景

多個工作執行緒都會把任務丟到看起來回應最快的同一條 queue。短時間內每個執行緒都做了對自己最合理的事，結果那條 queue 因為被擠爆而變成全體最慢的路。

真正難點

沒有人故意做壞決策，也沒有人能靠單獨換路立刻改善自己很多；於是大家一起停在一個穩定但昂貴的均衡裡。

個體依局部成本選最便宜路徑
→
共享資源承受擁塞外部性
→
系統停在穩定但不划算的均衡
Key Point

無政府代價不是在說分散系統一定差，而是在說『穩定』本身不足以證明『好』。

Lenses

四個透鏡，拆開局部理性怎麼變成全局浪費

只要把外部性和均衡條件畫清楚，很多表面上像協作失敗的問題，其實是機制設計問題。

自利路由

每個 agent 根據自己眼前成本換路，未必會替系統做出成本最小的流量分配。

擁塞外部性

你加入某條熱門路時，不只是為自己省事，也把等待成本加給後來的人；這個帳若沒被計入，均衡就會偏掉。

均衡黏著

當所有人都停在互相擠壓的狀態時，單一參與者往往沒有足夠誘因先改，於是壞結果反而穩。

機制修正

排程器、價格、背壓或路由限制能把個體看到的局部成本調整得更接近全局成本。

你看到的狀態 局部看起來為何合理 全域為何變糟
大家都選同一條低延遲路 目前看起來最快 共同擁塞後把總等待時間推高
熱門 cache 被一窩蜂搶 命中率短期較高 回源壓力與鎖競爭被一起放大
任務都挑同一種機器型別 單機成本最低 整體利用率與尾端等待反而惡化
服務實例都把流量導向相同下游 健康檢查分數最好看 單一熱點成為整體瓶頸

Psychology

為什麼均衡很容易被誤認成合理終點

只要系統暫時不再劇烈移動，人就會把它看成『大概就這樣了』，忘記穩定也可能只是壞結果卡住了。

局部 dashboard 視角

每個人通常只看得到自己的 queue time、自己的 latency、自己的 throughput，很難自然感覺到自己對別人的外部性。

無人惡意的安心感

因為沒有明顯壞人，大家更容易低估系統性浪費，誤以為只要每個人都照規則做事就夠了。

均衡看起來像結論

當沒有人想單獨偏離時，我們很容易把『無人偏離』偷換成『已經夠好』。

這頁真正要校正的是一種判斷習慣：把穩定狀態誤當成設計已完成，而不是繼續追問它是否把外部性算進去了。

Applications

把它帶回排程、路由與共享資源

凡是多個 agent 共同搶一組資源，無政府代價都可能以不同名字出現。

多佇列排程

若任務只按眼前最短路徑自選 queue，常會形成熱門 queue 更熱門、冷門 queue 更閒的擁塞不對稱。

網路與服務路由

健康分數、短期 RTT 或局部快取命中率若被過度依賴，流量可能被共同推向同一熱點。

共享治具與驗證資源

每個團隊都想把工作丟到最熟、最快借到的治具上，長期卻會讓整體等待時間比集中排程更差。

  • 看到均衡時別急著滿意，先估一下它和全局最適之間的差距是否大到值得重新設計規則。
  • 真正有效的修法常不是要求大家更無私，而是把局部成本改寫得更接近真實全局成本。

Misreadings

三種很常見的過度延伸

無政府代價不是一句『分散就不好』，也不是中央控制的自動授權書。

所有分散系統都注定糟 校正

不是。很多分散系統的 PoA 很小；概念的價值在於量化差距，而不是先替任何架構宣判。

只要中央排程就一定更好 校正

中央也可能看不全、反應慢或成本太高。問題是怎樣的機制能縮小差距，而不是盲目崇拜集中。

這只跟交通路網有關 校正

只要存在共享資源、局部回應與外部性，queue、cache、路由、排程都會重演相同結構。

Questions

懷疑局部理性正在拖垮整體時，先問這兩題

每個參與者在做決策時，漏算了哪些加給別人的成本？

只要那個成本沒有回到局部視角裡，均衡就可能天然偏向擁塞與浪費。

如果不改人，只改規則、價格或排程，均衡會不會移到更好的位置？

這一問能把討論從責怪參與者，拉回真正可設計的機制層。
