Backward Induction

逆向歸納從最後一個選擇往前推，讓你看清楚今天的承諾是否會在未來節點上站得住。

Read For

適合談判、release gate、技術債償還承諾、roadmap sequencing 與任何今天選擇會改變明天選項的問題。

Orientation

先問最後會怎麼選，再決定現在怎麼走

在序列賽局中，逆向歸納從最後一個決策點開始，逐步往前推每個玩家在該節點的理性選擇。

Core Tension

很多計畫在今天看起來合理，是因為它假設未來會做某件不再有誘因的事；倒推會把這種承諾拆穿。

01 先畫終局

列出最後可選動作與各自 payoff。

02 再倒推節點

每個中間節點都假設玩家會選當時最有利的行動。

03 最後檢查承諾

若未來不會自願執行，需要現在創造可信約束。

Setup

這版先欠著，下版真的會還嗎

為了趕客戶 demo，團隊把 workaround 合進主線，會議紀錄寫著下版清理。

最小場景

到了下版，清理沒有新功能可展示，還可能引入 regression；新的 deadline 讓團隊再次延後。

真正難點

今天的承諾若沒有預留時間、測試與 owner，未來節點的誘因會把它推翻。

Key Point

逆向歸納把「之後再說」改成「到時候為什麼真的會做」。

Lenses

四個倒推視角

倒推不是算命，而是把選擇、誘因與承諾放到時間序列裡檢查。

Game tree

把每個節點的可選動作列出來，避免只看眼前一步。

Subgame

從每個局部遊戲看玩家在那一刻的最佳回應。

Credible commitment

若想讓未來行動可信，需要現在鎖定資源、制度或架構。

Path dependence

早期選擇會改變後續選項；不是每條路都能無成本回頭。

承諾未來誘因提高可信度
下版補測試 新功能又優先 把測試列為 merge gate
未來移除 legacy 客戶仍在用 先收集使用量並公布 deprecation date

Psychology

未來的自己通常比較忙

人很容易相信未來會更有空、更理性、更願意做不討喜的事。

規劃樂觀

今天低估未來需求，承諾看起來比實際便宜。

承諾折現

沒有立即成本的承諾很容易被說出口，也很容易被忘掉。

局部理性

每個節點單看都合理，整條路卻把系統帶到不想要的位置。

倒推讓計畫多一點冷感：不是問大家願不願意，而是問到時候還會不會願意。

Applications

工程決策常是序列賽局

Roadmap、重構、相容性與 release gate 都不是一次性選擇；每一步會改變下一步的誘因。

Release gate

若允許一次 bypass，未來每次 deadline 都會引用先例。

Technical debt

承諾償還要有 budget、owner 與阻止繼續借債的機制。

Platform compatibility

一旦承諾支援舊接口，未來移除會面對客戶與營收壓力。

  • 每個「之後」都要配一個未來節點：誰決策、當時誘因、拒絕成本、成功標準。
  • 若未來節點不會自然做對的事，今天就要設 gate、budget 或不可逆約束。

Misreadings

倒推不是只顧眼前利益

它不是叫你短視，而是讓長期計畫面對每個節點的實際誘因。

所有人都完全理性 校正

現實中人會犯錯；倒推仍可用來檢查承諾和誘因。

結論一定悲觀 校正

若有可信承諾，倒推也能支持長期投資。

只適合賽局理論課本 校正

工程 roadmap、deprecation 與談判同樣有序列決策。
