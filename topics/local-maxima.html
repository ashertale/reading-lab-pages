局部極大值

Local Maximum

有些系統會讓你一路做出局部更好的選擇，結果卻把自己卡在一座小山頂上，再也走不到遠方更高的山。

Read For

這一頁要把『每一步都比上一步好』和『整體最後真的最好』拆開，因為很多決策卡住，不是因為沒有進步，而是進步方式把自己困在局部盆地裡。

主題類型
最佳化陷阱 / 搜尋策略
核心問題
Improving nearby can trap you away from better global outcomes
主要判準

讀完產出

Orientation

局部一直變好，並不保證整體最後會到最好

局部極大值最迷人的地方，在於它讓你感覺自己一直在進步；最危險的地方，也正是在這裡。

Core Tension

若每次都只接受眼前更好的微調，你會很穩；但真正更高的方案，有時必須先下坡、繞路或承受短期不適。

01 先看目標地形

如果問題空間充滿山谷與高地，只做局部改進很容易卡住。

02 再保留探索預算

沒有試錯空間時，團隊自然只會走看起來最安全的鄰近改良。

03 最後分辨穩定與停滯

有時你以為系統成熟了，其實只是再也不敢離開現在這座小山。

Setup

從一個明明每次改完都比較順，整體卻一直跨不過去的系統開始

局部極大值在演算法、產品與組織策略裡都很常見，只是名稱不同。

最小劇情

某產品持續微調既有流程，短期數據每次都略有提升，但始終無法跨進真正能打開新市場的產品型態。

真正難點

真正更好的方案，可能需要先拆掉一部分現在有效的東西，這對團隊來說很難承受。

持續做局部更好的改動
→
系統在小範圍內穩定上升
→
更高價值方案因需要跳脫而一直碰不到
Key Point

局部極大值提醒你：短期單調進步，有時正是長期停滯最舒服的樣子。

Lenses

四個看最佳化陷阱的角度

不是每個穩定平台都叫局部極大值，但當更大躍遷必須先承受局部損失時，這個概念特別有用。

鄰近性偏好

團隊是否只願意做看起來立刻變好的小改動？

探索成本

更遠的方案是否需要短期指標下滑、重訓成本或組織重配？

地形可見性

你是否根本看不見其他高地，只看得到眼前周圍？

切換護欄

有沒有 sandbox、試點或分階段策略，讓你能在不全押的情況下試離開現在位置？

情境 局部最優看起來像 全域代價可能是什麼
產品改良 現有流程越調越順 錯過需要重構體驗才能打開的新價值
系統性能 一直微調參數 核心架構問題被延後，永遠碰不到更高上限
團隊流程 現有會議與分工越來越熟 其實整體協作模式已不適合目前規模

Psychology

為什麼局部變好這麼容易讓人留下來

因為單調進步會帶來強烈安全感，而需要先變差一點的躍遷對多數團隊都很反直覺。

損失比收益更刺眼

先下坡的方案即使終點更高，也更難被接受。

已有成果會被保護

一旦現狀能交付，大家自然會把它當成不可輕動的資產。

遠方高地不夠具體

新的更好狀態若不夠可視，團隊很難為它承受短期不適。

局部極大值不是叫你亂跳，而是提醒你別把穩定小進步自動等同於全域正確方向。

Applications

把探索預算帶回架構、產品與流程演化

對工程團隊來說，局部極大值最常出現在『每次修一點都比較好，但怎麼總覺得天花板很低』的地方。

架構演進

一路 patch 舊系統很有產值，但可能讓真正該拆的核心越來越拆不動。

模型調參

一直微調現有 pipeline 可能卡住，真正突破可能來自換資料、換目標或換方法。

團隊工作法

現有流程再優化也許只能把摩擦降一點，真正的改善可能需要重新定義 ownership。

  • 若你懷疑自己卡在局部極大值，請刻意保留一點資源做不保證立刻進步的探索。
  • 不要只問現在怎麼再好 5%，也要問要到下一個級別是否需要先換地形。

Misreadings

三個常見誤讀

局部極大值不是所有停滯的萬用解釋，但它很適合提醒你檢查是否過度偏好鄰近改良。

所以應該常常大翻修 校正

不是。探索也有成本，重點是知道何時小修已經碰到天花板。

只要目前變好就一定沒問題 校正

局部指標變好不代表整體目標方向沒偏。

這只發生在演算法 校正

產品策略、組織設計與個人習慣都可能卡在同樣形狀。

Questions

我現在是在穩定爬坡，還是在舒服地卡住

下面兩個問題很適合放在策略回顧時。

若要達到下一級成果，是否勢必得先承受一些短期指標變差

如果是，代表你可能已接近局部極大值。

我們現在的改善是否只在局部最方便的位置打轉

若是，就需要刻意增加探索角度。
