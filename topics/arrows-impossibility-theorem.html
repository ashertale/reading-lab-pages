阿羅不可能定理

Arrow's Impossibility Theorem

阿羅不可能定理說明，在看似合理的公平條件下，把個人排序彙整成集體排序不可能同時滿足全部要求。

Orientation

沒有一套投票規則能免費保住所有公平直覺

阿羅定理的力量在於：問題不只是某個投票制度設計不好，而是在一組合理條件下不存在完美彙整規則。

Core Tension

我們想把個人偏好變成集體選擇，又希望規則尊重一致性、無獨裁和選項獨立；這些願望彼此衝突。

01 列出公平條件

先說規則要保住哪些性質，而不是只說要民主或公平。

02 承認犧牲

任何實作都會放棄某個條件或限制問題範圍。

03 看策略誘因

彙整規則會改變人如何表達偏好。

Setup

三個選民各有合理排序，集體排序卻可能循環

最小場景是 A 勝 B、B 勝 C、C 又勝 A。

最小劇情

三位成員對三個方案排序不同。兩兩投票時，A 多數勝 B，B 多數勝 C，但 C 又多數勝 A；集體偏好無法形成穩定線性排序。

真正難點

每個人的排序都一致，問題出在彙整後的集體排序不一定一致。

個人偏好各自合理
→
多數比較產生循環
→
集體排序無法穩定
Key Point

阿羅定理要求設計者說清楚：你的排序系統打算犧牲哪個公平條件。

Lenses

把彙整規則的犧牲攤開

不同投票和排序系統不是誰純粹公平，而是各自選擇不同代價。

無獨裁

是否避免單一人的偏好總能決定集體排序？

帕累托

若所有人都偏好 A 勝 B，集體是否也如此？

獨立無關選項

A 和 B 的相對排序是否不受 C 的存在影響？

Psychology

人容易把不喜歡的結果歸咎於制度壞，而不是條件互相衝突

投票結果常讓輸家覺得規則不公平；阿羅定理提醒我們，完美公平規則可能根本不存在。

公平直覺很多元

不同人重視代表性、穩定性、簡單性或抗操弄。

制度透明反而暴露代價

說清楚犧牲什麼會讓制度看起來不完美，但也更誠實。

排序被誤當客觀

產品榜單或模型排名常像事實，其實背後有彙整選擇。

設計 ranking 或 voting 時，不要承諾完全公平；承諾你選擇哪種公平，並說明代價。

Applications

在排序和投票系統中明示取捨

產品排名、社群投票和多準則決策都會遇到偏好彙整問題。

產品排序

多指標 ranking 要說明權重和不可保證的公平性質。

社群投票

投票制度不同會影響少數偏好、策略投票和候選人排序。

決策會議

用多輪投票或評分前，先承認結果依賴彙整規則。

Misreadings

不可能定理不是說集體決策沒用

它說沒有符合全部條件的完美規則；實務仍可選擇適合情境的折衷。

把它當反民主 校正

定理不是否定投票，而是揭露投票規則的限制。

以為所有規則一樣差 校正

不同情境可以選不同代價。

忽略問題限制 校正

限制偏好形狀或選項數，有時能避開部分困難。

Questions

問排序規則犧牲了什麼

下面兩問適合用在投票制度、ranking 和評分系統設計。

我們要求這個彙整規則保住哪些公平條件？

列出無獨裁、帕累托、獨立性、抗操弄或簡單性。

哪個條件在本情境可以放掉？

把犧牲說清楚，避免把排名包裝成客觀事實。

延伸路線

可接著讀 social choice theory，理解個人偏好如何被彙整成集體選擇。
