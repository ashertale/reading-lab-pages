Kelly Criterion

有優勢不代表該押滿；Kelly 準則問的是在不確定且可重複的賭局裡，該投入多少比例才不會讓成長被波動毀掉。

Orientation

押對方向仍可能押錯大小

Kelly 準則在可重複且有機率優勢的賭局中，選擇最大化長期對數成長率的下注比例。

Core Tension

下注太少浪費優勢，下注太多則讓波動與連敗吞掉本金；最危險的是把不確定估計當成確定優勢。

01 先估 edge

沒有正優勢，Kelly 不會神奇創造收益；輸入估錯時，輸出會很危險。

02 再算比例

重點是投入比例，不是只判斷要不要參與。

03 最後打折

實務上常用 fractional Kelly，承認模型誤差、風險承受度與不可重複性。

Setup

一個看起來很划算的專案

想像團隊發現某個新架構很可能降低 20% 成本，但若判斷錯，會拖慢主線六個月。

最小場景

正期望值不代表要把全隊押上去；你可以用小團隊、prototype、里程碑與停止條件控制投入比例。

真正難點

工程討論常只問該不該做，卻少問應該用多少資源下注，以及估計錯誤時能不能活下來。

Key Point

Kelly 的工程啟發是：不要把一個好機會變成過大單點風險。

Lenses

從下注大小看風險配置

Kelly 不只談機率，也談資本路徑；長期成長被大幅回撤破壞後，很難靠平均期望值挽回。

優勢大小

勝率與賠率共同決定合理比例；高勝率但小收益不一定值得重押。

波動與回撤

過度下注會讓連續失敗造成不可恢復損失，即使平均期望值為正。

估計誤差

工程機會的勝率與收益常是粗估，應對 Kelly 結果保守打折。

不可重複性

若只玩一次或輸了會失去關鍵能力，對數成長模型不能直接套。

決策語句 Kelly 會追問 工程轉用
這方向值得做 值得押多少？ 先做 bounded prototype。
勝率很高 賠率與估計誤差呢？ 把收益、損失與信心分開列。
錯了再說 錯了會不會出局？ 保留主線與回退資源。

Psychology

信心會偷偷變成倉位

人一旦相信某個方向，就容易把資源集中視為果斷；Kelly 把信心與投注比例拆開。

正確方向的過度自信

看對趨勢不等於承受得住路上的波動。

全押顯得有魄力

組織文化有時獎勵大膽投入，卻低估失敗後失去選項的成本。

估計被包裝成事實

勝率、收益與損失多半不是精確數字；模型越漂亮，越需要折扣。

Kelly 的冷靜感來自一句話：edge 是方向，size 是生存。

Applications

用在專案投注與風險分層

Firmware roadmaps 常要在平台重構、新功能、良率改善與可靠性投資間下注；Kelly 可作為比例思考。

架構重寫

若收益大但估計誤差也大，先以 spike 或平行路徑下注，避免全隊停擺。

供應替代

即使新料成本優勢明顯，也不要在 qualification 前把所有版本押到單一新來源。

可靠性投資

若某 defect class 反覆造成 field cost，可逐步增加資源，而不是只在事故後一次性大押。

  • 把每個大機會拆成勝率、收益、損失與投入比例，而不是只寫優缺點。
  • 對粗估機會使用折扣後比例，並設 milestone 重新估算。

Misreadings

Kelly 不是叫你精準賭博

在工程決策裡，它更像一個提醒：估計越不確定，投入比例越不能裝作精準。

公式能給唯一答案 校正

輸入通常有誤差，fractional Kelly 或更保守策略才常見。

有 edge 就該最大化 校正

最大化長期成長不等於符合組織風險、現金流或任務約束。

可直接套到一次性決策 校正

一次性、不可逆或輸了會出局的決策，需要更保守的風險框架。
