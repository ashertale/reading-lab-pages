沉沒成本謬誤

Sunk Cost Fallacy

當過去投入越多，人越容易把繼續投入誤認成理性，而不是重新評估未來價值。

Read For

這頁要幫你看清：沉沒成本不是叫你輕易放棄，而是提醒你，過去已經不可回收的投入不應綁架下一個決策。

Orientation

已經投入的東西，為什麼總想逼你繼續投

沉沒成本謬誤讓人把停止看成浪費，把繼續看成負責；但真正浪費的可能是讓壞決策繼續消耗未來。

Core Tension

已花掉的時間、錢、名聲和情感不能被拿回來；可是它們會讓退出變得痛，於是人會用更多投入來證明前面的投入沒有白費。

01 先標出不可回收

把已經無法回收的成本寫出來，並明確禁止它成為繼續投入的理由。

02 再估未來現金流

只看從現在開始的收益、成本、風險與機會成本。

03 最後設退出條件

專案啟動時就定義 kill criteria，避免到了該停時只能靠情緒判斷。

Setup

把問題縮到最小

最小情境是：一個專案越做越不像會成功，但因為已經投入很多，大家反而更難停下來。

最小劇情

團隊已經花三個月重寫工具鏈，但新工具仍不穩。繼續投入的理由從「會更好」慢慢變成「都做這麼久了」。

真正難點

問題不是投入本身，而是投入變成身分、承諾與面子的證據。此時停止不只是策略選擇，還像承認前面錯了。

過去投入已不可回收
→
心理上不願承認損失
→
繼續投入以維持故事
Key Point

沉沒成本謬誤的核心檢查是：如果今天從零開始知道現況，你還會選擇同一個方向嗎？

Lenses

四種讓人不願停下的力量

沉沒成本會透過財務、情緒、社會承諾與組織流程一起施壓。

不可回收成本

已經花掉的時間與金錢不能回來，但它們會在心理上要求一個合理化故事。

損失厭惡

承認停止會讓損失變得具體，繼續則讓人感覺還有機會把它救回來。

公開承諾

如果專案曾被高調宣布，停下來會牽動面子、績效與跨部門信任。

機會成本失焦

繼續投入會占用本可用在其他更高價值工作的時間與注意力，但這個成本常被低估。

決策問題 謬誤版本 較健康版本
是否繼續專案 已經花很多，不能停 從今天起的預期收益是否仍高於替代方案
是否保留功能 這功能做很久了 是否仍有人使用、維護成本是否合理
是否重構到底 都拆一半了 剩餘風險與收益是否仍支持繼續
是否承認方向錯 現在停太丟臉 越早停越能保留未來資源

Psychology

為什麼付出越多，人越難承認該停

人不只在意結果，也在意自己過去的選擇是否看起來合理。

不想承認損失

停止會把損失從暫時帳面變成心理上已結案的失敗。

想維持自我一致

如果我曾經堅持這是好方向，現在反轉會威脅到我對自己判斷力的看法。

組織獎懲錯位

有些文化懲罰取消專案，卻不懲罰長期消耗，於是大家寧願拖著。

它把行為經濟學的沉沒成本概念，轉成產品與工程決策中的退出設計。

Applications

把它帶回工程與產品

沉沒成本謬誤常出現在工具重寫、平台遷移、功能保留、供應商選型與專案收斂。

平台重寫

重寫到一半發現風險比預期大時，應比較剩餘成本與替代路線，而不是用已完成比例逼自己繼續。

功能清理

很少人用的 feature 即使曾花很多心力，也可能該移除；該看的是目前價值與維護成本。

硬體選型

已投入某供應商 bring-up 不代表必須量產到底；若風險升高，要重新看交期、品質與替代方案。

  • 在專案開頭定義 kill criteria，例如性能、穩定性、成本或客戶採用率低於門檻就重新決策。
  • review 時要求所有繼續投入理由都指向未來價值，而不是已花成本。

Misreadings

三個常見誤讀

沉沒成本謬誤不是鼓勵遇到困難就放棄，而是要求把決策基準放回未來。

所以投入過的都該放棄 校正

不是。若未來價值仍高，繼續是理性的；問題是不能只因為投入過就繼續。

只適用金錢 校正

不只。時間、名聲、情感、政治資本和團隊士氣都可能變成沉沒成本。

停下來就是浪費 校正

很多時候，停下來是在阻止浪費擴大，讓資源回到更有機會的地方。

Questions

把它變成你的專案檢查

下面兩個問題適合在 milestone review 或專案轉向前使用。

如果今天從零開始，我還會選這條路嗎

如果答案是否定的，繼續的理由就必須非常強，而且要能指向未來收益。

繼續投入的機會成本是什麼

把同一批人、時間和 budget 放到其他工作上，是否能換到更高的可靠性或客戶價值？

延伸路線

延伸閱讀可看 loss aversion、escalation of commitment、opportunity cost；也可以回到產品管理裡的 kill criteria。
