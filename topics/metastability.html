亞穩態

Metastability

亞穩態不是少見怪事，而是非同步訊號進入雙穩態元件時無法完全消除的機率風險。

Read For

適合 clock domain crossing、GPIO async input、FPGA/ASIC synchronizer 與高可靠硬體 review。

主題類型
數位電路 / 時序風險
核心問題
Signals near decision boundaries may not settle in time
主要判準
setup/hold、MTBF、同步級數、clock domain
讀完產出
能用機率與架構降低 CDC failure，而不是追求不存在的完全消除
邊界 延遲 機率

Orientation

數位電路也有無法立刻決定的瞬間

當輸入在 setup/hold window 附近變化，雙穩態元件可能短暫停在不穩定的中間狀態，輸出延遲才 settle。

Core Tension

同步器不能讓亞穩態不發生；它只能提供更多時間讓狀態收斂，把錯誤傳到下游的機率降到可接受。

01 先承認非同步邊界

GPIO、reset、跨 clock domain 訊號都可能觸發亞穩態。

02 再給收斂時間

多級 synchronizer 增加 resolve time，改善 MTBF。

03 最後處理語意

單 bit、pulse、多 bit bus、counter 都需要不同 crossing 策略。

Setup

外部 interrupt 剛好撞上 clock edge

一條外部 ready 訊號沒有同步就進入 CPU domain；在某次邊界條件下，狀態機收到半拍脈衝。

最小場景

第一級 flip-flop 輸出延遲 settle，第二級在錯誤時間取樣，形成偶發且難重現的控制脈衝。

真正難點

測試台 clock 相位固定時可能永遠看不到；field 中獨立時脈會慢慢掃過所有相位。

非同步變化
→
踩進 setup/hold 邊界
→
同步器降低但不消除失敗機率
Key Point

CDC bug 常不是邏輯錯，而是時間邊界被架構忽略。

Lenses

把亞穩態當機率工程處理

正確問題不是會不會發生，而是失敗率是否低於產品壽命與安全需求。

Setup/Hold

違反取樣窗口會讓 flip-flop 無法在預期時間輸出穩定 0 或 1。

MTBF

用 clock rate、data rate 與器件參數估算平均故障間隔。

同步級數

多一級增加 latency，也增加亞穩態收斂時間。

多 bit crossing

多位元不可用獨立兩級同步器直接搬，否則位元可能來自不同時間。

訊號類型常見策略注意
單 bit level 兩級或多級 synchronizer 確認 MTBF 與 latency
Pulse toggle 或 handshake 避免脈衝太窄被漏掉
多 bit data FIFO、gray code、valid handshake 避免 bit skew 造成混合值

Psychology

最難說服的是它沒在模擬裡出現

亞穩態事件稀有、相位敏感、又常被 digital simulation 抽象掉，因此容易被低估。

可重現性低

錯誤可能幾天一次或特定溫壓下出現，很像隨機 field issue。

二值思維

工程師習慣 0/1 邏輯，卻忽略物理電路需要時間決定。

固定相位測試

tester clock 若和 DUT 固定相位，可能剛好避開最危險區域。

CDC review 的價值在設計階段最大，因為等 field issue 出現時通常已很難定位。

Applications

每個跨時脈邊界都要有名字

實務上要把 CDC path 清單化：來源 domain、目的 domain、訊號語意、同步策略與驗證方法。

GPIO async input

外部按鍵、interrupt 或 sensor ready 進入 system clock 前要同步與去彈跳。

FPGA domain crossing

不同 PLL domain 間使用 async FIFO 或 handshake，不要直接接 bus。

Reset crossing

reset deassertion 也需要同步，避免不同 flop 在不同 clock edge 脫離 reset。

  • CDC lint、design review 與 waiver 要一起看；waiver 應說明為何安全。
  • 對 pulse crossing 設計最小寬度或 toggle protocol，避免快 domain 到慢 domain 被吃掉。

Misreadings

同步器是降低機率，不是魔法消除

亞穩態的教訓是謙虛：物理層沒有承諾在有限時間內永遠完成判斷。

兩級同步器保證安全 校正

它只是讓 failure probability 很低；高頻或高 data rate 可能需要更多分析。

模擬通過就沒問題 校正

一般 RTL simulation 不會自然呈現亞穩態收斂時間。

多 bit 各自同步即可 校正

各 bit 可能在不同 cycle 被取樣，組成從未存在過的值。
