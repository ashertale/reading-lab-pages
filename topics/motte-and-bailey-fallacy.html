城堡與外庭謬誤

Motte-and-Bailey Fallacy

有些主張會先用很大、很有擴張力的版本去拿好處，一旦被挑戰，又退回一個較弱、較安全、幾乎沒人反對的版本自保；真正讓人難受的不是弱版本，而是兩者被來回偷換。

Read For

學會辨認一個說法是不是在不同壓力下偷偷換版本，避免和一個永遠會後退的主張纏鬥。

Orientation

最耗人的論戰之一，就是對方永遠有一個更安全的版本可以退回去

這題真正難的地方，不在背立場，而在承認：只要一個人能在擴張主張拿好處、在受挑戰時退回安全版本，他就能同時享受兩個版本的優勢，卻不必承擔一致責任。

Core Tension

只要一個人能在擴張主張拿好處、在受挑戰時退回安全版本，他就能同時享受兩個版本的優勢，卻不必承擔一致責任。

01 先把強版本和弱版本分開寫下來

只要版本被拆開，偷換空間就會立刻縮小。

02 再問剛才拿好處的是哪個版本

如果宣傳、權威或道德壓力靠的是強版本，就不能只用弱版本來防守。

03 最後要求單一責任邊界

主張若要被接受，就應接受它在相同版本下被檢查。

Setup

從最小思想實驗看張力在哪裡

城堡與外庭謬誤要先看到的，不是誰講話大聲，而是主張如何在壓力來時偷偷換殼。

最小劇情

某團隊先高調宣稱『我們的 AI 會自動理解使用者真正意圖』，被追問失誤與責任邊界時，又退回『我們只是說它能提供一些輔助建議』。

真正難點

難點在於較弱版本往往本身並不荒唐，所以人很容易在追問過程中忘記最初其實被主張的是更大的版本。

先提出擴張性很高的主張
→
被質疑時退回安全版本
→
壓力一過又繼續借大版本拿好處
Key Point

城堡與外庭謬誤最重要的識別點，不是某個版本對不對，而是兩個版本是否被偷換來回使用。

Lenses

從不同判準看同一題

這種謬誤真正難抓的，不是主張強或弱，而是它在受壓時究竟躲回哪個安全版本。

擴張版本

外庭通常範圍大、吸引力強，能替說話者拿到注意力或道德位置。

防守版本

城堡通常較窄、較安全，讓人很難直接反對。

偷換節點

真正要抓的，是何時從大版本滑回小版本，又何時偷偷滑回去。

判準 它會保住什麼 代價
主張與防守用同一版本 辯論比較辛苦 但責任邊界清楚
主張大、退守小 短期很有彈性 對話品質和責任界線會一起崩掉
不追版本切換 表面較客氣 實際上很容易被牽著在不同命題間繞圈

Psychology

為什麼這題會讓人卡住

人會被這種話術卡住，常因為自己在回應哪個主張這件事已經被對方悄悄改掉。

大話吸引力

大版本比較容易拿到注意力、認同或權威位置。

退守安全感

一被質疑就退到弱版本，能快速降低被打穿的不適。

旁觀者失焦

若沒有把版本寫清楚，旁觀者很容易只記得最後那個較安全的說法。

Applications

放回真實判斷後它會改變什麼

公共辯論、產品宣稱和內部提案都怕這種切換，因為責任常在最需要釘住時被悄悄抽走。

產品宣傳

不要讓行銷大話和實際規格邊界在不同場景裡來回切換。

政策辯論

當對方退回安全版時，提醒大家最初被拿來動員的到底是哪一版。

內部提案

把 claim、assumption 和 actual guarantee 分欄，能大幅降低偷換。

  • 產品敘事: 把宣稱版和保證版拆開寫，避免偷換。
  • 政策辯論: 追問最初動員用的是哪一版主張。

Misreadings

別把它讀成單純表態

若一開始就把 城堡與外庭謬誤 聽成「只要立場有層次就是謬誤」，後面連「不是。問題是層次被偷換使用，而不是本來就有細緻版本。」這層校正都很難真正進場。

只要立場有層次就是謬誤 校正

不是。問題是層次被偷換使用，而不是本來就有細緻版本。

弱版本就一定不對 校正

弱版本可能完全合理，問題在於它不能替強版本背書。

這只是辯論技巧 校正

產品宣稱、政策口號和內部規格常一樣會中。

Questions

下次遇到相似判斷先問這兩句

你現在 defend 的這個版本，和你剛剛拿來主張或動員的是同一個版本嗎

這通常是拆掉偷換的第一步。

如果只保留你現在這個安全版本，原本那些較大的承諾、聲量或資源還拿得到嗎

若拿不到，偷換大概已經發生。
