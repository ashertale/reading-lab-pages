Secretary Problem

當選項依序出現且拒絕後不能回頭，最難的不是評估當前選項，而是決定何時停止比較。

Orientation

看太少會草率，看太久會錯過

秘書問題描述一串候選人依序出現，每次都必須立即接受或放棄，目標是最大化選到最佳者的機率。

Core Tension

早期接受可能錯過後面的更好選項；一直觀察則可能讓真正最佳者在探索期被拒絕。

01 先用前段校準

經典策略會先觀察一段樣本，不接受，只用來建立門檻。

02 再抓第一個超越門檻

探索期後，遇到比之前所有候選更好的選項就停止。

03 最後承認模型限制

真實世界常可回頭、候選分布不隨機、評分也不只相對排名。

Setup

到底要不要等下一個候選方案

想像你要選一個供應商、測試治具或設計方案；候選依序出現，每個都有報價期限。

最小場景

前幾個方案讓你建立品質與價格的感覺，但若一直等，你可能錯過已經足夠好的方案；若太早定案，又可能少看市場。

真正難點

真實決策常把『再等等』說成謹慎，卻沒計入時間、機會與候選流失成本。

Key Point

秘書問題的價值不是硬套 37% 規則，而是把探索期與停止條件提前寫清楚。

Lenses

停止規則背後的三個條件

經典解法依賴隨機順序、不可回頭、只追求最佳者等假設；用到現場時必須先檢查。

隨機順序

若候選順序被市場、推薦或內部流程偏置，經典機率不再直接成立。

不可回頭

若可保留候選、談判或排隊，策略會不同；但延遲仍有成本。

單一最佳

工程方案常有多目標，未必存在全局最佳；門檻要按關鍵維度設定。

時間成本

探索期越長，學得越多，也越晚交付並增加錯過成本。

設計方案 探索期要有限 多目標權重與 prototype 成本。

Psychology

再看看聽起來總是安全

延後決策給人保留選項的感覺，但序列選擇裡，等待也會消耗真正可用的候選。

錯過不可見

拒絕掉的好選項不會一直提醒你，它只在後面候選變差時才讓人後悔。

完美主義偽裝成審慎

一直等更好，可能是在逃避停止條件，而不是增加資訊。

早期樣本被過度代表

前幾個候選若特別好或特別差，會扭曲門檻；要小心樣本偶然性。

停止規則能降低事後自責：你不是憑情緒停，而是照事前同意的門檻停。

Applications

用在供應商、方案與 release 時機

Firmware 專案裡，很多選擇都有不可逆成分：零件、工具、架構、供應商、release date。

供應商評估

先看一批候選校準價格、交期與品質門檻，避免第一個可用方案就鎖死。

方案選型

Prototype 階段要有結束條件，不然探索會吞掉實作時間。

Release timing

等待更多 bug 修完會增加信心，也會延後客戶價值；停止條件要明確。

  • 在開始搜尋前定義探索期、最低門檻與不可妥協條件。
  • 若可回頭或可並行保留候選，明確寫出保留成本，不要把它當免費。

Misreadings

37% 不是萬用人生公式

秘書問題的經典數字依賴特定假設；真正可轉用的是停止規則思維。

照 37% 就最佳 校正

若候選順序不隨機、可回頭或評分多維，比例需要調整。

只適合招募 校正

任何不可逆序列選擇都可借用它的探索/停止框架。

停止就是放棄更好 校正

停止是承認搜尋成本；不是否定後面可能有更好選項。
