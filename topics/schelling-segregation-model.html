謝林隔離模型

Schelling Segregation Model

謝林隔離模型展示：每個個體只要有溫和的局部偏好，整體也可能走向高度分隔。

Orientation

溫和偏好也能堆出強烈分隔

Schelling 模型把個體放在格子上，讓他們依鄰居組成決定是否移動；簡單規則能產生出乎意料的整體格局。

Core Tension

個體可能只想要少量熟悉感，最後卻形成高度同質群聚；宏觀結果不能直接從單一個體意圖讀出。

01 先看局部規則

每個代理人只根據附近狀態做決定。

02 再看迭代過程

一次小移動會改變別人的鄰域，觸發下一輪移動。

03 最後看整體 pattern

系統層級可能出現個體沒有明說追求的結果。

Setup

每個人只想稍微相似，社群卻越來越分開

一個平台推薦規則偏好相似興趣；使用者只要覺得 feed 稍微舒服就留下，久了之後跨群內容越來越少。

最小場景

每次推薦只多給一點同溫內容，使用者也只多互動一點；整體網路逐步變成分隔社群。

真正難點

沒有單一 actor 下令隔離，pattern 卻從局部規則和反覆互動中長出來。

局部偏好設定
→
代理人移動或互動
→
宏觀群聚浮現
Key Point

模型價值在提醒你：系統結果可能比個體偏好更極端。

Lenses

讀這個模型要同時看規則與限制

簡化模型能提供洞見，也會省略現實中的權力、制度與歷史。

偏好閾值

每個代理人能接受多少異質鄰居，會影響分隔速度與程度。

空位與移動

能否移動、移動成本與可選位置會改變結果。

局部到全局

局部滿意不等於全局整合；每個人都滿意時，整體可能很分隔。

模型邊界

現實隔離還包含政策、經濟、歧視與歷史因素，不能被格子模型取代。

模型元素系統對照設計問題
鄰居偏好 推薦相似度、team fit 是否過度放大同質性
空位 可加入社群或職缺 是否限制跨群流動
移動成本 轉組、轉社群、換工具 摩擦如何改變長期格局

Psychology

人會把系統結果簡化成個人態度

看到分隔結果時，我們容易直接推回個體意圖，而忽略規則、摩擦與迭代效果。

意圖歸因

宏觀分隔可能被解讀成大家都強烈排斥，其實模型顯示溫和偏好也足以放大。

均衡錯覺

每個人局部滿意，不代表系統整體健康。

模型過度延伸

漂亮模擬會讓人忘記它省略了制度與資源不平等。

這個模型最有用的地方，是訓練你在個體理由之外看互動規則。

Applications

產品與組織都會長出分群

只要系統有局部偏好、移動選擇與反覆互動，就可能出現 emergent segregation。

推薦系統

稍微偏好相似內容，可能逐步降低跨群曝光。

組織分組

大家都想和熟悉技術或熟人合作，久了會形成知識孤島。

社群設計

若新手只能看見同類新手，學習路徑可能被早期分群鎖住。

  • 用 agent-based simulation 試探局部規則的長期 pattern，而不是只看單次 A/B 指標。
  • 設計跨群橋接機制時，同時降低移動成本並提供安全的混合情境。

Misreadings

模型不是現實的全部原因

謝林模型能說明局部規則如何放大分隔，但不能替制度、歷史與權力因素消失責任。

所以都是個人偏好造成 校正

現實分隔常有政策、資源與歧視結構，模型只展示一種機制。

個體沒有惡意就沒問題 校正

沒有惡意仍可能產生壞的整體結果。

只要改個人態度 校正

若規則和摩擦不變，系統可能繼續產生同樣 pattern。
