磨耗平均

Wear Leveling

Flash 位址看起來固定，底下的實體區塊卻有擦寫壽命；磨耗平均把寫入分散開，避免最常改的邏輯資料先把某幾格用死。

Read For

適合看 boot counter、config storage、log ring、SSD/SD 卡壽命、EEPROM emulation 與現場設備長期資料保存。

Orientation

同一筆設定不該永遠磨同一格 flash

Wear leveling 透過映射、搬移與輪替，讓邏輯上的熱資料不要集中消耗少數實體 block。

Core Tension

應用程式想要簡單地更新同一個檔案或變數；儲存介質需要把擦寫成本分散到更大的實體空間。

01 先估寫入熱點

counter、log header、allocation table 和設定頁常比資料本體更熱。

02 再抽象實體位置

logical address 不應永久綁定同一個 physical erase block。

03 最後保掉電恢復

mapping、valid bit、CRC 與 commit marker 必須能在斷電後重建。

Setup

Boot counter 每天寫，十年後先壞的是那一頁

一個看似無害的 uint32 counter，如果每次開機都覆寫同一 EEPROM page，壽命會由最熱的那幾個 cell 決定。

最小場景

產品每天重啟 50 次，boot_count、last_error 與 config checksum 都寫在同一頁。

真正難點

平均寫入量看似很低，但 erase granularity、metadata 寫入與掉電重試會放大局部磨耗。

邏輯位址反覆更新
→
FTL 或韌體搬到新實體區

Key Point

Flash 壽命不是看資料有多大，而是看哪些 erase block 被反覆消耗。

Lenses

分散磨耗時要看熱資料和冷資料

只搬動一直更新的資料是 dynamic wear leveling；若冷資料永遠占著新 block，壽命仍可能不平均。

Dynamic leveling

熱資料每次更新改寫到不同 block，避免同一位置被快速用完。

Static leveling

偶爾搬動長期不變的冷資料，讓低磨耗 block 也加入輪替。

FTL mapping

Flash translation layer 把 logical block 對應到 physical block，並維護有效/無效資料。

Garbage collection

回收含少量有效資料的 block 時，搬移成本與寫放大會出現。

設計選項得到什麼代價
簡單 ring buffer 容易實作、適合小型 EEPROM emulation 容量利用率與搜尋恢復成本上升
FTL mapping 邏輯介面穩定、分散寫入 mapping table 與掉電一致性複雜
Static leveling 讓冷資料所在 block 也被輪替 背景搬移造成額外寫放大

Psychology

小小一筆設定最容易被當成沒成本

工程師會警覺大量 log，卻常忽略 counter、flag、last state 這種每次事件都寫的小資料。

容量錯覺

資料只有 16 bytes，不代表它對 erase block 的消耗也只有 16 bytes。

測試時間太短

開發板跑一週沒壞，無法代表 field 十年的擦寫分布。

把 SD 卡當硬碟

消費級卡的內部 wear leveling 與掉電保護常不可見，不能替工業需求背書。

可靠性 review 要把寫入頻率換算成 erase cycles，並把 metadata 寫放大一起算進去。

Applications

從 EEPROM emulation 到產品事件紀錄都會碰到

只要資料需要長期保存且會反覆更新，就該設計磨耗與斷電恢復，而不是等 field 壞頁出現。

EEPROM emulation

用多個 flash page 輪替保存 key-value，靠 sequence number 與 CRC 找最新有效版本。

Event log

以 circular log 寫入事件，保留 erase block 對齊與壞塊處理策略。

Config storage

低頻但關鍵的設定更新要有雙分區、commit marker 與回滾版本。

  • 把最大寫入頻率、最小保固年限、erase cycle spec 與可用 block 數放進同一個計算表。
  • 斷電測試要在 erase、program、mapping update、commit marker 寫入各階段觸發。

Misreadings

平均磨耗不是壽命無限

它只能改善分布；資料保持、壞塊、寫放大與掉電恢復仍會限制可靠性。

控制器會全部搞定 校正

裸 NOR/NAND、MCU flash、EEPROM 和 SD 卡的保護層級差異很大。

Ring buffer 就完整 校正

ring buffer 分散寫入，但仍要處理有效版本搜尋、GC 與斷電中間態。

只看平均寫入量 校正

最熱 metadata 或 allocation table 可能比 payload 早到壽命上限。
