基本歸因謬誤

Fundamental Attribution Error

看到別人做出糟糕選擇時，我們很容易先想到他這個人怎麼樣，卻低估情境、資訊與制度摩擦其實對行為有多大塑形力。

Read For

這一頁要把『這人就是不行』和『這個情境本來就很容易把人推錯』拆開，因為許多協作衝突都在高估人格、低估系統。

Orientation

別人出錯時，我們太容易先看見他的缺點，而不是他所站的地形

基本歸因謬誤抓到的是一種很穩定的不對稱：看自己時強調情境，看別人時強調人格。

Core Tension

把問題快速讀成人格缺陷很省力，但它會讓你跳過更能被修復的介面、資訊與制度原因。

01 先問他看見了什麼

很多看似荒謬的行為，是在資訊不完整或心智模型錯位下很可預期的結果。

02 再問情境怎麼推

流程設計、優先序壓力與獎懲結構都會塑造行為。

03 最後才談人格模式

若要上升到人格推論，至少要看到跨情境、重複且更穩定的證據。

Setup

從一次你差點想直接給人貼標籤的失誤開始

基本歸因謬誤在衝突、事故與跨部門合作裡特別容易發作。

最小劇情

對方把需求做錯，你第一反應是『他很不用心』；後來才發現規格版本混亂、命名誤導且 review 邊界不清。

真正難點

人格解釋看起來最完整，但往往也最容易讓真正可修的機制被跳過。

看到他人失誤
→

→

Key Point

基本歸因謬誤最值得警覺的地方，是它會把系統問題偽裝成個人問題。

Lenses

四個看歸因失衡的角度

不是說人格永遠不重要，而是說它通常不該成為第一個、也是唯一一個解釋。

資訊落差

對方當時拿到的資料、限制與時序，和你現在看到的一樣嗎？

制度摩擦

流程是否本來就容易讓人遺漏、誤解或被迫取捨？

重複模式

這是單次情境失手，還是跨場景都反覆出現的穩定行為？

角色視角

換成你站在同樣資訊與壓力下，真的一定不會犯同樣錯嗎？

現象 人格歸因版本 情境歸因版本
回覆很慢 他不負責 可能卡在權限、資訊來源或優先級衝突
把需求做反 他不細心 可能規格含混、術語誤導或版本不同
會議態度差 他難合作 可能長期壓力、責任不清或溝通結構已失衡

Psychology

為什麼人格解釋這麼有吸引力

因為人格敘事簡單、完整、情緒上也更有結尾感。

人格故事更省力

用性格解釋別人的行為，比重建情境與制度更快。

情境常不可見

外部觀察者看不到對方當下的限制與資訊缺口。

責怪能快速收束情緒

一旦有了『就是他』的故事，混亂會立刻變得有出口。

這個偏誤不是叫你永遠替人找藉口，而是提醒你別太快把診斷偷換成宣判。

Applications

把情境視角帶回事故與協作

對工程與跨團隊合作來說，能不能抵抗基本歸因謬誤，直接影響你修的是人還是系統。

Incident review

若每次都先找誰出包，流程與介面缺陷就會持續藏在同樣位置。

Code review

看到奇怪寫法時，先問是設計限制、歷史相容還是知識落差，比直接貼能力標籤更有用。

跨部門協作

把他人摩擦先讀成態度問題，常會讓原本能修的對齊機制變成長期敵意。

  • 每次想給人貼標籤前，先列出三個足以產生同樣行為的情境因素。
  • 如果最後真的要談責任，也請先把系統摩擦寫清楚，避免下次重演。

Misreadings

三個常見誤讀

基本歸因謬誤不是說人格不重要，而是說我們通常太早太重地把它拿來解釋他人。

所以都是系統的錯 校正

不是。只是責任判斷不該跳過情境分析。

這和 Hanlon's Razor 完全一樣 校正

兩者相近，但這個偏誤更聚焦於高估人格、低估情境的心理機制。

只在衝突時才有 校正

日常回覆、評論與管理判斷也都很常見。

Questions

我現在看到的是這個人，還是這個人加上他所處的地形

下面兩個問題很適合放在評價他人前。

如果把同樣情境換到另一個人身上，我還會這麼確定這是人格問題嗎

這能幫你看見情境力量被低估的程度。

對方當時有哪些限制是我現在站在事後看不見的

很多錯誤解釋會在這一步被鬆開。
