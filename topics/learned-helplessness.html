習得性無助

Learned Helplessness

如果一個人反覆學到努力不會改變結果，他可能在後來有路可走時也不再嘗試。

Read For

分辨能力不足和控制感被破壞的差異，避免把不行動誤判成懶惰或抗拒。

主題類型
心理學 / 控制感

Repeated failure teaches passivity

Orientation

人停止嘗試，有時是因為過去教會他嘗試也沒用

習得性無助描述的是控制感被反覆破壞後的退縮；它看起來像不努力，實際上可能是行動和結果失去連結。

Core Tension

失敗經驗能讓人校準現實，但若每次努力都沒有可見回饋，人會學到不要浪費力氣。

01 先檢查控制路徑

問這個人是否真的有可以影響結果的槓桿，而不是只要求他更積極。

02 讓小行動產生可見結果

重建控制感需要短回饋、低門檻和可觀察改善。

03 避免懲罰式環境

若制度讓人怎麼做都被否定，後續不行動可能是合理適應。

Setup

一個團隊多次提出問題都沒被處理，後來就不再提了

最小場景是人曾經行動過，但行動和結果之間沒有穩定連結。

最小劇情

韌體團隊連續幾季回報測試設備不足，每次都被要求先用臨時 workaround。後來新專案再出現風險時，大家只私下抱怨，不再正式升級。

真正難點

表面上是團隊消極，實際上是流程反覆教會他們：正式提出問題也不會改變資源分配。

嘗試沒有改變結果
→
控制感逐步下降
→
可行動時也不再出手
Key Point

習得性無助的校正，不是喊話要積極，而是讓行動重新看得見效果。

Lenses

把不行動拆成能力、權限與歷史回饋

同樣是不動，背後可能是沒有技能、沒有權限，或過去學到動也沒用。

技能缺口

是否知道該怎麼做？如果不知道，需要訓練與範例。

權限缺口

是否有權限改變結果？如果沒有，要求負責只會加深無助。

回饋歷史

過去行動是否真的曾被採納、回應或帶來改善？

Psychology

控制感是行動的燃料，不只是情緒安慰

人需要相信自己的行為和結果有連結；這個連結斷掉後，連簡單行動都會變重。

失敗可被學習

反覆不可控的失敗會教人預期未來也不可控。

低能量是保護

如果努力一直沒有結果，減少嘗試是一種節省成本的適應。

重新嘗試需要證據

單次鼓勵通常不夠，必須讓小行動真的產生可見變化。

修復無助感的第一步，是提供一個小到能成功、真到能改變的控制點。

Applications

讓人看見自己哪一步真的有用

團隊流程、使用者產品和個人成長都需要設計可見控制感。

團隊流程

對風險升級建立回覆 SLA 和採納紀錄，讓提出問題不再像丟進黑洞。

使用者體驗

錯誤訊息要給下一步和可修復範圍，避免使用者只學到自己做什麼都錯。

個人成長

把大目標切成能快速得到回饋的小練習，讓努力和改善重新接上。

Misreadings

無助不是性格定案

習得性無助是從經驗中學來的，也需要透過新的經驗逐步拆掉。

把它當成懶惰 校正

不行動可能是過去回饋設計造成的學習結果，不是單純態度問題。

只靠激勵口號 校正

沒有控制點的鼓勵會再次證明努力無效。

一次成功就算修復 校正

控制感需要穩定累積，不能靠單次例外完全恢復。

Questions

重建最小可控制點

下面兩問適合用在流程改善、coaching 和產品錯誤設計。

這個人哪一步真的能改變結果？

如果找不到清楚控制點，就先不要把問題定義成態度。

下一個小行動會得到什麼可見回饋？

回饋要短、具體且真的能影響後續選擇。

延伸路線

可接著讀 locus of control，理解人如何解釋自己對結果的控制程度。
