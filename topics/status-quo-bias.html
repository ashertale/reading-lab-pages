現狀偏誤

Status Quo Bias

維持現狀常被感覺成中立，但很多時候它其實也是一種有代價的選擇，只是風險被藏在不作為裡。

Read For

這一頁要把『先不要動』的安心感和『其實已在為現狀付費』拆開，因為很多保守決策真正躲的是被看不見的代價。

主題類型
決策偏誤 / 預設依附
核心問題
People disproportionately prefer the current state
主要判準
預設力量、切換成本、損失感、替代方案清晰度
讀完產出
能分辨哪些決策是在理性保守，哪些只是被預設與慣性推著留在原地

Orientation

不改，不代表沒有選；很多時候只是把選擇包裝成自然延續

現狀偏誤提醒你：人對現有配置的偏好，常超過它實際值得的程度。

Core Tension

保留現狀能降低短期不確定，但也會讓過時設計、低效流程與錯失機會持續累積成本。

01 先把現狀也當成選項

只要看見維持原狀同樣有成本，判斷就會比較公平。

02 再拆開真風險與熟悉感

很多『先不要』其實不是風險分析，而是對不熟的抗拒。

03 最後降低切換摩擦

若替代方案很難試、很難退，現狀偏誤就會被制度放大。

Setup

從一個大家都知道不理想、卻一直沒換掉的流程開始

現狀偏誤常不會以戲劇方式出現，而是以一連串『先這樣就好』慢慢定居。

最小劇情

團隊都知道現有 issue 流程很繞，但因為搬遷成本、學習成本與短期不確定，最後大家還是繼續用。

真正難點

真正被高估的常不是新方案風險，而是現狀的無形安全感。

現狀成為預設
→
替代方案看起來有切換痛
→
維持原狀被誤當成中立
Key Point

現狀偏誤最值得警覺的地方，是它會把時間推移本身造成的代價藏起來。

Lenses

四個看預設慣性的角度

不是所有維持現狀都錯，但你至少要知道自己在守什麼。

熟悉紅利

現狀是不是因為熟悉，而不是因為真的更好？

切換摩擦

新方案是否因導入過程麻煩而被過度懲罰？

隱性成本

現狀正在持續吞掉哪些時間、品質或機會？

可逆程度

如果新方案可試可退，現狀偏誤通常就比較容易被鬆動。

情境 現狀偏誤看起來像 被遮住的成本
工具更換 大家都習慣了別動 效率損失與知識摩擦持續累積
產品預設 先維持目前流程 使用者一直在繞路但數據未必立即顯示
職涯選擇 現在至少安全 長期磨耗與成長停滯被視為背景雜訊

Psychology

為什麼現狀看起來總比較沒風險

因為現狀的代價多半分散在時間裡，不像改變那樣在前面就顯得刺眼。

熟悉感被誤讀成安全

人很容易把已知的不舒服看得比未知的可能性更可承受。

切換痛會被放大

導入與學習成本通常在短期非常可見。

不作為像是沒有責任

選擇不改時，失敗常被解讀成自然延續而不是決策結果。

現狀偏誤不是反保守，而是反對把熟悉誤認成中立。

Applications

把預設意識帶回流程與產品設計

對工程與產品團隊來說，很多該改的東西之所以沒改，不是因為評估後不值得，而是因為現狀自己會說話。

流程改革

若沒有低風險試點，大家自然會高估改變成本、低估原流程成本。

產品預設設定

使用者常沿用預設，不代表預設真最適合，只代表切換本身有摩擦。

技術債治理

架構拖著不動時，現狀偏誤會把『先別動』包裝成穩健。

  • 做選項比較時，把『維持現狀』也寫成顯性方案，連同代價一起估。
  • 若想對抗現狀偏誤，先把替代方案做成可逆、可試、低承諾的版本。

Misreadings

三個常見誤讀

現狀偏誤不是說所有不變都錯，而是提醒你很多不變其實不是中立。

只要願意改就是進步 校正

不是。重點是不要讓預設本身偷走判斷。

保守就是偏誤 校正

保守可以理性，但要建立在看見現狀成本之後。

這只發生在個人選擇 校正

制度、產品預設與團隊流程都會放大它。

Questions

我是在保護價值，還是在被預設牽著走

下面兩個問題很適合放在變更評估前。

如果今天從零開始，我還會選現在這個預設嗎

若不會，現狀偏誤就很可能已在工作。

不改的成本是否也被我像改的成本那樣認真看待

若沒有，評估本身就已經歪了。
