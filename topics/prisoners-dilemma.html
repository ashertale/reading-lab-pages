囚徒困境

Prisoner's Dilemma

當每個人都選擇對自己最安全的行動，整體結果反而可能比互相信任更差。

Orientation

理性的自保，為什麼能把雙方一起帶壞

囚徒困境的刺點在於：每個人單獨看都像做了理性選擇，但兩個理性選擇加起來，卻比雙方合作更差。

Core Tension

如果我相信你會合作，我背叛能多拿好處；如果我相信你會背叛，我更應該先背叛保護自己。於是背叛成了個體優勢策略，合作卻是群體較好結果。

01 先分清楚個體與群體

個體最優不一定堆成群體最優，這是困境的核心。

02 再看能否承諾

不能溝通、不能綁約、不能懲罰時，合作很難被保護。

03 最後設計制度

重複互動、透明紀錄與可信承諾能改變背叛的成本。

Setup

把問題縮到最小

經典版本裡，兩個嫌疑人分開審訊，各自選擇保持沉默或供出對方。兩人都不知道對方怎麼選。

最小劇情

如果兩人都合作，彼此得到較輕結果；如果一方背叛而另一方合作，背叛者得到最好結果；如果兩人都背叛，雙方都比合作時更差。

真正難點

困境不是不知道共同利益在哪，而是每個人都怕自己單方面合作後被利用。合作需要被制度保護，不只是被道德呼籲。

雙方都知道合作較好
→
個體仍有背叛誘因
→
理性自保導向較差均衡
Key Point

囚徒困境訓練你不要只問「大家是否願意合作」，還要問「合作在這個制度裡是否是安全選擇」。

Lenses

四種看合作困境的透鏡

同一個合作失敗事件，可以從 payoff、信任、時間尺度與制度設計切開來看。

Payoff 結構

如果背叛在任何對方選擇下都比較安全，合作就會變成脆弱選項。這不是溝通技巧可以單獨解決的問題。

資訊與透明度

看不見對方行動時，人會假設最壞情況。透明紀錄能讓合作和背叛留下可追蹤後果。

重複互動

一次性互動鼓勵拿完就走；重複互動讓聲譽、報復與修復機會進入計算。

可信承諾

合作需要承諾可以被相信。合約、流程、review gate 與 shared ownership 都是在替承諾加硬度。

選擇結構 個體感受 系統結果
雙方合作 需要相信對方也守約 共同收益較高。
單方背叛 背叛者短期最賺 合作方被懲罰，信任受損。
雙方背叛 每個人都覺得自己在自保 形成較差但穩定的均衡。
制度保護合作 背叛成本提高 合作開始變成理性選擇。

Psychology

為什麼缺乏信任時，背叛總像更穩的選擇

人不一定想破壞合作；很多時候，人只是害怕成為唯一守規則的人。

被利用的恐懼

合作方承擔風險，背叛方拿走額外收益。只要這種不對稱存在，人就會把背叛當保險。

對稱懷疑

我懷疑你會背叛，你也懷疑我會背叛。即使雙方都不想要壞結果，互相預期也會把壞結果推近。

短期壓力

deadline、績效與責任歸屬會放大短期自保。長期合作價值如果不進入評分，就很容易被犧牲。

重點是說明信任如何被 payoff 結構塑形，而不是把合作失敗歸咎於單一人格特質。

Applications

把 payoff 結構放回協作制度

跨模組、跨團隊、跨供應商合作很常像囚徒困境：大家都知道透明協作較好，但每一方都有理由先保護自己。

Release 風險揭露

每個團隊都怕自己先揭露 blocker 會被貼標籤。若制度懲罰揭露而獎勵表面綠燈，最後所有人都更晚知道真風險。

Module ownership

若 bug 歸屬決定績效，團隊會投入大量精力證明不是自己的問題。共同 debug 的收益高，但個體誘因鼓勵甩鍋。

供應商協作

供應商怕承認問題影響商務，客戶怕共享資訊被反咬。沒有可信承諾時，雙方都保留資訊，根因分析就變慢。

  • 讓早期揭露風險得到保護，不要只獎勵「從來沒有紅燈」。
  • 把合作成果和共同修復納入評分，降低單方背叛的短期好處。

Misreadings

三個常見誤讀

囚徒困境常被講成「人很自私」，但那會錯過制度設計的重點。

所以人性就是不可信 校正

不是。它說的是在特定 payoff 下，自保會壓過合作。改 payoff，行為就可能改變。

只要叫大家溝通就好 校正

溝通有幫助，但如果背叛仍然低成本高收益，光靠善意很難穩定合作。

合作一定是道德選擇 校正

合作也需要條件。沒有透明、承諾與懲罰機制時，要求單方合作可能只是要求單方承擔風險。

Questions

把它變成你的制度設計工具

在這個制度裡，先誠實的人會不會吃虧

如果最早揭露風險的人承擔最多責任，大家自然會晚一點說、少一點說、模糊一點說。

你如何讓合作變成安全選擇

需要透明紀錄、共同收益、失信成本、重複互動，還是第三方仲裁？少哪一個會讓合作崩掉？
