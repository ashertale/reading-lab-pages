慈惠原則

Principle of Charity

慈惠原則要求你先把對方論點理解成最合理版本，再批評它；這不是客氣，而是避免打倒稻草人。

Read For

分辨善意詮釋、鋼人化、論點重建和真實分歧，讓討論更接近問題本身。

主題類型
論證原則 / 理解對手
核心問題
Interpret others in their strongest reasonable form
主要判準
善意詮釋、鋼人化、分歧定位、批評品質
讀完產出
能在 code review、技術爭論和跨部門溝通中先重建對方最佳論點

Orientation

先打對方最強版本，才知道自己是否真的贏了

慈惠原則不是要求同意，而是要求在批評前避免故意或無意地把對方說法弱化。

Core Tension

快速反駁很省力，也很容易讓討論變成誤解對誤解；善意重建會慢一點，但能把真正分歧浮出來。

01 先重述再反駁

用對方會點頭的方式描述其主張。

02 補足合理背景

若一句話有多種解讀，先選較合理、較一致的版本。

03 保留批評權

慈惠不是放過錯誤，而是讓批評打在真正論點上。

Setup

你說這個架構太複雜，可能其實是在說風險邊界還沒講清楚

最小場景是一場技術討論因措辭粗糙而偏離主題。

最小劇情

review 中有人說新架構太複雜。若直接回他抗拒變更，討論會升溫；若先重建為他擔心故障域、oncall 和 migration 成本，團隊就能針對具體風險討論。

真正難點

對方原話可能不精準，但其中仍可能有值得處理的訊號。

Key Point

慈惠原則讓討論從抓語病，轉向找出最值得回應的版本。

Lenses

把對方說法翻譯成可檢查的論點

好的詮釋要既公平又可被對方修正。

語意修復

對方詞不達意時，能否補成合理表述？

背景假設

這個主張依賴哪些經驗、風險或價值排序？

分歧焦點

重建後剩下的真正不同意在哪裡？

Psychology

人很容易把反對者想成笨或壞，這能節省理解成本

慈惠原則刻意增加一點理解成本，用來降低錯誤衝突成本。

動機歸因

反對意見常被解讀成不配合或不懂。

勝負心

弱化對方能讓自己更快顯得有理。

群體身份

內外團隊對立時，慈惠會變得更難。

在反駁前加一句我理解你最強的意思是，常能把整場討論降溫。

Applications

把慈惠原則制度化到 review 流程

Code review、架構會議和跨部門談判都能用重建步驟降低誤傷。

Code review

先說你理解作者要保護的 tradeoff，再指出缺口。

架構討論

把反對意見翻成風險假設和驗證需求。

跨部門溝通

先重建對方 KPI 壓力，再談自己的限制。

Misreadings

慈惠原則不是無限替對方找藉口

如果最合理版本仍然錯，就應該清楚批評；慈惠提升批評品質，不取消判斷。

把善意當同意 校正

你可以公平理解後仍強烈反對。

忽略惡意行為 校正

反覆操弄或不誠實不需要無限慈惠。

替對方改到面目全非 校正

重建版本要讓對方能認領。

Questions

問對方會不會認同你的重述

我能否用對方會接受的方式重述這個論點？

若不能，先問清楚再批評。

最強版本仍然失敗在哪裡？

讓批評集中在真正缺口。
