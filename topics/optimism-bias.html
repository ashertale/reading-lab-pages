樂觀偏誤

Optimism Bias

人知道壞事會發生，卻常相信它比較不會發生在自己身上；這讓希望感變成低估風險的入口。

Read For

學會把「這次真的不同」拆成可檢查證據，而不是讓自己自動站到風險分布之外。

Orientation

樂觀偏誤不是不知道風險，而是覺得風險比較像別人的統計

它最常出現在我們承認一般情況很糟，卻相信這次自己的情況剛好不在一般情況裡。

Core Tension

樂觀能支撐行動與承諾；但沒有基準率與預案約束時，樂觀會把緩衝、備援與壞情境測試都推到太晚。

01 先拿回基準率

看相似專案、相似人群或相似決策的歷史結果，避免只從內部劇本估計未來。

02 再檢查差異證據

若你說這次不同，需要說清楚是哪個條件不同，以及它如何改變失敗機率。

03 最後把希望轉成預案

真正成熟的樂觀不是取消緩衝，而是讓團隊有能力在壞情境出現時仍能前進。

Setup

每個延期專案在開始時，都覺得自己不是會延期的那一類

樂觀偏誤的最小場景，是你看見過去案例，卻把它們當背景故事，而不是拿來校準自己的估計。

最小劇情

團隊知道相似 migration 平均要六週，仍估四週，理由是這次需求比較清楚、人也更熟。直到整合測試開始，才發現未知工作沒有因此消失。

真正難點

有時這次真的不同；難的是分辨「不同」是有證據的條件變化，還是讓自己保持士氣的故事。

承認一般風險
→
把自己列為例外
→
緩衝與預案被削薄
Key Point

樂觀偏誤不一定讓你說謊；它更常讓你只選擇那些支持順利劇本的真話。

Lenses

把樂觀拆成願景、證據與承壓能力

希望可以保留，但它要和外部資料、反例與預案一起進入計畫。

願景

願景負責讓人願意投入，但願景不是估時工具。它可以說明想去哪裡，不能單獨說明多久會到。

差異證據

如果這次比歷史案例更安全，應該能指出具體差異：更小範圍、更成熟工具、更少依賴或更強驗證。

承壓能力

樂觀計畫仍要能承受壞情境。緩衝、分階段 rollout 與 kill switch 是讓希望落地的結構。

Psychology

相信自己會比較順利，能讓人開始行動，也會讓人少看壞情境

樂觀有動機價值；問題是在估計風險時，動機價值容易被誤認成機率證據。

希望感保護行動

如果每次都先想最壞情況，人可能不敢開始。樂觀讓人投入，但也因此不想太早面對摩擦。

身份感要求例外

團隊常覺得自己比平均更努力、更聰明、更懂脈絡；這些可能是真的，卻不必然足以改變基準率。

壞情境傷士氣

提早討論失敗會讓會議變沉重，因此風險常被放到「之後再細化」，直到之後變成現在。

校正樂觀偏誤，不是降低野心，而是讓野心配得上足夠的風險承載能力。

Applications

讓樂觀變成可執行的結構，而不是口頭保證

專案、release 與安全習慣都需要希望；但希望要被轉成緩衝、早期訊號與可退路徑。

專案估時

先用相似專案的外部資料給初始估計，再讓團隊提出可驗證差異；不要反過來先寫理想劇本。

Release 風險

若你相信這次會順利，請把信念轉成 staged rollout、監控指標、rollback 條件，而不是取消預案。

安全與健康習慣

人知道事故會發生，但常低估自己中獎機率。把防護設計成預設流程，比提醒自己小心更可靠。

Misreadings

校正樂觀偏誤不是把團隊變悲觀

目標不是消滅希望，而是避免希望偷走資料、緩衝與反證。

把風險討論當作唱衰 校正

好的風險討論是替承諾加地基，不是拆承諾。

用士氣替代證據 校正

士氣讓人願意做事，證據讓人知道怎麼配置資源；兩者不能互相冒充。

事後把錯全怪給人 校正

樂觀偏誤常是流程問題：估時、review 與預案設計都要讓壞情境有位置。

Questions

讓「這次不同」交出證據

下面兩問適合放在 planning、risk review 與 personal safety check 裡。

我的例外證據是什麼

哪些具體條件讓這次不同於過去失敗案例？它們如何被驗證，而不是只被相信？

壞情境已經有路嗎

如果最可能的延誤或失敗真的發生，我們是否已有緩衝、降級、回復或停止條件？

延伸路線

可接著讀 planning fallacy，對照樂觀如何在專案估時裡變成系統性延誤。

Sources
