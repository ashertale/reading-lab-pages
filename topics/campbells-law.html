坎貝爾定律

Campbell's Law

當一個社會指標被拿來做強力決策，它不只會失真，還可能反過來腐化原本想改善的制度過程。

Read For

這一頁要把『量化管理』和『量化把現場搞壞』拆開，因為真正危險的不是有指標，而是指標被拿去承擔過多控制權。

Orientation

一旦分數開始決定命運，現場就會開始圍著分數重排

坎貝爾定律比古德哈特定律更偏制度層，它提醒你：高壓使用指標時，連被測量的人與流程都會一起變形。

Core Tension

管理者需要可比較的數字，但數字一旦成為升遷、淘汰與資源分配的主軸，現場就會學會優化分數，而不是優化真正的公共目標。

01 先看指標承擔多大權力

越直接綁到懲獎、排名與生存，越容易把指標從量測工具推成行為控制器。

02 再找制度會怎麼自我調整

被測的人不是靜止樣本；他們會學、會應對，也會把流程改成更好看。

03 最後設計反失真機制

若不得不用高權重指標，就要同時補上抽查、質性判讀與副作用監測。

Setup

從一套看似合理的評量制度開始

坎貝爾定律最常出現在教育、醫療、客服與組織績效管理這類『不得不評分』的場景。

最小劇情

學校把標準化測驗成績當作校務主指標，結果老師把更多時間轉向刷題、壓縮非考科教學，成績上升卻不代表學習更完整。

真正難點

你很難完全不用指標，但只要把它推到過高權重，它就會開始塑造行為，甚至塑造被測量本身。

Key Point

坎貝爾定律不是反對衡量，而是提醒你：當量測與控制糾纏太深，量測會變成一種制度干預。

Lenses

四個看制度失真的角度

同樣是分數上升，有可能是品質真的變好，也可能只是現場把可量部分放大、把難量部分藏起來。

壓力來源

指標只是 dashboard，還是已經成為裁員、預算與聲望的直接依據？

代理落差

這個數字只是逼近目標，還是已經取代了原本更難但更重要的目標？

操弄空間

分類規則、樣本挑選、報表時間窗，是否都能被現場策略性利用？

制度副作用

一旦大家只追指標，哪些本來重要卻不易計分的工作會被擠掉？

場景 分數變好看起來像 真正該追的風險
教育測驗 平均成績上升 深度理解、寫作與非考科能力是否被犧牲
客服 KPI 結案更快 問題是否被過早關單、轉單或切小件數
工程團隊績效 bug 關閉數增加 高風險根因是否反而被延後處理

Psychology

為什麼組織很難抗拒單一分數

因為一個數字能把模糊世界壓成看似清楚的治理界面，讓管理者誤以為自己終於抓到控制桿。

數字帶來治理幻覺

當情況複雜時，單一指標讓人感覺可比較、可追責、可快速做決定。

現場會理性回應制度

一旦分數影響資源分配，人自然會把努力轉去最能抬高分數的地方。

副作用常先被當例外

因為報表主趨勢看起來在進步，制度傷害很容易被解讀成個別雜訊。

這個定律抓的是『高權重指標如何反過來改寫制度』，不是簡單地說所有量化都無效。

Applications

把指標節制帶回工程與管理

對產品與 firmware 團隊來說，坎貝爾定律常出現在品質指標、速度指標與供應鏈管理指標開始主導現場的時候。

品質儀表板

若 release gate 只剩單一通過率，團隊可能先優化測試配置，而不是先處理真正脆弱路徑。

供應商品質管理

若只看退貨率，供應商有動機調整申報與分類，而不是改善製程穩定度。

客服與 field issue

若只追結案時間，最難查的 intermittent 問題會被制度性地推到畫面外。

  • 高權重指標旁邊要放反指標，專門看它可能製造出的副作用與盲點。
  • 如果現場開始出現大量『為了報表而設計的流程』，通常不是人變壞，而是制度正在教這麼做。

Misreadings

三個常見誤讀

坎貝爾定律最容易被誤讀成反數據口號，但它真正要求的是治理節制。

所以不要量化 校正

不是。重點是別讓單一指標承擔超出它能力的治理權重。

只要多加幾個 KPI 就行 校正

如果它們都被同一套壓力結構綁住，多指標也可能一起被遊戲化。

現場作弊才是問題 校正

更根本的是制度設計把人推向只顧報表的局面。

Questions

這個分數是在描述現場，還是在重寫現場

下面兩個問題很適合放進任何指標治理審查。

如果把這個指標權重再拉高一倍，現場最先被擠掉的是什麼

那通常就是它沒有量到、卻正在被犧牲的價值。

分數一旦變好，我有沒有獨立方法知道真實品質也同步改善

若沒有，就代表你可能正在把治理交給一個會自我腐化的代理量。
