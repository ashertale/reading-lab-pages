紐科姆悖論

Newcomb's Paradox

紐科姆悖論把選擇者卡在兩種理性之間：如果預測者已經很準，現在的選擇到底是在改變結果，還是在提供證據？

Orientation

如果你的選擇透露了你是誰，選擇就不只是拿走哪個盒子

這個悖論讓人看見：當外部預測高度準確時，行動不只像原因，也像一份關於自己策略的訊號。

Core Tension

兩盒論者抓住當下因果：透明盒裡的錢已固定；一盒論者抓住預測關聯：會一盒的人通常已被放入更多錢。

01 分清時間點

預測已完成、選擇才發生；爭論在於現在的選擇是否仍該考慮自己作為可預測型的人。

02 分清理性口徑

因果決策看當下能改變什麼，證據決策看行動提供了什麼資訊。

03 看承諾是否可穩定

若你的決策規則本身會被預測，規則就可能比單次動作更重要。

Setup

盒子裡的錢已放好，但你仍要決定自己是哪種選擇者

最小場景是一個可靠預測者已根據你會怎麼選放好獎金。

最小劇情

透明盒有一千元，不透明盒可能有一百萬或零元。預測者若預測你只拿不透明盒，就已放入一百萬；若預測你兩盒都拿，就讓不透明盒空著。

真正難點

現在多拿透明盒看似不會影響已放好的錢，但這個選擇又像是在證明你原本就是會兩盒的人。

Key Point

紐科姆悖論不是賭運氣，而是在問理性應該只看當下因果，還是也看可預測的策略身份。

Lenses

用三種問題拆開選擇

同一個盒子問題，可以被理解成因果、證據或承諾規則的衝突。

因果問題

既然盒內金額已固定，拿透明盒是否只是在多拿一千元？

證據問題

如果選擇和預測高度相關，兩盒行為是否強烈指向不透明盒原本就是空的？

策略問題

你希望自己成為可被預測為一盒的人，還是當場逐利的兩盒人？

Psychology

人會在『現在能控制』和『我透露了什麼』之間搖擺

這題難在兩邊都像理性：一邊拒絕迷信過去，一邊拒絕忽略高準確預測。

反事後魔法

兩盒直覺保護我們不要相信現在動作能改變過去。

型態焦慮

一盒直覺提醒我們，自己採用的決策規則可能早已被對方利用。

小額誘惑

透明盒的一千元很具體，容易讓人忽略選擇背後的策略訊號。

把這題帶回設計時，重點是問系統會不會預測並獎懲某種策略，而不是只看單次收益。

Applications

預測系統會改變人如何選擇

推薦、信用評分和反作弊都會讓人的行動同時成為結果和證據。

風控模型

使用者行為不只是造成風險，也會被模型用來推斷他屬於哪一類人。

平台激勵

若平台預測並獎勵長期合作型行為，單次取巧可能影響未來待遇。

協定設計

當節點的策略會被其他節點觀察，承諾規則比當下最小收益更重要。

Misreadings

不是預測者會魔法

悖論成立不靠改變過去，而靠預測準確度和決策規則之間的關係。

把它當成命定論 校正

預測準不代表你沒有選擇，而是你的選擇規則可能很穩定。

只問哪盒錢比較多 校正

真正爭點是該用哪種決策理論評估。

忽略預測可靠度 校正

如果預測者不準，一盒論的吸引力會大幅下降。

Questions

問行動是在造成結果還是在透露型態

下面兩問適合用在預測模型和激勵制度 review。

對方是在回應我的行動，還是在預測我的規則？

如果是後者，單次收益計算可能不足。

我希望被系統看成哪一類參與者？

當未來待遇取決於可觀察行為，策略身份本身就是成本。
