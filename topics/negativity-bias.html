負面偏誤

Negativity Bias

壞消息比較容易抓住注意、留下記憶並改變評價；它有保護價值，也因此常取得過大的決策席位。

Orientation

壞消息不是比較真，只是比較容易被大腦標成必須立刻處理

負面偏誤提醒我們，威脅訊號有正當優先權；問題是它常從優先處理變成獨占解釋。

Core Tension

忽略壞消息會危險，過度聽壞消息也會失真。好的判斷要保留警覺，同時要求它回到比例、基準線與可重複證據。

01 先承認它為何刺眼

負面訊號通常意味著損失、危險或信任破口，大腦優先處理它有演化與實務理由。

02 再補回分母

一次事故、一則抱怨或一段壞回饋，需要放回總量、頻率與影響範圍裡看。

03 最後保存正向證據

順利、穩定與有效防線也要被記錄；否則團隊只會記得哪裡破了，不記得什麼正在保護系統。

Setup

一次故障可以讓整份 release 回顧失去比例

當負面事件有名字、有截圖、有情緒，它就比九件順利完成的事情更容易成為團隊記憶。

最小劇情

某次 release 有九個模組順利上線，一個模組發生延遲。回顧會議只圍著延遲打轉，最後整個流程被評為不可靠。

真正難點

故障值得分析，但它不應該自動代表全局；你需要知道它是例外、趨勢，還是被情緒放大的記憶。

Key Point

負面偏誤最需要的校正，不是安慰自己沒事，而是補上完整樣本與防線紀錄。

Lenses

把壞訊號拆成注意、記憶與決策三層

同一個負面事件可能先抓住注意，接著霸占記憶，最後讓決策過度防守。

注意捕捉

紅色告警、客訴與事故會先跳出來。第一步不是壓掉它，而是標記它取得注意的原因。

記憶留存

壞事比較有故事性，順利則像什麼都沒發生。沒有刻意紀錄，穩定工作會在記憶裡消失。

決策比例

壞訊號進入決策時，要帶著基準率、頻率、嚴重度與已存在防線，而不是只帶著情緒強度。

Psychology

人會先處理可能傷害自己的東西，這讓負面偏誤很像負責任

負面偏誤不是少數人的悲觀，而是一般注意系統對威脅與損失的優先處理。

自保需要快速反應

好消息錯過了通常還有機會，壞消息錯過了可能付出代價；因此大腦會給壞訊號較高優先權。

壞故事比較完整

事故有起點、轉折與責任線，順利運作卻常沒有劇情。記憶偏好故事，於是壞事留下來。

信任修復成本高

一次破壞可能需要多次穩定表現才能修復。這種不對稱讓負面訊號在關係與產品體驗裡特別有重量。

校正負面偏誤的關鍵，是讓壞訊號被量化、被定位，也讓有效的正常運作被看見。

Applications

把壞消息放大之前，先補上分母與防線

在事故、回饋與支援資料裡，負面訊號通常最早到場，但它不該單獨寫結論。

事故復盤

除了 root cause，也記錄哪些防線有作用、影響如何被限制、哪些監控提前給了訊號，避免只留下失敗敘事。

團隊回饋

回饋表若只收集問題，團隊會以為全局都在惡化。把有效協作也寫成具體行為，才有比例。

使用者支援

客訴很重要，但要和使用量、重複率、嚴重度與 silent majority 的行為資料一起看。

Misreadings

校正負面偏誤不是把壞消息塗成好消息

正向思考不能替代風險分析；但風險分析也不能只靠最刺眼的事件。

用樂觀抵消負面 校正

負面訊號需要處理，不是用好話蓋過。校正方式是補證據，不是補情緒。

把警覺當作完整判斷 校正

警覺是入口，不是結論。它要被轉成頻率、嚴重度與可行行動。

以為多記錄好事就會失去危機感 校正

記錄有效防線能讓危機感更準確，知道該修哪裡，也知道什麼不該亂改。

Questions

讓壞訊號進入比例尺

這件壞事的分母是什麼

它佔全部事件、使用量、客戶數或流程次數的多少？嚴重度和頻率是否一致？

哪些事情正在正常運作

如果要避免過度修正，我是否同時記錄了有效防線與順利完成的案例？

延伸路線

可接著讀 loss aversion，理解損失為何在選擇中取得比收益更重的權重。

Sources

來源與整理方式

本頁以 negativity bias 的注意與記憶偏重為核心，再用 loss aversion 補上決策裡的損失不對稱。

核心概念

支撐負面事件比正面事件更容易影響注意與判斷的主線。

相鄰脈絡
