雙迴路學習

Double-Loop Learning

雙迴路學習不只修正行動，還回頭檢查支配行動的假設、目標和規則；它問的是為什麼我們一開始會這樣判斷。

Read For

分辨 single-loop 的調參與 double-loop 的前提重估，避免只把錯誤修到下次再來。

Orientation

修正錯誤之前，先問錯誤為何被當成合理選擇

單迴路學習像調整溫控器：偏離目標就修正行動；雙迴路學習會問目標、指標和控制邏輯本身是否有問題。

Core Tension

組織很擅長修補表面偏差，卻常迴避讓偏差反覆出現的政策、價值排序和權力關係。

01 分清偏差層級

是執行出錯，還是目標、指標或假設本身需要改？

02 暴露支配變數

把默認的成功定義、風險容忍和權責邊界說出來。

03 處理防衛反應

若討論威脅身份或權力，組織會自然把問題推回技術細節。

Setup

事故後只加一個檢查項，可能只是讓同一套錯誤更慢地發生

最小場景是一場復盤停在修流程，沒有觸碰深層假設。

最小劇情

線上事故來自趕 release 時跳過風險討論。復盤結論先是新增 checklist；但深入追問後發現，團隊把準時發布視為唯一真正被獎勵的目標。

真正難點

若獎勵結構不變，checklist 會在壓力下再次被形式化或繞過。

行動偏差被修補
→
支配假設被追問
→
目標或規則被改寫
Key Point

雙迴路學習把復盤從怎麼避免這次錯，推到為何我們的系統會持續產生這類錯。

Lenses

從動作、規則到價值排序逐層追問

不是每個問題都需要雙迴路，但反覆出現的問題通常不能只靠調參。

行動層

哪個步驟、工具或判斷需要修正？

規則層

流程、指標和權限是否鼓勵了這個行為？

價值層

我們真正優先的是速度、穩定、成本、學習還是面子？

Psychology

人會把前提當成空氣，直到有人指出它也是選擇

雙迴路學習常不舒服，因為它把隱性承諾拉到桌面上。

防衛性推理

人會保護自己已投入的策略和身份。

局部修補較安全

改 checklist 比改 KPI 或權力分配容易。

成功案例成為枷鎖

過去奏效的假設會變成不被檢查的常識。

復盤若沒有改變任何指標、權限或停止條件，它很可能還停在單迴路。

Applications

把學習設計成可改前提的流程

incident review、產品策略和管理制度都需要允許前提被安全挑戰。

事故復盤

除了 action item，也檢查哪些目標和獎勵讓風險被接受。

產品策略

指標下降時不只改功能，也問原本的成功定義是否錯。

組織管理

用決策紀錄保存當時假設，後續才能重新評估。

Misreadings

雙迴路學習不是每件事都要哲學化

簡單錯誤可以直接修；反覆錯誤、重大風險和策略失準才需要改前提。

否定單迴路 校正

調參和修 bug 仍然必要。

把追問變成責備 校正

前提檢查要看制度，不是找人認錯。

空談文化 校正

真正的雙迴路要改規則、指標或權限。

Questions

問哪個前提讓這個行動看起來合理

下面兩問適合放進復盤模板和策略 review。

如果同樣壓力再來，什麼會讓人做出同樣選擇？

找規則和獎勵，而不只找錯誤步驟。

我們願意改哪個支配變數？

例如 KPI、權限、排程承諾或風險門檻。

延伸路線

可接著讀 organizational learning，理解組織如何保存、修正和防衛自己的知識。
