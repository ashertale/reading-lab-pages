Orientation

如果人沒有算出最優解，這一定代表他不理性嗎

碰到 有限理性，最容易出錯的不是沒聽過它，而是把 資訊有限、時間有限 與 計算有限 混成一句好聽卻無法執行的話。

Core Tension

真實世界裡的理性從來不是在無限時間、完整資訊與完美計算下發生，而是在限制裡做夠好的選擇。

01 先把 資訊有限 拉回起點

很多爭論表面在談結論，其實一開始就把 資訊有限 當成默認前提。

02 再問 時間有限 是誰在付

現場最常漏掉的不是立場，而是 時間有限 到底由誰承擔、何時結帳。

03 最後確認 計算有限 誰能檢查

真正把事情做完的，往往不是表態，而是把 計算有限 翻成可被檢查的動作。

Setup

先別問你支不支持，先問 資訊有限 和 時間有限 誰該先落筆

把討論縮到一次具體決策，資訊有限 是否先被交代、時間有限 是否被偷省、計算有限 是否有驗收方式，就會一眼看出來。

最小劇情

面對模糊需求、緊迫時間與有限注意力，多數決策其實是在有限資源下找可接受答案，而不是全局最優。

真正難點

如果人沒有算出最優解，這一定代表他不理性嗎

有人先把 資訊有限 當既成前提
→
真正要付的 時間有限 被留到後面
→
收尾時才發現 計算有限 根本沒有操作方式
Key Point

把 有限理性 用好，重點不在記住一句格言，而在於知道 資訊有限 要先說、時間有限 要有人扛、計算有限 要能驗收。

Lenses

先把 限制中的理性、最優神話退位 與 環境形塑策略 分開看

如果不把 限制中的理性、最優神話退位 與 環境形塑策略 拆開，你很容易以為大家在討論同一條原則，實際上是在搶不同的判準。

限制中的理性

順著 限制中的理性 讀，你會發現爭點不在口號，而在 資訊有限 是否真的被變成判準。

最優神話退位

把焦點放在 最優神話退位 時，你其實是在說：時間有限 應該先被寫清楚，否則後面的結論都站不穩。

環境形塑策略

環境形塑策略 這條線在乎的，不是誰比較有氣勢，而是 計算有限 一旦沒被交代，現場會怎麼開始失真。

Psychology

為什麼人一忙起來就最容易把 有限理性 用歪

這類原則常被用歪，不是因為內容太難，而是因為把它說成一句漂亮話，比真的交代 資訊有限、時間有限 和 計算有限 容易太多。

最優崇拜

碰到 最優崇拜 時，最危險的是大家以為原則已經被引用，實際上 資訊有限 還沒被具體說明。

事後容易嫌決策不夠完美

事後容易嫌決策不夠完美 會讓人覺得只要把話說得夠完整，就不必再補 時間有限 的證據或代價。

限制常在事前沒被寫出來

限制常在事前沒被寫出來 常不是純粹錯誤，而是一種討論捷徑：先把自己放在合理的一邊，再晚點處理 計算有限。

一條原則會變空話，通常不是因為字太抽象，而是因為 計算有限 從來沒有被翻成現場動作。

Applications

回到 產品決策、現場排故 與 流程設計

真正有差的，不是會議上有沒有人引用 有限理性，而是 產品決策 和 現場排故 裡的決策流程有沒有因此變得更可檢查。

產品決策

在 產品決策 這種場景，有限理性 幫你的不是贏辯論，而是把 資訊有限 先釘在桌面上，避免事後才補。

現場排故

現場排故 最需要 有限理性 的時候，通常就是大家都急著往前走，卻還沒把 時間有限 寫成共識。

流程設計

只要到了 流程設計，你就能看出 計算有限 究竟是被真的承擔，還是只是被口頭提過。

Misreadings

三個常見誤讀

這類頁面最常被讀歪的地方，不是概念太抽象，而是大家太快把 資訊有限 當成價值宣示，忘了它還得連著 時間有限 和 計算有限 一起成立。

有限理性 只是好聽口號 校正

真正難的是把 有限理性 翻成一組可檢查的判準，而不是把它當成結論本身。

只要背了 有限理性 就能直接套用 校正

原則若不落回 資訊有限、時間有限 和代價分配，通常會在現場變成誤用。

有限理性 只適合高層決策 校正

其實從 code review 到 現實中的理性 的日常判斷，都會碰到同樣的壓力。

Questions

把 資訊有限、時間有限 與 計算有限 變成會議上的追問

下面兩個問題不是要你再複述一次原則，而是逼 產品決策、現場排故 這類場景裡的人把空白前提補完。

如果 產品決策 現在就要拍板

誰要先把 資訊有限 說清楚？時間有限 由誰承擔？若 計算有限 無法驗收，你還願意讓決定往前走嗎？

如果團隊說『有限理性 不是大家都同意嗎』

怎樣才算真的用了 有限理性，而不是只借它替既有結論加上道德光澤？
