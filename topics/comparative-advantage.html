比較利益

Comparative Advantage

比較利益說明分工不只看誰做得更好，而是看誰放棄的其他選項成本更低。

Read For

理解機會成本、相對效率和分工交換如何讓弱者也可能有專長位置。

Orientation

高手不該做所有事，因為高手的時間也有替代成本

即使一個人做兩件事都比別人強，他仍可能應該專注在相對優勢最大的工作，把另一件交給機會成本較低的人。

Core Tension

絕對能力直覺會讓最強者被塞滿所有重要任務；比較利益則要求看每個選擇放棄了什麼。

01 計算機會成本

做 A 時犧牲多少 B，比單看做 A 的速度更重要。

02 找相對差距

誰在某工作上的劣勢較小，可能就該負責那項工作。

03 交換創造總量

分工後透過交換或協作，雙方都能比各做各的更好。

Setup

資深工程師 code review 很快，但讓他一直修小 bug 可能很貴

最小場景是一個人樣樣都強，但時間被低相對價值任務占滿。

最小劇情

資深工程師修 bug 和設計架構都比新人快。可是他花一天修零碎 bug，就少一天處理關鍵架構風險；新人修 bug 較慢，卻能用這些任務累積上下文。

真正難點

若只看絕對速度，所有事都該交給資深者；若看機會成本，分工會不同。

Key Point

比較利益把分工問題從誰比較強，改成誰做這件事犧牲最少。

Lenses

用放棄的東西重新排序工作

真正的成本藏在沒做的第二好選項裡。

相對效率

各人在每項工作上相對於其他工作的效率如何？

替代成本

把此人放在這裡，放棄了哪個高價值任務？

交換機制

分工後成果如何交接，避免協作成本吃掉收益？

Psychology

人會把快的人當成所有瓶頸的解法

能力可見，機會成本不明顯；所以組織很容易把最能幹的人塞進所有洞。

明星過載

強者被分配太多不同類型任務。

新人低估

新人絕對效率低，但在某些任務上的機會成本也低。

短期救火

眼前速度掩蓋長期能力培養和系統風險。

排任務時加一欄此人不做什麼，比只看誰最快更接近總體最佳。

Applications

用機會成本做分工，而非用職級直覺

團隊排程、外包決策和個人時間管理都能用比較利益檢查。

團隊分工

把資深者留給高槓桿設計和 review，新人承接可學習且可回滾的任務。

外包決策

外部廠商不必比內部更強，只要內部時間另有更高價值用途。

個人管理

你擅長的事也可能該委派，若它擠掉更關鍵的工作。

Misreadings

比較利益不是只替自由貿易背書

它是一個機會成本模型，實務還要處理交易成本、學習、安全和公平。

忽略交易成本 校正

分工需要交接和協調，收益不一定自動實現。

把現況能力固定化 校正

分工也會影響誰得到學習機會。

只看總量不看分配 校正

總產出提高仍可能需要補償和制度安排。

Questions

問這個人做此事放棄了什麼

下面兩問適合用在排程、委派和外包判斷。

誰做這件事的機會成本最低？

不要只問誰做得最快。

分工後的交接成本會吃掉多少收益？

把協作摩擦也算進去。

延伸路線

可接著讀 opportunity cost，理解每個選擇背後被放棄的最佳替代。
