奈奎斯特-香農取樣定理

Nyquist-Shannon Sampling Theorem

離散取樣能不能保留連續訊號，不取決於資料點看起來多不多，而取決於你是否先管住頻寬與混疊。

Orientation

取樣不是把時間切成格子那麼簡單

奈奎斯特-香農取樣定理說明：若訊號已被限制在某個頻寬內，足夠高的取樣率可以保留重建所需資訊。

Core Tension

工程現場的麻煩在於訊號通常不會自動 band-limited；若前端沒有先處理高頻成分，再密的後端分析也可能是在分析混疊後的假象。

01 先界定頻寬

你需要知道想保留的最高頻率，以及不想保留的高頻是否會進入 ADC 或資料流。

02 再配置取樣率

理想定理要求取樣率相對於頻寬有足夠餘裕；現場還要加上 filter roll-off 與時鐘誤差。

03 最後檢查混疊路徑

一旦高頻折回低頻，資料裡看到的慢變化可能根本不是原始訊號的慢變化。

Setup

看似平滑的曲線可能是錯的

把情境縮到一個電流感測：你每 10 ms 記一次 log，圖上看起來波動很小，於是判定 power rail 穩定。

最小場景

實際上 rail 上有高頻 switching noise 與短脈衝，但取樣率太低且前端濾波不足，資料點把高頻折成一條似乎有週期的低頻擺動。

真正難點

資料不是少一點精細度而已；混疊會創造錯誤的形狀，讓你追一個原本不存在的低頻原因。

連續訊號含高頻
→
低率取樣且濾波不足
→
高頻折回成假低頻
Key Point

真正危險的是混疊後的資料仍然很像合理訊號，除非你回頭查取樣與前端濾波。

Lenses

從四個位置查取樣是否可信

取樣定理不只是一條 2B 規則；它同時要求你說清楚訊號假設、濾波位置、時鐘品質與重建目的。

band-limited 假設

定理的乾淨版本建立在頻寬有限的訊號上；未濾波 sensor 不能自動套用。

anti-aliasing filter

ADC 前的 analog filter 比後處理更關鍵，因為混疊一旦發生就難以用數位方法分離。

取樣時鐘

jitter、drift 與非均勻取樣會改變可重建性，特別是在高頻或相位敏感量測裡。

目的差異

監控趨勢、估 RMS、重建波形與抓 transient，需要的取樣與濾波策略不一樣。

決策 可接受假設 需要警覺
低頻溫度 log 變化慢且 sensor 本身有慣性 突發熱事件可能被平均掉。
電流 transient 前端頻寬與取樣率一起設計 短脈衝可能被漏掉或折成假週期。
振動分析 取樣率高於分析頻帶且有 anti-aliasing 機械諧波會讓混疊更像真特徵。

Psychology

圖很順，人就容易相信資料可靠

取樣問題常被視覺化掩蓋：漂亮平滑曲線會降低懷疑，鋸齒或亂點反而讓人想查。

平滑造成安全感

低取樣率常讓圖變得更乾淨，卻不代表訊號更穩，只代表你看不到變化。

資料點數被當成品質

長時間 log 很多點不等於頻域資訊足夠；取樣間隔仍可能錯過關鍵瞬間。

後處理的錯覺

濾波、平均與插值能讓曲線好看，但無法還原取樣前已經混在一起的頻率。

對 firmware debugging 來說，先查量測鏈，比直接解讀曲線更省時間。

Applications

用在 ADC、sensor 與 log 設計

這個定理在 firmware 裡不只是 DSP 教科書；它會影響 sampling ISR、DMA buffer、telemetry rate 與 field log 的可信度。

ADC 前端

選 ADC sample rate 時，同時規格化 analog filter、input bandwidth 與 layout noise，不要只看 MCU 最大取樣率。

sensor fusion

不同 sensor 的取樣率與延遲若沒對齊，後端融合可能把 alias 與 phase lag 當成真動作。

field telemetry

為了省 bandwidth 而降採樣前，先決定要保留的是平均趨勢、尖峰、週期性還是故障前兆。

  • 每一條低頻結論旁邊都應該能回答：高頻成分是否在取樣前被控制住？
  • 若資料來源已經混疊，不要把後處理圖形解釋成原始物理現象。

Misreadings

取樣定理最常被背成口號

它不是只要超過兩倍就萬事安全；那只是理想條件下的核心邊界。

取樣率大於兩倍就夠 校正

錯。現實 filter 有轉換帶，訊號也可能有未預期高頻，通常需要設計餘裕。

混疊只是解析度下降 校正

不是。混疊會把頻率折成別的樣子，讓錯誤結論看起來很有規律。

可以事後補救 校正

若高頻已在 ADC 前折回，數位後處理很難知道哪個低頻成分是真的。

Questions

讀任何 log 前先問取樣鏈

如果圖上看到低頻擺動

你能證明它不是高頻 noise 或 transient 被低取樣率折回來的結果嗎？

如果要降 telemetry rate

哪個故障特徵會先消失？你需要保留峰值、能量、頻率，還是事件順序？
