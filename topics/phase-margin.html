相位裕度

Phase Margin

相位裕度量的是回饋系統離震盪還有多少餘地；反應越快不一定越好，穩定餘度太小會帶來 ringing 或失穩。

Read For

適合看 DC/DC compensation、op-amp loop、馬達控制、thermal loop 與任何閉迴路調校。

Orientation

回饋系統離自我打架還有多遠

Phase margin 通常看 loop gain 為 0 dB 的頻率，當下相位距離 -180 度還剩多少。

Core Tension

提高 bandwidth 能讓反應變快，但也可能讓延遲與相位落後把負回授推近正回授，造成 ringing 或震盪。

01 先看 crossover

unity gain 附近的相位最能揭露穩定餘度。

02 再看時域現象

overshoot、ringing 與 settling time 通常能對應到頻域裕度。

03 最後調補償

compensation network 或控制參數要同時處理速度、相位與噪聲。

Setup

負載一跳，電源輸出開始響

板上 DC/DC 在負載瞬變時有過衝和 ringing；單看穩態電壓合格，卻無法說明動態穩定性。

最小場景

負載 step 後輸出震盪幾個週期才 settle，EMI 與 downstream reset 風險上升。

真正難點

迴路不是只有 DC accuracy；補償、輸出電容 ESR、layout 與負載都會改變相位。

loop gain 下降到 0 dB
→
相位已接近 -180 度
→
小擾動被放大成 ringing
Key Point

相位裕度把看似抽象的頻域圖，連到示波器上的過衝和震盪。

Lenses

用 Bode plot 看穩定餘地

相位裕度和增益裕度一起告訴你，回饋系統能承受多少模型誤差與條件變動。

Gain crossover

loop gain 等於 1 的頻率通常是計算 phase margin 的關鍵點。

Phase lag

感測、計算、濾波、plant delay 都會增加相位落後。

Compensation

零點與極點安排可改善相位，但也會影響 bandwidth 與噪聲。

Load variation

負載、溫度與元件容差會讓裕度在 field 中改變。

現象可能線索檢查
Ringing 明顯 phase margin 偏低 loop response 與補償網路
反應過慢 bandwidth 太低或裕度過保守 crossover frequency
不同板子差異大 元件容差或 layout 影響 pole/zero 量測多批次與溫度

Psychology

快反應很容易被誤認成好控制

Demo 時快速追上 setpoint 很吸引人，但沒有穩定餘度的快，會在擾動和量產差異下還債。

速度偏誤

大家先看到 rise time，晚一點才看到 ringing、噪聲與 overshoot。

穩態遮蔽

穩態誤差很小不代表動態過程安全。

模型過度信任

模擬 plant 太理想，field 的 delay 和 parasitic 會吃掉 margin。

控制 review 應把 bandwidth 和 margin 放在同一句話裡，避免只談快慢。

Applications

電源、放大器與韌體控制都需要裕度

凡是負回授系統，只要有延遲與頻率響應，就會遇到穩定餘度問題。

Power supply

DC/DC compensation 要在 transient response、phase margin 與 output capacitance 間取捨。

Op-amp loop

容性負載或不當 feedback network 會降低 phase margin，造成 ringing。

Firmware loop

取樣週期、濾波與計算延遲會把 digital control 的相位往後推。

  • 不要只驗證 nominal load；要在 min/max load、溫度與元件容差下看 transient。
  • 把 ringing frequency 和 loop crossover 對照，常能快速定位是哪段迴路在發聲。

Misreadings

裕度不是越大越漂亮

相位裕度太低危險，太高也可能代表迴路太慢或過度保守。

60 度永遠最佳 校正

常見目標有參考價值，但不同 plant、負載與產品需求會改變最佳點。

只看 phase margin 校正

gain margin、load transient、noise 與 saturation 也會影響穩定感。

韌體控制不用看頻域 校正

取樣與濾波同樣會產生 phase lag；digital loop 也需要裕度概念。
