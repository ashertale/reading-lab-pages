Orientation

如果問題反覆出現，為什麼只把執行再做得更好，常常還是不夠

讀 雙迴路學習，先別急著表態自己站哪一邊；更重要的是看在 反思與改進 裡，做法修正、前提反思 和 目標檢查 誰應該先被說清楚。

Core Tension

單迴路學習會修正做法，雙迴路學習則會進一步追問：我們原本拿來決定做法的目標、規則與假設是不是也該改。

01 先把 做法修正 拉回起點

很多爭論表面在談結論，其實一開始就把 做法修正 當成默認前提。

02 再問 前提反思 是誰在付

現場最常漏掉的不是立場，而是 前提反思 到底由誰承擔、何時結帳。

03 最後確認 目標檢查 誰能檢查

真正把事情做完的，往往不是表態，而是把 目標檢查 翻成可被檢查的動作。

Setup

問題不是立場，而是 目標檢查 有沒有被當成默認

把討論縮到一次具體決策，做法修正 是否先被交代、前提反思 是否被偷省、目標檢查 是否有驗收方式，就會一眼看出來。

最小劇情

專案一出錯就補 SOP、補提醒、補更多檢查；但如果根因在目標設定或角色邊界，補動作只是在原地打轉。

真正難點

如果問題反覆出現，為什麼只把執行再做得更好，常常還是不夠

有人先把 做法修正 當既成前提
→
真正要付的 前提反思 被留到後面
→
收尾時才發現 目標檢查 根本沒有操作方式
Key Point

把 雙迴路學習 用好，重點不在記住一句格言，而在於知道 做法修正 要先說、前提反思 要有人扛、目標檢查 要能驗收。

Lenses

先把 修行為與修前提分開、反覆事故不是巧合 與 規則也需要被檢查 分開看

如果不把 修行為與修前提分開、反覆事故不是巧合 與 規則也需要被檢查 拆開，你很容易以為大家在討論同一條原則，實際上是在搶不同的判準。

修行為與修前提分開

順著 修行為與修前提分開 讀，你會發現爭點不在口號，而在 做法修正 是否真的被變成判準。

反覆事故不是巧合

把焦點放在 反覆事故不是巧合 時，你其實是在說：前提反思 應該先被寫清楚，否則後面的結論都站不穩。

規則也需要被檢查

規則也需要被檢查 這條線在乎的，不是誰比較有氣勢，而是 目標檢查 一旦沒被交代，現場會怎麼開始失真。

Psychology

為什麼人一忙起來就最容易把 雙迴路學習 用歪

這類原則常被用歪，不是因為內容太難，而是因為把它說成一句漂亮話，比真的交代 做法修正、前提反思 和 目標檢查 容易太多。

先補動作最快

碰到 先補動作最快 時，最危險的是大家以為原則已經被引用，實際上 做法修正 還沒被具體說明。

前提改寫牽涉權力

前提改寫牽涉權力 會讓人覺得只要把話說得夠完整，就不必再補 前提反思 的證據或代價。

大家都熟悉現有目標

大家都熟悉現有目標 常不是純粹錯誤，而是一種討論捷徑：先把自己放在合理的一邊，再晚點處理 目標檢查。

一條原則會變空話，通常不是因為字太抽象，而是因為 目標檢查 從來沒有被翻成現場動作。

Applications

回到 事故復盤、流程改善 與 組織學習文化

真正有差的，不是會議上有沒有人引用 雙迴路學習，而是 事故復盤 和 流程改善 裡的決策流程有沒有因此變得更可檢查。

事故復盤

在 事故復盤 這種場景，雙迴路學習 幫你的不是贏辯論，而是把 做法修正 先釘在桌面上，避免事後才補。

流程改善

流程改善 最需要 雙迴路學習 的時候，通常就是大家都急著往前走，卻還沒把 前提反思 寫成共識。

組織學習文化

只要到了 組織學習文化，你就能看出 目標檢查 究竟是被真的承擔，還是只是被口頭提過。

Misreadings

三個常見誤讀

這類頁面最常被讀歪的地方，不是概念太抽象，而是大家太快把 做法修正 當成價值宣示，忘了它還得連著 前提反思 和 目標檢查 一起成立。

雙迴路學習 只是好聽口號 校正

真正難的是把 雙迴路學習 翻成一組可檢查的判準，而不是把它當成結論本身。

只要背了 雙迴路學習 就能直接套用 校正

原則若不落回 做法修正、前提反思 和代價分配，通常會在現場變成誤用。

雙迴路學習 只適合高層決策 校正

其實從 code review 到 反思與改進 的日常判斷，都會碰到同樣的壓力。

Questions

把 做法修正、前提反思 與 目標檢查 變成會議上的追問

下面兩個問題不是要你再複述一次原則，而是逼 事故復盤、流程改善 這類場景裡的人把空白前提補完。

如果 事故復盤 現在就要拍板

誰要先把 做法修正 說清楚？前提反思 由誰承擔？若 目標檢查 無法驗收，你還願意讓決定往前走嗎？

如果團隊說『雙迴路學習 不是大家都同意嗎』

怎樣才算真的用了 雙迴路學習，而不是只借它替既有結論加上道德光澤？
