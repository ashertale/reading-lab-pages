邊際效用遞減

Law of Diminishing Marginal Utility

同樣一單位資源，落在極度缺乏的人身上和落在已經很充足的人身上，通常不會帶來同樣多的主觀價值；越往後加，新增效用常越小。

Read For

把『多給一點』從直覺拉回邊際思考，分清楚資源加在哪裡真正比較有用。

主題類型
分配原則 / 價值遞減
核心問題
Each additional unit tends to add less value than the previous one

Orientation

不是所有多一點，都一樣值得

很多制度看起來是在追效率，最後卻會被這個張力拉歪：只要價值取決於你原本已經有多少，新增同樣一單位資源帶來的好處就不會線性增加。

Core Tension

只要價值取決於你原本已經有多少，新增同樣一單位資源帶來的好處就不會線性增加。

01 先看接收者原本有多少

同一單位資源加在不同基底上，意義通常完全不同。

02 再算新增一單位真的改變了什麼

真正要比較的是邊際影響，而不是總量視覺上的對稱。

03 最後把飽和區和匱乏區分開

資源最該優先補到的，往往是仍能產生巨大新增價值的地方。

Setup

先把制度壓回一個小場景

邊際效用遞減最適合從分配直覺開始看，因為同樣一份新增資源對不同起點的人，本來就不會等值。

最小劇情

同樣多的 debug 時間，給一個已經被打斷很多次的關鍵問題，和給一個本來就進度充裕的低風險項目，帶來的價值往往差很多。

真正難點

難點在於人容易把絕對量當成全部，忽略邊際值其實和基準存量高度相關。

Key Point

邊際效用遞減最實用的地方，是提醒你分配不該只看『有沒有增加』，還要看『這一單位加上去還剩多少新增價值』。

Lenses

幾個會讓結果分岔的判準

同一份新增資源值不值錢，先看它落在匱乏區、改善區，還是已經接近飽和的地方。

基準存量

已有很多的人再多一點，感受到的改善通常比缺乏者少得多。

主觀價值

效用不是純物理量，而和需求迫切度、替代性與心理狀態有關。

配置效率

若忽略邊際遞減，制度很容易把資源堆到已接近飽和的地方。

制度線索 表面上像什麼 真正風險
只看總量分配 表面整齊 可能把資源給到新增價值很低的地方
看邊際價值分配 比較需要判斷 卻更能讓有限資源真的產生效果
忽略飽和區 短期比較少爭論 長期容易養出低效率與隱性浪費

Psychology

這條制度動力抓住了哪種人性

人常被平均分配的整齊感說服，卻忘了真價值通常跟對方原本有多少密切相關。

整齊偏好

平均分配看起來最公平，但不一定最有邊際效果。

存量失明

人常看到『又多了一份』，卻忘記那份對不同人可能完全不同價值。

總量迷戀

只要總預算或總福利在上升，人就容易忽略新增部分其實越來越不值得。

這一段要看的，是 整齊偏好、存量失明 這類反應如何把 邊際效用遞減 指向的制度壓力在現場放大。

Applications

把它帶回管理、政策與組織設計

回到時間排程、產品取捨與福利配置，這個概念是在幫你把資源送到邊際最值錢的地方。

資源排程

有限的專家時間先投到最卡、最痛、最缺的節點，通常比平均灑更有邊際價值。

產品功能

多一個高級設定選單對重度用戶或許有感，但對已經困惑的一般用戶可能幾乎沒新增價值。

福利設計

同樣的補助對處於匱乏狀態的人通常更能明顯改變生活條件。

  • 資源排程: 把關鍵時間投在邊際價值最高的瓶頸。
  • 產品設計: 別讓新增功能落到已經接近飽和的價值區。

Misreadings

幾種常見但危險的治理讀法

若一開始就把 邊際效用遞減 聽成「所以越多越沒用」，後面連「不是。問題是新增效用會遞減，不是絕對沒有用。」這層校正都很難真正進場。

所以越多越沒用 校正

不是。問題是新增效用會遞減，不是絕對沒有用。

邊際遞減只和金錢有關 校正

時間、注意力、工具與功能複雜度都可能有同樣結構。

只要效用遞減，就不該追求更多 校正

有些情境仍值得追加，只是你得知道追加的理由和邊際值在哪。

Questions

把它變成下一次治理審查的追問

這一單位資源如果給到別處，邊際效果會不會更高

沒有問這題，很多分配其實只是按習慣。

我們現在投入的地方，是還在解決匱乏，還是已經進入接近飽和的區間

分清這點，配置策略會差很多。
