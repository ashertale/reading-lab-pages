Orientation

情緒很擅長回應一個人，卻不擅長隨著一萬個人一起放大

範圍忽略提醒我們，直覺感受常在規模變大時飽和；如果不補數量校準，優先順序會被故事牽走。

Core Tension

故事讓問題變得可感，數量讓資源配置變得公平。少了故事會冷，少了數量會偏。

01 先保留具體畫面

案例能讓人理解受害、痛點或風險，不必把故事視為偏誤來源。

02 再補規模敏感度

問影響從十人變成千人時，資源、注意與行動是否也有所調整。

03 最後用優先順序落地

把規模、嚴重度、可修復性與成本放在同一張表上，避免只追最有畫面的問題。

Setup

一個有臉孔的案例，常比一個更大的統計更容易拿到資源

最小場景不是人不在乎數量，而是數量太大時會變成抽象背景，無法同等推動情緒。

最小劇情

支援團隊看到一位客戶寫了很痛的長信，立刻想優先修他的痛點；同時另一個沉默問題每天影響上千人，卻只在 dashboard 上是一個小百分比。

真正難點

那封信是真實痛苦，不該被忽略；但它不能單獨決定優先順序，除非也和影響規模一起被評估。

Key Point

範圍忽略的校正不是拿掉故事，而是把故事接到分母上。

Lenses

用故事、數量與排序三層一起讀問題

故事負責讓人進入問題，數量負責讓資源比例不失真，排序則把兩者轉成行動。

故事層

問這個案例揭露了什麼機制：痛點在哪裡、誰受影響、為什麼現有系統接不住。

數量層

補上受影響人數、頻率、總時長、損失大小與不確定範圍，讓感受有比例尺。

排序層

把規模和嚴重度轉成明確取捨：先修、延後、監測、或只用溝通處理。

Psychology

大數字會讓感受變鈍，不是因為人冷漠，而是因為想像力先到極限

我們能想像一個人的痛苦，也能稍微想像十個人的痛苦；再往上，情緒就很難線性延展。

情緒有飽和點

一百與一千在數學上差十倍，在感受上卻可能都只是「很多」。這讓資源配置容易低估大規模問題。

案例比較有抓手

具體人物、截圖或故事提供入口；沒有入口的統計雖然更大，卻更難讓人採取行動。

抽象會降低責任感

當影響只以百分比或大數字出現，責任感容易分散，因為沒有人能直接想像完整後果。

校正範圍忽略，需要把統計重新具體化，也把故事重新量化。

Applications

優先順序要同時看痛感與規模

在產品、公益與安全決策裡，最有畫面的問題不一定最大，最大的問題也不一定最容易被感到。

產品 triage

把客訴故事和影響人數、頻率、revenue、使用路徑一起看，避免單一高情緒案例支配整個 sprint。

風險溝通

只報數字會冷，只講個案會偏。好的風險溝通要讓人看見一個案例，同時知道它代表多大範圍。

公益與安全

捐助或安全投資要把可見痛苦與總體影響合併；否則資源會流向比較會說故事的問題。

Misreadings

校正範圍忽略不是把人變成冷冰冰的表格

故事與數量各自補對方的缺點；缺任何一邊，判斷都容易歪。

以為故事都會誤導 校正

故事能揭露機制和痛點。問題不是故事存在，而是故事沒有接到規模資料。

只看最大數字 校正

規模很重要，但嚴重度、可修復性與成本也要一起看；最大不必然最先。

把情緒反應當成資源配置 校正

情緒可以開啟注意力，不能單獨決定預算、排程或優先級。

Questions

檢查自己的規模敏感度

規模變大時我有跟著調整嗎

如果影響從一百人變一萬人，我的預算、排程或注意力是否有合理變動？

故事和分母有接上嗎

這個具體案例代表多少人、多少次、多久、多少損失？它是典型、極端，還是早期警訊？

延伸路線

可接著讀 psychic numbing，理解大規模痛苦為何反而讓情感反應變鈍。
