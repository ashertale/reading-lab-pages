Liar Paradox

說謊者悖論用一句『這句話是假的』逼問真值系統：當語句談論自己的真偽時，真假標籤會開始互相咬住。

Orientation

一句話談到自己的真偽時，規則不再只是套用規則

如果它是真的，它說自己是假的；如果它是假的，它說自己是假的又似乎是真的。問題不是語句調皮，而是真值規則遇到自指。

Core Tension

語言需要能談論語言，但若讓語句在同一層直接評價自己的真值，就可能產生無法穩定標記的迴圈。

01 找出自指入口

語句是否直接或間接談到自己的真值、可證性或狀態？

02 分開語言層級

談論物件的語言和談論語言本身的語言，可能需要不同層級。

03 限制通用真值謂詞

不是每個系統都能安全地說『所有真句子』而不產生迴圈。

Setup

把『這句話是假的』標成真或假，都會立刻反咬

最小場景是一句話的內容剛好指向自己的真值。

最小劇情

你要在資料表中替語句 S 標記 true 或 false，而 S 的內容是『S 是 false』。標 true 會讓內容說它 false；標 false 又讓內容看起來成立。

真正難點

一般語句可由外部事實驗證，自指語句把驗證目標拉回標記行為本身。

語句指向自身
→
真值標記被內容反轉
→
系統無法穩定收斂
Key Point

說謊者悖論揭露的是語意規則的邊界，而不是單一怪句子的答案。

Lenses

用語意層級看自指是否越界

處理自指時，不同方案會選擇分層、放棄二值、限制語句，或接受某些不完整性。

層級方案

把談論物件的語言和談論真值的後設語言分開。

多值方案

允許某些語句既不真也不假，或有第三種狀態。

限制方案

禁止語言形成能直接套用於自身的真值謂詞。

Psychology

人會想替每句話都貼真假，悖論逼你承認分類器也有輸入限制

真假二分很有力，但說謊者悖論提醒我們，判準本身成為被判斷對象時，直覺會失效。

二值直覺很強

日常溝通讓人習慣每句話都該真或假。

自指被低估

看似短句，實際上把分類規則、語句內容和系統邊界纏在一起。

修補會移動成本

分層、多值或限制各自保住一部分直覺，也犧牲一部分表達能力。

遇到自我描述系統時，先問哪一層在談論哪一層，不要急著貼標籤。

Applications

自我引用會出現在規則、審核和分類系統裡

雖然原型是邏輯語句，工程系統也常讓規則處理規則本身。

規則引擎

若規則能修改或評估自己的啟用條件，需要分層和防迴圈機制。

審核流程

自動審核器若以自身輸出作為訓練真值，可能形成自我確認。

型別與 schema

描述資料的 schema 若也被同一 schema 描述，要清楚定義 meta-level。

Misreadings

不是所有自我提及都會爆炸

自指本身可用；危險在於自指穿過真值、證明或分類規則時沒有分層。

把它當文字遊戲 校正

悖論指出形式系統的限制，不只是語言玩笑。

以為答案只是假的 校正

標成假會讓語句內容成立，問題仍在。

禁止所有自指 校正

很多系統需要 meta data 和 introspection，重點是設計層級。

Questions

問規則是否正在評估自己

這個判準是否能套用到自身？

如果能，是否會形成反轉、循環或無限遞迴？

哪一層語言在描述哪一層對象？

把 object level 和 meta level 分清，再決定是否允許交叉。

延伸路線

可接著讀 self-reference，理解自我引用如何在邏輯、語言和系統設計中產生力量與限制。
