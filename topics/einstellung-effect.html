Einstellung Effect

曾經成功的解法會變成思考捷徑；它幫人快速開始，也可能擋住更簡單、更適合當下的新路。

Orientation

舊解法最危險的時候，是它真的曾經救過你

定勢效應不是反經驗，而是提醒你：經驗會先提供一個熟悉入口，接著可能把問題限制在它會回答的範圍。

Core Tension

成熟團隊需要累積模式，但模式若不接受重新定義問題，就會把新情境硬塞回舊工具。

01 先描述問題，不先描述解法

把現象、限制與目標分開寫，避免第一句就把問題翻成某個熟悉方案。

02 保留一輪陌生解

在套用老方法前，刻意列出至少一個不沿用既有套路的解釋或實驗。

03 用結果檢查慣性

若舊解法只能局部改善，卻讓問題變得更複雜，就該回頭重述題目。

Setup

一個熟悉 bug 會讓人先拿起熟悉工具

最小場景是工程師看到症狀像舊問題，於是還沒驗證假設，就先沿著舊路查下去。

最小劇情

系統延遲上升，團隊立刻檢查 cache 和 database index，因為上次也是這裡出事。兩天後才發現真正原因是新 retry policy 造成排隊放大。

真正難點

舊經驗不是錯，它只是太快取得主導權，讓新的問題邊界沒有機會被畫出來。

症狀喚起舊模式
→
檢查路徑提前收窄
→
更簡單解法被延後看見
Key Point

定勢效應的校正，是在使用經驗前先保留一小段重新命名問題的空間。

Lenses

把經驗、假設與題目重新拆開

舊解法要能繼續使用，但它必須先通過當前問題的條件檢查。

症狀相似

看起來像舊問題，不等於因果也相同；先列出相同與不同條件。

解法成本

熟悉解法啟動快，但可能帶來額外複雜度、技術債或錯誤診斷。

替代實驗

設計一個能推翻舊假設的小實驗，比繼續加深舊路徑更有價值。

Psychology

成功經驗會留下可重用捷徑，也會留下看不見的邊界

人喜歡先用知道有效的方法，因為它降低不確定；但問題改變時，這份確定感就會變成遮蔽。

熟悉感降低焦慮

遇到新問題時，能立刻套用舊模式會讓人感到掌控。

身份綁住方法

某種解法若代表專業能力，放下它會像放下自己的優勢。

局部進展掩蓋方向錯誤

舊方法常能帶來一點改善，讓團隊以為方向對，卻拖慢真正突破。

定勢效應需要的不是丟掉經驗，而是讓經驗在問題被重新描述後再進場。

Applications

在 debugging 與設計評審中安排反慣性步驟

只要團隊有強項，就會有被強項牽引的風險。

Debugging

先寫三個可能根因，再選工具；不要讓第一個工具決定你願意看見的根因。

架構設計

若每個問題最後都落到同一套架構，檢查是不是組織技能在替需求做選擇。

流程改善

既有 SOP 若每次只被微調，可能是因為沒有人重新問流程目標是否已變。

Misreadings

定勢效應不是叫你否定專業直覺

專業直覺有價值，問題是它需要被當成假設，而不是答案。

把老方法都當包袱 校正

老方法可能仍是最佳解；要檢查的是它是否未經比較就先贏。

以為創新就是換工具 校正

新的工具若仍服務舊問題定義，也只是換外皮。

只在個人身上找慣性 校正

團隊流程、職責與工具鏈也會讓某些答案總是先出現。

Questions

讓舊答案晚一分鐘進場

如果不能用最熟的方法，我會怎麼描述問題？

這個禁用條件能迫使我看到哪些被舊工具遮住的可能？

哪個小實驗能推翻我的第一個假設？

我是否能快速證明熟悉解法其實不適用，而不是只找支持它的線索？

延伸路線

可接著讀 functional fixedness，理解工具功能如何限制解題想像。
