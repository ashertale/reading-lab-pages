團體迷思

Groupthink

群體並不是人越多就自然更聰明；當維持和諧、速度或一致信心被看得太重時，群體反而會更難看見明顯漏洞。

Read For

看懂為什麼有些看似高共識、高效率的會議，其實正是壞決策最容易長出來的地方。

主題類型
群體偏差 / 決策失真
核心問題
Groups can trade critical evaluation for harmony and confidence

異議空間、領導壓力、同質性、自我審查
讀完產出
能在會議、評審與跨部門合作中更早識別『和諧吃掉質疑』的信號。

Orientation

很多群體失誤不是因為沒人有答案，而是因為沒人願意把答案講破

這種偏差之所以頑固，是因為 只要維持一致的壓力大過檢查想法的壓力，群體就會開始把保留吞回去，把信心推高，把漏洞留到事後才看見。

Core Tension

只要維持一致的壓力大過檢查想法的壓力，群體就會開始把保留吞回去，把信心推高，把漏洞留到事後才看見。

01 先替異議留制度空間

沒有明確的反方角色與低風險反對入口，異議很容易先在腦中就被消音。

02 再區分速度和深度

會議很快結束不代表決策品質高，可能只是質疑沒有成功進場。

03 最後讓信心接受獨立檢查

越是全場都很有把握的案子，越值得安排外部或逆風審查。

Setup

從最小日常場景看偏差怎麼啟動

團體迷思不是一群人突然變笨，而是保留意見開始沒有安全出口的那個時刻。

最小劇情

設計評審裡大家都知道時程很緊、方案也有洞，但因為負責人態度很強、全場又想趕快過關，最後只留下看起來一致的支持紀錄。

真正難點

難點在於團體迷思在當下常看起來像效率、默契與高信心，而不是像偏差。

維持和諧與速度的壓力升高
→
異議開始自我審查
→
群體信心上升但檢查深度下降
Key Point

團體迷思真正可怕的地方，不是大家意見一樣，而是不同意見在還來不及被檢查時就先消失了。

Lenses

別讓一個標籤吃掉整個機制

這裡把一致壓力、異議成本和風險檢查拆開，因為共識出錯通常不是單一人格問題。

自我審查

成員會先在心裡把保留過濾掉，避免破壞氣氛或招來負面標籤。

領導影子

只要權威立場太早太強，整場討論就更容易往確認而非檢查滑去。

一致幻覺

沒人反對常被當成全體認同，反而進一步壓低剩餘異議的存活機會。

心理拉力 當下看起來像什麼 代價
高和諧但低異議 會議很順 決策盲點往往最難在當下被看見
刻意保留反方檢查 過程較慢較吵 卻較能提早抓出群體正在共同忽略的東西
領導先表態太滿 方向很清楚 其他人的校正功能很容易一起失效

Psychology

大腦為什麼會把這條近路誤認成好判斷

只要場面明顯偏向某個答案，人就會先算說出反話的社會代價，再算那個反話本身對不對。

歸屬需求

人自然不想成為那個破壞氣氛的人，尤其在高壓時刻更是如此。

一致安全感

只要大家看起來一致，群體就會主觀感到比較安全。

責任稀釋

群體裡做錯時，個體較容易覺得『應該不是只有我沒看見』。

Applications

放回真實現場後它會改變什麼

回到設計評審、跨部門會議與高壓時程，這個概念能幫你看見哪裡正在把檢查能力換成表面整齊。

設計評審

指派反方或預先匿名收保留意見，常比現場即興問『有人有問題嗎』更有用。

事故前檢討

越是大家都覺得方向對時，越值得安排一輪 premortem 或外部檢查。

跨部門決策

讓不同部門先各自寫下主要風險，再合併討論，能降低現場從眾壓力。

  • 會議設計: 用流程保護異議，而不是只問一句『有人反對嗎』。
  • 評審文化: 讓高信心提案接受外部或逆風檢查。

Misreadings

這個偏差最常被怎樣誤解

若一開始就把 團體迷思 聽成「所以共識都很危險」，後面連「不是。問題不在有共識，而在共識是不是經過足夠異議與檢查。」這層校正都很難真正進場。

所以共識都很危險 校正

不是。問題不在有共識，而在共識是不是經過足夠異議與檢查。

只要有人唱反調就不會團體迷思 校正

若反對沒有制度支撐，很容易被快速消化掉。

這是團隊感情太好才會發生 校正

高壓、權威強和目標緊迫的團隊同樣容易中。

Questions

兩個值得帶走的校正問題

現在這個場面裡，誰最可能其實有保留但最不敢講

那個位置通常最值得被主動保護。

如果今天一定要有人當反方，我們最希望他挑哪一種洞

先把這題說清，檢查才有入口。
