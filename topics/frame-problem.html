框架問題

Frame Problem

真正困難的不是讓系統知道某個行動會改變什麼，而是讓它知道其他幾乎所有沒變的東西不必每次都重算一遍。

Read For

分清楚『知道會變什麼』和『知道其他什麼不必重新想』之間的差別，避免低估常識推理的成本。

Orientation

最難的常不是要加進新資訊，而是要排除海量其實不用重算的舊資訊

很多人以為 框架問題 只是理論爭辯；真正卡人的地方，是 只要系統得在真實世界裡行動，它就必須判斷哪些後果重要、哪些背景維持不變；這件事遠比列出單一規則難得多。

Core Tension

只要系統得在真實世界裡行動，它就必須判斷哪些後果重要、哪些背景維持不變；這件事遠比列出單一規則難得多。

01 先問行動之後哪些變化真的重要

如果所有可能後果都一視同仁，系統會被自己要更新的世界壓垮。

02 再問哪些背景可暫時視為穩定

常識推理的核心之一，就是知道哪些東西暫時不用重算。

03 最後承認相關性不是免費的

只要場景稍微複雜，相關性篩選本身就會成為主要難題。

Setup

從最小思想實驗開始

框架問題真正麻煩的不是列規則，而是世界一動起來之後，你還得知道哪些東西根本不用跟著重算。

最小劇情

機器人把杯子從桌上移到架上後，系統不只要知道杯子位置變了，還要知道房間牆壁、燈光、其他家具大多沒變，否則推理成本會立刻爆炸。

真正難點

難點在於人類對『哪些東西大致維持原樣』幾乎是無感完成，所以很容易低估那其實是巨大背景知識工程。

執行一個行動
→
世界裡有些事情變了
→
系統還得知道更多事情其實沒變
Key Point

框架問題不是小小的邏輯瑕疵，而是任何想在真實世界裡持續行動的系統都得面對的相關性管理。

Lenses

真正有差別的是你保住哪個判準

困難不在寫出規則，而在每次世界有事發生時，系統要知道哪些後果值得追、哪些背景其實根本沒動。

世界更新

行動不只產生主效果，還牽動系統要如何維護對世界狀態的整體描述。

相關性篩選

真正昂貴的不是列出所有事實，而是知道在此刻哪些事實值得被重新帶進來。

常識背景

人類習以為常的背景穩定性，對系統卻常是需要被明示或學會的能力。

分析線 它先保住什麼 會放掉什麼
只列主要效果 規則看起來很乾淨 一進複雜環境就不知道哪些背景仍成立
每次重算所有狀態 理論上完整 計算與表示成本會迅速失控
有選擇地維護背景 設計較麻煩 卻比較接近真實世界裡可持續運作的推理

Psychology

人為什麼會在這裡混用兩套標準

人之所以常低估這題，是因為對我們來說『大部分東西都沒變』太自然，反而不容易看見那是巨大能力。

常識透明錯覺

人類太習慣自動忽略不重要變化，所以會低估這份篩選本身的難度。

規則幻覺

只要主效果規則寫得很清楚，人就容易誤以為世界更新問題已經解決。

背景失聲

那些沒有變的東西因為不吵不鬧，反而最容易在設計時被完全忘記。

這段不是替 框架問題 選立場，而是讓 常識透明錯覺、規則幻覺 這類直覺如何混掉前提與標準變得可見。

Applications

把它帶回 AI、模型與工具設計

狀態管理、規則引擎和機器人規劃都會在這裡卡住，因為重算太多和假設不變都同樣危險。

代理工具設計

任何會根據使用者動作持續維護狀態的工具，都得想清楚哪些上下文該保留、哪些該重算。

機器人任務規劃

真實環境中的相關性判斷，常比單一步驟規劃更難。

事件驅動系統

系統收到新事件時，哪些快取、規則和衍生狀態真的需要更新，這就是工程版的框架問題。

  • 狀態管理: 別讓每次事件都把整個世界重算一遍。
  • 規則引擎: 先定義哪些背景前提該維持、哪些事件會真正使其失效。

Misreadings

幾種看起來乾脆但其實偷換前提的讀法

若一開始就把 框架問題 聽成「這只是早期 AI 的理論麻煩」，後面連「任何要在狀態豐富世界裡持續推理的系統，都還會遇到它。」這層校正都很難真正進場。

這只是早期 AI 的理論麻煩 校正

任何要在狀態豐富世界裡持續推理的系統，都還會遇到它。

只要更多算力就能硬解 校正

算力能幫忙，但不會自動告訴你什麼相關、什麼可忽略。

只跟機器人有關 校正

介面狀態管理、快取更新與規則引擎一樣在面對相同結構。
