格雷碼

Gray Code

格雷碼讓相鄰數值只改變一個 bit，降低多位元同時翻轉被取樣成錯誤中間值的風險。

Orientation

相鄰位置只讓一個 bit 動

Gray code 的典型形式是 binary reflected Gray code；相鄰碼字的 Hamming distance 為一。

Core Tension

數學上 7 接 8 很自然，但硬體接點、光學讀頭或跨 clock sampling 不會保證多個 bit 同時翻轉。

01 先看相鄰差異

如果相鄰狀態會同時改多個 bit，取樣窗口就會變危險。

02 再看錯誤後果

Gray code 把中途錯讀限制在相鄰碼字附近，而不是跳到遠處。

03 最後轉回內部格式

系統常在邊界用 Gray code，內部運算再轉回 binary。

Setup

0111 到 1000 不會在物理世界同時完成

自然二進位從 7 到 8 要四個 bit 同時翻轉；若其中兩個接點先動，讀值可能短暫變成完全錯的位置。

最小場景

旋轉編碼盤跨過邊界時，讀頭看到 0111、0101、0000、1000 之類的中間組合。

真正難點

錯誤不是來自解碼器算錯，而是來自物理轉換需要時間。

Key Point

格雷碼用編碼順序吸收硬體不同步，而不是要求硬體完美同步。

Lenses

一位差異如何改變邊界風險

Gray code 的價值通常出現在數位邏輯與類比、機械或非同步邊界交會處。

Hamming distance

相鄰碼字距離一，代表一次轉換只需一位翻轉。

Cyclic adjacency

某些 Gray code 首尾也只差一位，適合環狀位置。

Conversion cost

內部要比較大小或加減時，通常要轉成 binary。

Sampling window

若接收端可能在轉換中取樣，Gray code 能降低不合法中間態。

場景二進位風險Gray code 幫助
Rotary encoder 多接點不同步造成大跳值 相鄰位置最多一位不同
Async FIFO pointer 跨 clock domain 同時取樣多 bit 同步指標時降低錯誤組合
QAM mapping 鄰近星座點誤判造成多 bit error 鄰近 symbol 通常只差一 bit

Psychology

人腦喜歡十進位順序，硬體在乎轉換路徑

工程師若只看數值表，會覺得 binary 最直接；但硬體真正經歷的是一段從舊碼到新碼的過程。

忽略過渡瞬間

規格表只列穩態碼字，卻沒描述接點切換的順序差。

把解碼錯當雜訊

大跳值可能不是訊號太吵，而是編碼在邊界不適合。

同步器信心過高

多 bit bus 跨時鐘時，單顆 synchronizer 不能保證整個數字一致。

遇到跨域或位置取樣問題時，先問相鄰狀態有幾個 bit 會同時改變。

Applications

在邊界用 Gray，在內部再做運算

Gray code 常是介面編碼，而不是整個系統的算術格式。

位置編碼器

絕對式旋轉編碼盤可降低接點邊界的非同步讀值跳動。

Asynchronous FIFO

write/read pointer 以 Gray code 跨 clock domain，降低多 bit sampling mismatch。

通訊調變

星座圖相鄰點使用 Gray mapping，可讓鄰近誤判造成較少 bit error。

  • 若使用 Gray counter，文件要標出邊界格式與內部 binary 格式的轉換點。
  • Gray code 降低相鄰轉換錯讀，不取代 debouncing、filtering 或 synchronizer chain。

Misreadings

格雷碼不是更神秘的二進位

它解的是相鄰轉換錯讀，不是資料壓縮、加密或一般錯誤修正。

可以防所有讀值錯誤 校正

若訊號品質太差或跳過多個位置，Gray code 也無法保證正確。

內部運算也更方便 校正

Gray code 對相鄰轉換友善，但算術運算通常不如 binary 直接。

用了就不用同步 校正

跨 clock domain 仍需要 synchronizer；Gray 只是降低多 bit 同時變的危害。
