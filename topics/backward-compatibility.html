Core Tension

一個系統若每次演化都讓舊客戶、舊資料或舊設備立刻失效，升級就會變成破壞；向後相容是在新功能與既有生態之間找承接方式。

01 先圈出 演化承接 會在哪裡破

演化承接 常不是抽象風險，而是會穿過具體接口、排程或權限邊界。

02 再追 舊客戶 怎麼穿過邊界

舊客戶 如果沒被提前畫出來，就很容易在現場才以更貴的形式出現。

03 最後查 破壞風險 何時從小事變事故

破壞風險 一旦被低估，局部看似沒事的設計很快就會變成穩定性壓力。

Setup

風險不是突然出現，而是 舊客戶 一路堆出來的

把問題壓回一個最小接口或一次時序交會，演化承接 何時變成風險、破壞風險 何時開始放大，就不容易再被模糊帶過。

最小劇情

協定、檔案格式、韌體與 API 只要在現場還有舊版本存活，就得思考升級不是只改新端，而是整個生態怎麼平穩過渡。

Lenses

先把 升級不是重開局、舊端存活壓力 與 承接成本 分開看

表面像同一個故障，底下往往是 升級不是重開局、舊端存活壓力 和 承接成本 在不同邊界各自出力。

升級不是重開局

升級不是重開局 通常不會單獨出現；它多半會穿過接口、權限或排程邊界，把 演化承接 轉成真正的穩定性壓力。

舊端存活壓力

舊端存活壓力 通常不會單獨出現；它多半會穿過接口、權限或排程邊界，把 舊客戶 轉成真正的穩定性壓力。

承接成本

承接成本 通常不會單獨出現；它多半會穿過接口、權限或排程邊界，把 破壞風險 轉成真正的穩定性壓力。

Psychology

團隊通常在哪一步把 向後相容 當成暫時小事

工程現場最危險的，不是沒人提風險，而是大家太容易把 演化承接 當成晚點再補也行的小瑕疵。

新功能值得一切錯覺

新功能值得一切錯覺 會讓人把『目前沒爆』誤讀成『設計沒問題』，於是風險一路被推往更後面。

舊包袱被浪漫化或被忽略

舊包袱被浪漫化或被忽略 最危險的地方，在於它常把局部成功包裝成全局安全。

測試只看最新版

只要 測試只看最新版 一進場，團隊就容易先相信現況撐得住，而不是先問代價正在往哪裡堆。

越早把 演化承接、舊客戶 這些心理近路說出來，越不容易在 postmortem 才第一次承認它們存在。

Misreadings

三個常見誤讀

這些誤讀常把 向後相容 降成實作瑣事，結果真正的邊界壓力根本沒被談到。

向後相容 只是實作細節 校正

它常是 演化承接 最先破掉的邊界，所以不是『之後再補』就沒事。

只要再加更多防呆，向後相容 就會消失 校正

如果 舊客戶 的壓力來源沒處理，防呆最後只會變成更昂貴的補丁。

向後相容 只和大型分散式系統有關 校正

很多單機、韌體或流程系統，一樣會以更小規模的 破壞風險 形式重演。
