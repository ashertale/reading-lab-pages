基準率忽略

Base Rate Neglect

當個案敘事很鮮明時，人會把整體分布忘掉，讓判斷看起來像在讀證據，其實是在追故事。

Orientation

當鮮明線索壓過母體分布

基準率忽略不是不會算數，而是大腦太容易被具體故事吸走，導致分布資訊在關鍵時刻被邊緣化。

Core Tension

個案細節讓你覺得自己抓到了真相，但沒有 base rate 的故事，只是在沒有底圖的情況下放大局部線索。

01 先問母體

在討論個案前，先說清楚事件在整體母體中的稀有程度。

02 再看證據力度

不是每一條線索都足以推翻先驗分布，很多線索只是讓故事更像。

03 最後才更新

判斷應該是 prior 被 evidence 修正，而不是被單一案例直接取代。

Setup

把問題縮到最小

這個偏誤最常出現在你覺得自己很重視證據的時候，因為個案線索往往比抽象比率更有存在感。

最小劇情

某人看到一個看起來很像資安入侵的異常封包，於是立刻判斷系統已被攻破，卻沒先問該型態在正常流量裡本來就有多常見。

真正難點

只要案例描述夠具體，人的直覺就會覺得它比冷冰冰的母體比例更可靠，哪怕兩者其實應該一起看。

個案細節很鮮明
→
整體分布被忘掉
→
低機率事件被誤判成高機率
Key Point

基準率忽略最危險的地方，不是算錯，而是把代表性的感覺誤當成真正的機率更新。

Lenses

三個拆解這個偏誤的角度

如果你想避免被個案帶著走，可以把判斷拆成三層：母體、證據、更新幅度。

母體分布

先問事件本來有多常見。罕見事件需要更強的證據才值得大幅改判。

證據品質

線索到底是真正區分度高，還是只是語意上看起來很像答案。

更新幅度

好的判斷不是全有全無，而是根據證據強弱逐步調整信心。

看法 看起來像 真正風險
只看個案像不像 直覺很有把握 稀有但吸睛的解釋被過度高估。
先看基準率 判斷沒那麼戲劇化 若證據真的很強，仍要允許大幅更新。
把兩者一起算 思路比較慢 需要紀律，但能避免把故事錯當分布。

Psychology

為什麼個案故事總比基準率更有說服力

人比較容易記住臉、事件與情節，不容易記住抽象的分母，所以 base rate 常在心理上先輸一半。

故事比比例更抓人

一個具體案例會讓人感到自己正在接觸真相，分布資料則常被誤認成抽象背景噪音。

代表性會冒充機率

只要某個案例很像你心中的典型，大腦就會把像不像直接轉譯成多不多。

重點不是要求每次都手算 Bayes，而是提醒你不要讓 vivid cue 直接霸占判斷。

Applications

把先驗機率放回告警與診斷

在工程與產品現場，這個偏誤常出現在 debug、告警解讀、異常分流與風險評估。

異常告警判讀

某個指標長得像攻擊、不代表最可能是攻擊；若誤報本來很多，就不能直接把單次命中當定案。

Field issue 分流

看到兩台機器同時出現同一症狀，直覺會想成 systemic bug，但也要先看該症狀在正常使用裡本來就多不多。

  • 遇到強烈個案時，先補一句：這件事在母體裡有多常見？
  • 把判斷寫成「先驗 + 證據 + 更新」，比直接喊結論更不容易失真。

Misreadings

別把它縮成一句「學 Bayes 就好」

很多人知道要看證據，卻忘了沒有 base rate 的證據其實缺了一半脈絡。

所以只要看基準率 校正

不是。基準率提供底圖，但強證據仍然能大幅修正判斷。

只有統計人才會遇到 校正

任何需要分辨常見噪音與罕見真因的工作，都在跟這個偏誤打交道。

Questions

用兩個提問把分布請回桌上

下面兩個問題，適合在看到非常像答案的案例時先拿來踩煞車。

如果這類事件其實很罕見

你現在手上的證據，真的強到足以推翻低基準率，還是只是敘事上很有代表性？

如果把分母補回來

你是否還會做同樣判斷，還是信心其實應該比直覺低很多？
