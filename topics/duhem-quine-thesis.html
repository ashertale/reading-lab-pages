Duhem-Quine Thesis

Orientation

失敗結果通常只告訴你某個組合不對

一個預測依賴主理論，也依賴儀器、資料處理、環境條件和很多未明說的假設。

Core Tension

當觀察不符合預期，你可以放棄主假說，也可以懷疑儀器、樣本、初始條件或統計方法；證據本身不會自動指定要丟哪一個。

01 列出依賴鏈

把預測成立需要的背景條件攤開。

02 不要單點定罪

觀察反例先排除常見量測與操作問題。

03 限制修補自由

輔助假設不能每次失敗都臨時改到理論永遠不可受傷。

Setup

模型預測錯了，可能錯在模型，也可能錯在資料管線

最小場景是一個指標異常，團隊要判斷理論是否被反駁。

最小劇情

風險模型預測低風險帳號不會大量退款，但新週期退款暴增。調查發現同時有促銷規則改版、標籤延遲和一批新使用者來源。

真正難點

觀察結果與模型不合，卻不足以單獨說模型理論錯；整個推論環境都要被檢查。

理論產生預測

→
檢查整組依賴
Key Point

杜昂-蒯因論題讓反駁變得更謹慎：你要說明為何修這個環節，而不是別的環節。

Lenses

把反例拆成一張信念依賴圖

越複雜的系統，越需要看預測背後的附帶條件。

主假說

核心理論或模型是否真的導出這個預測？

輔助條件

量測、樣本、初始化和環境是否照預期成立？

修補原則

修改哪個假設最節省、最可檢驗、也最不投機？

Psychology

團隊常選擇最不痛的修補

證據不足以自動決定犧牲誰，因此組織利益會影響解釋路徑。

保護核心投資

昂貴模型或長期理論較不容易被立即放棄。

責任轉移

失敗可能被推給資料、操作或例外樣本。

修補成癮

每次都補一個 auxiliary，最後理論看似安全卻失去可測性。

當你聽到這不是模型錯，是資料問題，請要求畫出依賴鏈和可驗證的修補。

Applications

把實驗失敗當成依賴鏈檢查

科學實驗、軟體 debug 和 ML 評估都會面對多重假設糾纏。

系統 debug

錯誤輸出可能來自演算法、cache、時鐘、權限或資料同步。

ML 評估

模型退化可能是分布漂移，也可能是標籤政策改變。

實驗設計

預先記錄輔助假設，能降低事後任意修補。

Misreadings

不是說任何理論都無法被反駁

它說反駁需要連同背景條件一起處理，而不是把證據變成免疫盾牌。

滑向相對主義 校正

證據仍然有力量，只是不會單獨完成所有歸因。

無限保護理論 校正

輔助假設若不可測，就會削弱理論品質。

忽略實務判準 校正

簡潔、可重現和預測力仍可引導修補選擇。

Questions

問這個預測依賴哪些前提

除了主假說，還有哪些條件必須為真？

列出資料、儀器、環境和推論步驟。

我們為何修改這一項，而不是其他項？

要求修補具備可測性和事前理由。

延伸路線

可接著讀 theory-ladenness of observation，理解觀察如何被背景理論塑形。
