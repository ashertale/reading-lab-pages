Cyclic Redundancy Check

CRC 用 GF(2) 多項式除法替資料加上一小段檢查值；它擅長抓偶發錯誤，但不是防竄改的安全機制。

Read For

適合設計 UART/SPI 封包、bootloader image、NVM 資料區與任何需要檢查 accidental corruption 的格式。

主題類型
錯誤偵測碼 / 資料完整性

Orientation

一小段餘式，守的是資料有沒有走樣

CRC 把 bit string 視為 GF(2) 多項式，除以約定的 generator polynomial，將餘式作為檢查值附在資料後。

Core Tension

它很適合抓通道噪聲、儲存腐敗與 burst error；但若參數沒寫清楚，兩個都叫 CRC-16 的實作可能完全不相容。

01 先定義完整參數

polynomial、init、refin/refout、xorout 與 check value 都要進規格。

02 再對齊錯誤模型

不同多項式對短訊息、長訊息與 burst error 的偵測能力不同。

03 最後別拿來防攻擊

CRC 是錯誤偵測，不是認證；惡意者可修改資料後重算。

Setup

兩端都說 CRC-16，封包卻永遠不通

一個 MCU 和 host tool 都實作 CRC-16，但 host 用 reflected input，MCU 用 normal bit order，測試時每包都 mismatch。

最小場景

規格只寫 CRC-16-CCITT，沒有給 init、xorout、bit order 與標準字串 check value。

真正難點

CRC 名稱常像共同語言，實際上參數族非常多；沒有測試向量就無法保證互通。

Key Point

CRC 規格不是一句演算法名，而是一組可重現的參數與測試向量。

Lenses

設計 CRC 時別只問幾位元

CRC-8、CRC-16、CRC-32 只是長度；真正的偵測能力取決於多項式與訊息長度。

Polynomial

generator polynomial 決定哪些錯誤型態容易被抓到。

Parameter set

init、reflection、final xor 與 residue 會影響互通結果。

Message length

同一多項式對不同最大訊息長度有不同 Hamming distance。

Threat model

偶發 corruption 用 CRC；惡意 tampering 需要 MAC、signature 或 authenticated encryption。

用途CRC 適合度補充機制
UART 封包噪聲 很適合 定義長度、序列號與重傳
Firmware image 完整性 可抓 accidental corruption 若要防竄改需簽章
NVM 設定區 適合偵測 bit rot 或寫入中斷 搭配版本、雙 bank 或 journal

Psychology

名字熟，反而讓人少問參數

CRC 太常見，工程師容易跳過最重要的規格細節。

複製貼上

網路範例可能使用不同 endian、reflection 或 initial value。

測試資料太乾淨

只用空字串或單一短封包測試，抓不到參數差異。

安全語感混淆

checksum、hash、signature 都在檢查資料，但威脅模型完全不同。

CRC review 最有用的問題通常很樸素：標準字串 123456789 的結果是多少？

Applications

封包、映像檔與 NVM 都需要不同規格粒度

韌體使用 CRC 時，關鍵不是把函式呼叫上去，而是把資料邊界與錯誤處理寫清楚。

通訊封包

CRC 覆蓋 header 還是 payload、是否包含 length，都會影響錯誤偵測與解析流程。

Bootloader image

CRC 可先抓傳輸或 flash corruption；正式安全啟動仍需簽章驗證。

NVM 資料

設定區可用 CRC 加版本號判斷資料是否完整，並在失敗時回退 default 或另一份 copy。

  • 規格中放入至少一組完整測試向量，包含輸入 bytes 與預期 CRC。
  • 把 CRC failure 的 recovery path 寫出來；偵測到錯誤但無法恢復，系統仍然不可靠。

Misreadings

CRC 不是加密，也不是萬用 hash

它的強項是快速偵測偶發錯誤，不是抵抗有意識的修改。

CRC 可以防止竄改 校正

攻擊者可改資料再重算 CRC；需要 authentication 才能防竄改。

CRC-32 一定比 CRC-16 好 校正

長度重要，但多項式與最大訊息長度也會影響偵測能力。

同名 CRC 必定相容 校正

CRC-16 家族有多種參數；不給 check value 就容易錯接。

Questions
