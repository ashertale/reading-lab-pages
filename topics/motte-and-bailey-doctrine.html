城寨論證

Motte-and-Bailey Doctrine

有些說法在安全時會提出大而有利的主張，一旦被質疑，就退回一個較弱但較難反駁的版本，等風頭過了再悄悄把兩者混回一起。

Read For

這一頁要把『我只是說一個溫和版本』和『剛才明明推的是更強版本』拆開，因為很多低品質爭論都靠這種滑移撐住。

論證策略 / 防守退位

Orientation

最難纏的論證，常不是錯得明顯，而是會在被打時自動縮成另一個版本

城寨論證之所以難處理，是因為它利用了兩種命題的模糊邊界：一個好推廣，一個好防守。

Core Tension

較強主張能帶來影響力，但不容易站住；較弱主張能防守，卻不足以支撐原先的雄心。把兩者混用，就會讓辯論總卡在影子裡。

01 先把命題拆清楚

對方現在在主張的是強版本，還是弱版本？

02 再要求版本一致

若退守到弱版本，就不能繼續享受強版本帶來的結論與姿態。

03 最後分開評估

兩個版本各自是否成立，應分開討論而不是混成一個模糊地帶。

Setup

從一個你一質疑，對方就說『我又沒說那麼多』的討論開始

城寨論證在公共討論、策略辯護與內部提案時都可能出現。

最小劇情

平常有人說『這套流程證明我們就是最先進的團隊』；一被追問衡量標準，就退回『我只是說它有一些值得肯定的地方』。

真正難點

若不把版本拆開，討論會一直在強主張獲利、弱主張避打之間來回。

先推高擴張主張
→
被質疑時退到較弱版本
→
安全後再把兩者混回一起
Key Point

城寨論證最值得警覺的地方，是它讓討論表面看似退讓，實際卻沒有真的釐清主張。

Lenses

四個看語意滑移的角度

不是每次澄清都在偷換，但如果退守版本無法支撐原先結論，就需要把它拆開。

主張強度

原先主張是否明顯比現在防守版更大、更有野心？

結論範圍

現在的弱版本是否其實撐不起剛才的強結論？

語意模糊度

哪些詞刻意模糊，讓強弱版本容易混用？

回切時機

被質疑時退、風險一過又進，是否呈現規律？

階段 話術看起來像 真正發生什麼
推進期 主張很大 借強版本拿影響力與姿態
防守期 我只是說較弱版本 借弱版本逃避原本的可檢驗承諾
收尾期 兩者好像差不多 把本該分開的命題重新混成一團

Psychology

為什麼這種滑移特別有效

因為人很容易記住氣勢較大的版本，卻在當下被較安全的退守版安撫。

強主張更有吸引力

它能快速帶出立場、身份與道德高度。

弱主張較難反駁

一旦退守到常識版，批評者看起來就像在挑小毛病。

記憶會模糊界線

聽者過一陣子後，往往只記得『好像大方向成立』。

城寨論證不是指每次澄清都不誠實，而是指強弱主張被策略性混用。

Applications

把版本對齊帶回提案與討論

對團隊來說，這個概念最有用的地方，是避免提案與爭論在模糊版本間來回滑動。

策略提案

先用大口號爭取支持，再在實作細節被問時退成模糊常識，最終讓承諾無法被驗證。

架構辯論

某設計平常被說成唯一正道，受挑戰時又改口成只是其中一種可行取向。

文化論述

口號很容易是強版本，真正落地時卻只剩弱版本可守。

  • 一旦發現主張在滑動，最有效的做法通常是把強版與弱版寫成兩句分開討論。
  • 如果有人退到弱版本，請同步問：那原先靠強版本得出的結論還成立嗎？

Misreadings

三個常見誤讀

城寨論證不是禁止修正說法，而是要求修正後別偷帶回原本的擴張收益。

只要改口就是城寨論證 校正

不是。真正關鍵在於強弱版本是否被策略性混用。

所有模糊表述都惡意 校正

有時只是概念未釐清，但一旦混用持續就需要拆開。

這只在公共辯論有 校正

內部提案、策略說明與技術宣稱都可能出現。

Questions

他現在守的是哪個版本，而這個版本還撐得起原來那句話嗎

如果只接受他現在的弱版本，原先較大的結論是否還能成立

若不能，就代表版本必須分開算。

哪些詞讓強弱版本可以被偷偷接上

把那些詞拆開通常會讓討論清楚很多。
