可辨識受害者效應

Identifiable Victim Effect

一個有名字、有畫面的人，比一群抽象受影響者更容易啟動同理；資源分配因此可能被故事而非規模牽引。

Read For

學會保留具體故事的動員力，同時把它接回分母、規模與公平排序。

Orientation

同理心很擅長抓住一個人，卻不自動懂得分配資源

可辨識受害者效應提醒我們，故事會讓痛苦變得可感，但故事的鮮明度不等於問題的優先級。

Core Tension

沒有故事，統計很難推動行動；只有故事，資源又可能流向最有臉孔而不是最需要的地方。

01 先保留故事

具體個案讓人看見機制與痛苦，不該被冷冰冰地排除。

02 再補上分母

問這個案例代表多少人、多少次、多少損失，以及是否典型。

03 最後回到分配

決定資源時要把鮮明度、嚴重度、規模與可修復性一起比較。

Setup

一封痛苦長信，可能比一千筆沉默資料更快排進 sprint

最小場景是一個具體使用者讓團隊很有感，但更大的群體問題只以數字存在。

最小劇情

客服轉來一位客戶的詳細抱怨，附上截圖與故事。團隊想立刻修他的痛點，卻忽略另一個每天影響數千人的小錯誤，因為它只在 dashboard 裡是一個百分比。

真正難點

個案很真實，統計也很真實；困難是不要讓較有畫面的真實壓過較大的真實。

具體故事喚起同理
→
抽象群體失去畫面
→
資源排序被鮮明度牽動
Key Point

可辨識受害者效應的解法，是讓故事成為入口，而不是讓故事單獨決定出口。

Lenses

把個案放回代表性與分配問題

一個故事值得被聽見，但它需要回答自己代表什麼。

代表性

這個個案是典型、極端、早期警訊，還是特殊條件下的例外？

規模

相同痛點影響多少人、多久、造成多大損失？沒有分母，故事會過度膨脹。

可修復性

修這個痛點能否同時幫到更多人，還是只處理單一個案？

Psychology

人會對臉孔負責，卻很難對大數字產生同等情緒

可辨識的人讓責任變近；抽象群體需要額外設計才能取得同樣注意。

故事降低想像成本

有名字和情境的人讓痛苦具體化，行動意願因此更容易啟動。

大數字讓情緒變鈍

數量越大，越容易變成抽象背景，反而不如一個故事有推力。

救得到一個人的感覺很強

可見的改善讓人覺得行動有效，群體改善則常比較難被情緒追蹤。

校正這個偏誤，不是削弱同理，而是讓同理能服務更公平的分配。

Applications

把故事變成 triage 線索，而不是完整排序

產品、公益與風險團隊都需要個案故事，但不能只靠故事排優先級。

產品支援

把高情緒客訴轉成可查欄位：影響人數、頻率、嚴重度與是否阻斷核心流程。

公益決策

募款故事應搭配每筆資源能幫助多少人，避免全部流向最會被想像的案例。

安全風險

單一事故很有畫面，但風險投資仍要看暴露面、重複率與可預防性。

Misreadings

不是要把具體受害者變回冷冰冰的數字

故事是必要入口；錯誤是把入口當成整個判斷。

以為講故事就是操控 校正

故事可以揭露真實機制；是否操控取決於有沒有補上代表性與分母。

只看最大群體 校正

規模重要，但嚴重度和可修復性也重要。最大不一定最優先。

把個案和統計對立 校正

好的決策會讓個案說明痛點，讓統計校準資源。

Questions

讓故事回答它代表什麼

下面兩問適合放在客服 triage、公益評估與風險排序。

這個案例代表多少人？

它是典型案例、極端案例，還是提醒我們去查更大樣本的線索？

如果沒有名字和截圖，我還會排同樣優先級嗎？

拿掉鮮明度後，嚴重度、規模與可修復性是否仍支持這個決策？

延伸路線

可接著讀 scope neglect，理解大規模影響為何不會自動帶來大規模情緒。
