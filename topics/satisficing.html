滿意化

Satisficing

在資訊有限、時間有限、注意力有限的現實裡，很多時候與其追最優，不如先找到足夠好的解，讓系統繼續前進。

Read For

這一頁要把『不夠完美』和『在現實限制下已足夠合理』拆開，因為很多卡住不是因為標準太高，而是忘了搜尋本身也有成本。

主題類型
決策策略 / 有限理性
核心問題
Choosing an option that is good enough rather than globally optimal
主要判準
搜尋成本、最低門檻、最優幻覺、行動速度
讀完產出
能更清楚地在需要最優與只需夠好的場景之間做區分

Orientation

不是每一題都值得找到全域最優，有些題目最重要的是不要把自己困死在搜尋裡

滿意化是有限理性的成熟策略，不是偷懶版本的理性。

Core Tension

追最優能提升品質，但如果搜尋成本高到吞掉行動機會，最優追求本身就會開始不划算。

01 先定最低可接受門檻

沒有門檻就容易在選項海裡一直漂。

02 再把搜尋成本算進來

多比較一輪不是免費，它會消耗時間與注意力。

03 最後區分高風險與低風險場景

不是所有題目都適合滿意化，但很多日常題目很適合。

Setup

從一個你一直想找到最好，結果遲遲沒辦法開始的選擇開始

滿意化特別適合資源有限、可逆性中等、選項很多的現實情境。

最小劇情

你一直想選最理想的工具鏈，結果比較了很久仍無法開始；其實一套符合關鍵條件、可日後再調的方案已足夠推進工作。

真正難點

真正要接受的不是品質變差，而是有些題目的最優資訊永遠拿不到。

搜尋空間很大
→
最優追求成本持續上升
→
改用足夠好門檻讓決策前進
Key Point

滿意化最有用的地方，是把有限理性從缺陷翻成策略。

Lenses

四個看夠好策略的角度

要用得好，關鍵不是降低標準，而是知道哪個標準是此刻最值得守的。

門檻清晰度

你知道什麼叫夠好嗎，還是只是隨便接受？

可逆性

選錯後是否還有調整空間？

搜尋成本

再花一輪找更好，值得嗎？

風險後果

這是能先前進再迭代的題，還是一次選錯代價很高的題？

情境 追最優的代價 滿意化能提供什麼
工具選型 比較時間吞掉開工時機 先用符合核心需求者上手
招聘初篩 每位都要最完整比較很慢 先用最低門檻過濾再深入
專案決策 想找完美方案導致停滯 先選足以安全前進的路徑

Psychology

為什麼人有時寧願卡住也不願接受夠好

因為『夠好』常被誤讀成將就，而『最優』則帶著一種道德上的高標幻覺。

最優承諾很迷人

它讓人覺得自己很認真、很負責。

將就感令人不安

即使方案已足夠好，人仍可能怕自己之後後悔。

搜尋成本不直觀

花時間比較看起來像努力，卻不總是有回報。

滿意化不是放棄思考，而是把思考放在最值錢的地方。

Applications

把夠好門檻帶回工程與管理決策

對工程團隊來說，很多拖延其實不是缺能力，而是沒有先定出『可以開始』的門檻。

架構選型

先找符合安全、效能與維運底線的方案，比一直等理想解法實際。

文件品質

先做到足以讓新人不迷路，比追一次寫到完美更常能真正發生。

產品迭代

若版本可回收，先推夠好方案收真實回饋，常比紙上找最優更有效。

  • 想用滿意化，先把最低可接受標準寫出來，否則很容易退成隨便。
  • 如果一個決策可逆又高頻，通常不值得為了理想最優耗掉太多前進時間。

Misreadings

三個常見誤讀

滿意化不是廉價妥協，而是有限理性條件下的實用策略。

夠好就是不專業 校正

在高搜尋成本情境下，懂得設門檻反而更專業。

任何事都該滿意化 校正

高不可逆、高風險題目仍可能值得更嚴格搜尋。

滿意化等於沒有標準 校正

相反地，它最需要清楚標準。

Questions

我是在追更好，還是在為了追最優而延誤真正重要的前進

如果今天必須做決定，我的最低可接受門檻是什麼

把門檻說出來通常就能縮短僵局。

再多找一輪，真的有很大機率找到顯著更好解嗎

若沒有，搜尋本身可能已不值。
