過度正當化效應

Overjustification Effect

當一件原本願意自己做、也覺得有意義的事，被過度包裝成獎懲與外部誘因後，人可能反而更少把它當成自己想做的事。

Read For

這一頁要把『多給誘因就會做更多』和『誘因可能重寫人做這件事的理由』拆開，因為很多管理設計都忽略了動機來源會被改寫。

Orientation

獎勵能推動行為，但有時也會把『我願意做』改寫成『我只是為了拿這個』

過度正當化效應抓到的是動機來源被替換的風險：外部獎勵不只是加法，有時會改變整個行動解釋。

Core Tension

獎勵能在短期提高產出，但若方式讓人覺得自己被控制，原本的興趣、責任感與自我驅動反而可能下降。

01 先分辨任務原本的動機來源

原本就靠興趣、責任或專業認同在撐的事，最怕被粗暴量化成獎懲。

02 再看獎勵語意

獎勵是在肯定成果，還是在把行為改寫成被控制的工作？

03 最後保留自主與意義

外部激勵若不尊重自主，長期常會吃掉內在動機。

Setup

從一件本來做得不錯、加了激勵後反而變味的工作開始

過度正當化效應在教育、社群貢獻、內部分享與創造性工作中特別值得注意。

最小劇情

團隊原本自發寫內部教學筆記；後來改成每月記點排名，短期數量上升，長期內容深度與主動性卻下降。

真正難點

問題不是獎勵本身，而是獎勵把『我想做』慢慢改寫成『我為了拿分做』。

Key Point

過度正當化效應提醒你：激勵不是只會增加，也可能置換。

Lenses

四個看誘因副作用的角度

越是仰賴興趣、認同與責任感的工作，越要小心外部獎勵的語氣與強度。

原本內在性

獎勵控制感

任務性質

創造性、教學性與關係性工作，通常比機械任務更怕被粗暴外獎改寫。

持續性

當獎勵移除後，行為是否比原本更難維持？

Psychology

為什麼獎勵有時會把事情做壞

因為人不只在做事，也在理解自己為什麼做；一旦理由被外獎接管，內在意義就可能退後。

理由會被重寫

如果外部報酬太顯眼，人會更容易把行為歸因到報酬。

控制感會被侵蝕

越像被管理，越不容易保持自發感。

移除獎勵後落差更大

當外部支架抽掉，內在動機若已被吃掉，行為就更容易掉下去。

過度正當化效應不是反獎勵，而是提醒獎勵會改寫人對自己行為的理解。

Applications

把動機保護帶回管理與學習設計

對工程團隊來說，知識分享、 code quality、 mentoring 與開發者體驗這類工作，很容易因錯誤激勵而失去原本的自發性。

知識庫貢獻

把分享完全量化成排名，可能會養出大量低價值內容。

Code quality

若只獎勵特定數字，工程師可能轉向對分數負責，而不是對品質負責。

學習文化

如果所有進修都變成被檢核任務，探索感會快速下降。

  • 在加獎勵前，先問這件事現在靠什麼內在動機在運作，別輕易把它換掉。
  • 好的激勵更像支撐與肯定，不像把人變成只對分數反應的裝置。

Misreadings

三個常見誤讀

過度正當化效應不是說不能用獎勵，而是說獎勵會帶副作用。

所有獎勵都不好 校正

不是。重點是任務性質、獎勵語意與是否侵蝕自主感。

只在小孩教育才有 校正

成人工作、社群與專業文化同樣會被影響。

只要獎勵夠大就不怕 校正

很大有時反而更容易蓋過原本的意義來源。
