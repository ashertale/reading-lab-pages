棘輪效應

Ratchet Effect

棘輪效應描述的是標準或支出一旦上調，回到原位變得很難；制度會記住高點，卻不一定記住原因。

Orientation

今天的臨時高標，明天可能變成新的最低標

棘輪效應讓一次上調變成未來基準：即使原本原因消失，組織和利益關係也會讓下降變得困難。

Core Tension

提高標準可以推動改善，但如果沒有退出條件，短期拉高會變成長期承諾，並誘發防守、灌水或隱藏資訊。

01 標明臨時性

任何緊急加班、折扣、補貼或額外審核都要寫出結束條件。

02 保留降回路徑

上調前先設計何時、由誰、根據什麼資料降回。

03 觀察防守行為

人若知道高點會變新標準，就會避免暴露真實能力或需求。

Setup

一次趕工成功，下一季就變成正常交期

最小場景是團隊為特殊情況付出額外成本，後來那個成本被當成常態能力。

最小劇情

團隊為大客戶臨時加班兩週完成客製功能。下一季銷售把同樣交期寫進標準合約，因為上次已經證明做得到。

真正難點

做得到不代表可持續；制度若只記錄高點，就會把耗損誤認成產能。

短期上調達成目標
→
高點被記成新基準
→
回退被解讀成退步
Key Point

棘輪效應的校正，是替上調加上原因、成本和降回條件。

Lenses

看高點如何被寫進下一輪規則

棘輪常不是一次命令，而是透過 KPI、預算、合約和期望逐步鎖住。

基準來源

新標準來自穩定改善，還是一次特殊投入？

受益者

誰從高基準中得利，誰承擔維持成本？

降回機制

哪些資料或決策能證明應該回到較低基準？

Psychology

人不喜歡承認回退，即使回退只是回到正常

高點具有敘事力量：曾經達到就像證明永遠可以，讓降回變得像失敗。

高點容易被英雄化

一次超常表現常被記成能力，而非代價。

利益會固化上調

享受新標準的人不一定看見維持成本。

防守行為會出現

若暴露餘裕會導致未來標準上調，人就會隱藏餘裕。

任何超常承諾都需要一張標籤：為何例外、成本多少、何時結束。

Applications

不要讓臨時上調默默變成制度

KPI、預算、客戶承諾和安全流程都容易出現棘輪。

KPI 設計

突破性數字若來自特殊活動，下一期目標要分開看可持續能力。

預算流程

避免部門因怕下期被砍而年底突擊花完預算。

產品承諾

臨時 SLA 或客製服務要註明期限，否則會進入客戶期望。

Misreadings

棘輪效應不是反對提高標準

它反對的是沒有成本標記和退出條件的上調。

把穩定改善也當棘輪 校正

若能力真的提升且成本可承受，基準上調可能合理。

只怪管理層貪心 校正

制度、合約、預算和使用者期待都會共同鎖住高點。

以為回退就是失敗 校正

回到正常水位可能是保護長期能力，而不是退步。

Questions

替每個上調寫退出條件

下面兩問適合用在 KPI、預算和承諾 review。

這次高標準來自可持續能力嗎？

請分清流程改善、臨時人力、風險透支和外部補貼。

誰有權把基準降回去？

若沒有人能降回，這次上調很可能已經被棘輪鎖住。

延伸路線

可接著讀 incentive，理解制度如何塑造人對基準和揭露的行為。
