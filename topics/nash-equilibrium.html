納許均衡

Nash Equilibrium

納許均衡描述的是一組策略互相卡住：在其他人不改的情況下，任何單一參與者都沒有理由獨自換策略。

Read For

分辨個體最佳回應、集體結果和協調困境，避免把穩定誤認成好。

Orientation

沒有人想單獨改變，不代表大家都喜歡這個結果

納許均衡關注的是策略穩定性：給定其他人的選擇，每個人的現有選擇都是自己的最佳回應。

Core Tension

均衡可能高效，也可能低效；囚徒困境中雙方背叛就是穩定但糟糕的結果。

01 固定他人策略

判斷單一玩家是否有誘因偏離時，先假設其他人不變。

02 區分均衡與最佳

穩定只是無人願意單獨改，不代表總福利最高。

03 改規則而非只勸人

若壞均衡穩定，需要改 payoff、資訊或協調機制。

Setup

兩個團隊都想寫文件，但誰先寫誰吃虧

最小場景是各方互相等待，因為單方合作成本太高。

最小劇情

兩個服務團隊都受益於共同 API 文件。若只有一方更新，另一方仍可享受成果；若雙方都不更新，文件腐爛。給定對方可能偷懶，自己也偷懶成為短期最佳回應。

真正難點

每個人的選擇在局部都合理，但合起來讓所有人都更差。

Key Point

納許均衡讓你從責怪個人，轉向檢查規則為什麼把人鎖在這組策略。

Lenses

找出每個人的最佳回應

先不要問大家應該怎麼做，先問在現有規則下各自怎麼做最划算。

玩家

誰在做策略選擇，彼此會不會影響報酬？

報酬

每種組合下成本、收益、風險和名聲如何分配？

偏離誘因

若其他人不變，單方改變是否更好？

Psychology

壞均衡會讓人看起來像不合作，其實是怕先合作吃虧

當信任不足或回報不對稱時，道德呼籲很難打破穩定狀態。

互相預期

我相信你不做，所以我也不做。

先行者懲罰

先投入的人承擔成本，收益卻被共享。

穩定造成麻木

卡久了以後，大家會把低效當正常。

要改壞均衡，通常要改獎懲、可見性或協調方式，而不是只要求大家成熟。

Applications

透過改 payoff 移動均衡

平台治理、團隊協作和市場競爭都可用均衡視角看行為。

平台規則

調整排名和費率可改變賣家的最佳回應。

團隊流程

把文件更新納入 release gate，讓合作不再是額外好心。

資安合作

共享威脅情報若有回饋和聲譽制度，參與誘因更穩。

Misreadings

納許均衡不是道德上好的結局

它只說在給定策略組合下，沒人能靠單方改變變得更好。

把穩定當效率 校正

很多低效狀態也很穩。

以為只有一個均衡 校正

同一遊戲可能有多個均衡。

忽略規則設計 校正

改 payoff 或資訊後，均衡可能移動。

Questions

問為什麼沒人願意先改

在其他人不變時，誰有理由單獨改策略？

若沒有，就先承認這是穩定狀態。

哪個 payoff 需要被改寫？

找出獎懲、資訊或承諾機制。

延伸路線

可接著讀 game theory 和 prisoners' dilemma，理解策略互動如何產生穩定但低效結果。
