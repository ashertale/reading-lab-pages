古德哈特定律

Goodhart's Law

當一個指標被拿來當目標，它就會開始失去衡量原本現象的能力。

Read For

Orientation

指標一旦成為目標，系統就開始學會作弊

古德哈特定律不反對量化。它提醒我們：當量化結果開始決定獎懲、資源與評價，指標就不再只是被動測量。

Core Tension

好的指標原本能讓我們看見複雜現象；但一旦它被宣布為目標，系統就會學會優化指標本身，而不一定優化指標背後真正重要的事。

01 先問它代理什麼

每個 KPI 都只是 proxy。先說清楚它想靠近哪個真目標。

02 再看誘因壓力

越是和升遷、預算、績效綁在一起，越容易被人優化到失真。

03 最後檢查副作用

如果指標變好但現場變差，通常不是人突然變壞，而是設計在邀請繞路。

Setup

把問題縮到最小

所有版本的古德哈特定律都可以縮成一個簡單故事：你想改善 A，但只能量到 B，於是你獎勵 B。

最小劇情

團隊想提高品質，於是把「每週關閉 bug 數」當 KPI。幾週後，關閉數上升了，但真正難修、最影響使用者的 bug 被拆小、延後或避開。

真正難點

指標不是謊言，它可能曾經有用。難點在於：當人知道它會被拿來控制結果，行為就會改變，原本的相關性也會被破壞。

用指標追蹤真目標
→
把指標綁上獎懲
→
系統開始優化指標而非目標
Key Point

古德哈特定律不是「數字都沒用」，而是「數字一旦被賦予控制權，就會改變它正在測量的世界」。

Lenses

四種看指標失真的透鏡

同一個 KPI 可能因為代理不良、誘因太強、樣本被挑選或現場刻意操弄而壞掉。把失真來源分開，修法才會不同。

代理落差

指標只是目標的替身。測試覆蓋率可以靠近品質，但它不等於品質；覆蓋到大量低風險分支，也不代表關鍵路徑更可靠。

誘因壓力

當指標決定獎懲，人會自然避開指標不鼓勵的工作。這不是道德故事，而是系統設計對行為的塑形。

選擇效應

被拿來排名或篩選後，留下來的樣本會改變。例如只看平均修復時間，團隊可能傾向先處理容易關掉的 issue。

操弄與遊戲化

當指標的計算方式透明且單一，現場就有機會直接優化計分方式，而不是改善原本想改善的現象。

失真來源 看起來像進步 實際風險
代理落差 覆蓋率、分數或件數提升 真目標沒有同步提升，甚至被錯誤指標遮住。
誘因壓力 團隊快速靠近 KPI 沒被計分的必要工作被冷落，例如可維護性、長期風險與知識傳承。
操弄 報表更漂亮、趨勢更平滑 指標開始描述填表策略，而不是描述現場狀態。
多指標制衡 短期不如單一 KPI 乾淨 比較難管理，但較能避免把複雜目標壓扁成單一數字。

Psychology

為什麼壓力一上來，人就會把指標當成果

指標的吸引力在於它讓混亂的東西變得可比較；危險也在這裡，因為可比較不等於完整。

數字帶來控制感

當問題模糊時，一個清楚數字會讓人感到可管理。於是我們容易把「可量」誤認成「重要」。

排名會改變注意力

被排名後，人會自然追問「怎樣讓分數更好」，而不是「這個分數還能不能代表真實狀態」。

代理會被當成本體

久了之後，報表本身會變成目標。團隊討論的是讓曲線漂亮，而不是讓使用者、系統或產品真的更好。

重點是說明人與組織如何在壓力下把 proxy 當成 true goal。

Applications

把 proxy 失真拉回度量設計

Firmware 或產品團隊通常非常依賴指標：測試通過率、coverage、bug count、field return、release velocity。古德哈特定律能幫你檢查這些數字何時開始反咬。

測試覆蓋率

coverage 是好的儀表，但若被當成唯一目標，團隊可能補大量容易覆蓋的測試，卻不碰 race condition、power state 或 exception path。

Bug 數與關閉速度

若只追求 bug 數下降，常見副作用是重新分類、降低嚴重度、拆分或延後真正複雜的問題。

交付速度

velocity 被當成目標後，設計債、文件、review 深度與長期可靠性可能被壓低，因為它們短期不容易出現在分數裡。

  • 評估 KPI 時，先問：「它想代理哪個真目標？哪些重要部分沒有被量到？」
  • 如果指標開始影響獎懲，就要同步設計反操弄機制、質性 review 與副作用監控。

Misreadings

三個常見誤讀

這個概念很容易被拿來當反 KPI 的口號，但那會錯過它真正有用的地方。

所以不要量化 校正

不是。它提醒你量化要保留脈絡，並避免把單一 proxy 當成完整目標。

只要換一個更好的指標就好 校正

有時需要換指標，但更常見的是要降低單一指標權重，並加入質性判斷與多重檢查。

這代表員工在作弊 校正

不一定。很多時候人只是理性回應制度。若制度只獎勵數字，數字就會成為真正的產品。

概念背景

Goodhart's Law 條目整理了 Goodhart 1975 年的原始脈絡，以及後來被 Strathern 普及的「measure becomes a target」版本。
