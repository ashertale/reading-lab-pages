阿比林悖論

Abilene Paradox

團隊可能一起做出一件每個人私下都不想做的事，只因每個人都誤以為別人想做、自己不想掃興。

Orientation

最荒謬的共識，是每個人都在配合一個不存在的願望

阿比林悖論好用的地方，在於它讓你看見：群體失敗有時不是衝突太多，而是衝突根本沒被說出來。

Core Tension

每個人都不想顯得難搞、消極或掃興，最後反而一起把團隊送去一個誰都不真心想去的地方。

01 先區分公開與私下偏好

會議上的點頭不等於真想要，很多只是風向判讀後的順從。

02 再看反對成本

如果說不同意的社會代價太高，沉默就會被錯當支持。

03 最後設計退出點

真正好的決策機制，會讓人能安全地說『其實我不贊成』。

Setup

從一場沒人想反對的會議開始

這個悖論之所以可怕，是因為表面上看不見爭執，事後卻每個人都說自己原本就不想。

最小劇情

產品會議裡有人丟出一個加班硬上線的提案，其他人看彼此都沒反對，也就跟著說可以，最後整隊做了一件事後人人抱怨的事。

真正難點

問題不在於有人強逼大家，而在於每個人都誤判了別人的真正偏好。

有人先拋提案
→
其他人把沉默讀成支持
→
團隊一起執行沒人真想要的方案
Key Point

阿比林悖論最有價值的提醒是：共識不能只看表面順暢，還要看反對意見是否有安全出口。

Lenses

四個看假共識的角度

要分辨這是不是阿比林悖論，可以先看這四件事。

偏好不可見

大家的真想法沒有被具體說出來，只剩表情、沉默與猜測。

風向敏感

只要第一個訊號像支持，後面的人就更傾向順著氣氛。

反對代價高

怕顯得不投入、怕拖進度、怕壞氣氛，都是常見的沉默來源。

事後集體後悔

若事情做完後每個人都說自己原本就保留，往往就是重要警訊。

表面現象 看起來像 其實可能是
全場沒人反對 大家都同意 大家都在等別人先說不
提案快速通過 團隊很對齊 反對成本高到沒人願意開口
事後人人抱怨 執行力差 決策當下根本沒有真共識
會後私聊很多保留 個別情緒問題 公開場域不安全

Psychology

為什麼沉默這麼容易被誤讀成支持

因為人類很擅長讀氣氛，卻很不擅長在高壓場域裡校正自己對氣氛的想像。

怕破壞和諧

很多人寧願忍一時，也不想成為那個拖慢會議的人。

順從看起來比較安全

當資訊不完整時，跟著大家走常比單獨唱反調更有心理安全感。

他人沉默容易被神化

我們很容易把別人的沉默解讀成深思熟慮，而不是同樣的猶豫。

這個悖論最刺痛人的地方，不是誰壞，而是整個團隊一起把沉默誤當成對齊。

Applications

把真偏好拉回決策現場

對工程團隊來說，阿比林悖論常躲在衝刺規劃、on-call 制度和硬上線決策裡。

排程承諾

每個人都覺得這個時程太緊，但又以為 PM 和主管已經心意已決，於是大家一起點頭。

技術債延期

明知道先上功能會留下巨大後患，卻因為沒人想當阻擋者而默許。

值班制度

成員私下都覺得規則不公平，但公開場域裡沒人先開口挑戰。

  • 重要決策前，先收一次匿名偏好或書面立場，減少氣氛誤判。
  • 主持會議時刻意問『誰不同意、為什麼』，而不是只問『有人要補充嗎』。

Misreadings

別把任何順從都叫阿比林

這個概念有特定結構，不是所有糟糕會議都能直接套。

只要大家後悔就是阿比林 校正

不一定。重點在於當下是否存在彼此誤判的假共識。

它跟 groupthink 一樣 校正

兩者相近，但阿比林更強調每個人其實私下都不想要。

解法就是多辯論 校正

辯論有幫助，但更關鍵的是降低表達不同意的成本。
