PID Control

PID 控制把誤差分成現在有多遠、過去累積多少、變化速度多快；調得好是穩定修正，調不好就是震盪、延遲或過衝。

這頁適合用在 fan control、motor loop、thermal policy、charger 與任何閉迴路 firmware：它會幫你把調參問題拆成三種誤差回應。

Orientation

回饋不是越用力越好

PID 控制器根據誤差的現在值、累積值與變化率產生控制輸出，讓系統逼近 setpoint。

Core Tension

P 讓反應快，I 消除長期偏差，D 增加阻尼；但每一項都可能引入噪聲放大、過衝、延遲或飽和問題。

01 先理解 plant

被控對象的延遲、慣性、飽和與噪聲決定 PID 參數能多激進。

02 再分三項角色

P 管現在，I 補累積偏差，D 看變化趨勢；不要把所有問題都用單一 gain 解。

03 最後保護實作

anti-windup、輸出限幅、取樣時間與濾波會決定迴路在現場是否穩。

Setup

風扇忽快忽慢的溫控迴路

想像裝置用 PID 控 fan speed；溫度超過 setpoint 後風扇猛轉，溫度下降又立刻降速，整機噪音和溫度都在震盪。

最小場景

P 太大造成過衝，I 在風扇飽和時持續累積，D 又直接吃到 noisy temperature reading；使用者聽到週期性拉轉。

真正難點

看起來是風扇策略差，其實是控制迴路沒有把 plant delay、量測噪聲與輸出限制一起處理。

Key Point

PID 調校的難點是讓反應夠快，同時不把系統推進震盪。

Lenses

把 P、I、D 各自拆開看

三個項目不是魔法旋鈕；它們各自處理不同形狀的誤差，也各自帶來副作用。

比例 P

誤差越大輸出越大，反應直接；太高會造成過衝與震盪。

積分 I

累積偏差可消除 steady-state error；飽和時可能 windup，解除後過度修正。

微分 D

對誤差變化率反應，提供阻尼；對量測噪聲敏感，常需要濾波。

取樣與延遲

取樣時間、計算延遲與 actuator response 會改變穩定性，不能只看連續公式。

現象 可能原因 先試方向
過衝大 P 太高、I windup、延遲大 降 P、加 anti-windup、檢查 plant delay。
長期偏差 I 不足或輸出受限 調 I 並確認 actuator 有能力達標。
輸出抖動 D 放大量測噪聲 濾波、調低 D 或改量測位置。

Psychology

調參最容易變成追著症狀跑

看到反應慢就加 gain，看到過衝就降 gain；沒有 plant 模型與測試步階，很容易來回補洞。

單一旋鈕迷思

把 P 加大能短期改善慢反應，但可能把延遲與飽和問題暴露成震盪。

穩定被誤認成保守

穩定不是慢，而是能在擾動後可預期地回到目標。

現場條件被低估

溫度、負載、磨耗、供電與 sensor placement 會讓 lab 參數在 field 變得不穩。

PID 調校需要實驗記錄：step response、overshoot、settling time、steady-state error，而不是只靠手感。

Applications

用在馬達、溫度與充電控制

Embedded PID 的挑戰常在實作細節：整數運算、取樣 jitter、輸出飽和、sensor noise 與安全限制。

Thermal control

Fan 或 power limit 迴路要平衡溫度、噪音與性能，避免使用者感知到週期性 oscillation。

Motor control

位置或速度迴路需要考慮負載變化、摩擦、PWM resolution 與量測回授。

Charger loop

電壓/電流控制要處理限流、溫升與保護狀態，不能讓積分項在飽和時失控。

  • 在韌體裡實作 PID 時，明確處理 saturation、anti-windup、取樣週期與輸出限幅。
  • 用 step response 和 disturbance test 記錄參數效果，不要只看單一穩態點。

Misreadings

PID 不保證任何迴路都會穩

PID 很通用，但 plant delay、非線性、飽和與噪聲會讓它需要額外保護與調校。

加 D 一定更穩 校正

D 可能增加阻尼，也可能把 sensor noise 放大成輸出抖動。

I 越大越快消除偏差 校正

I 太大會造成 windup 與過衝，特別是在 actuator 飽和時。

一組參數跑所有條件 校正

不同負載、溫度與老化狀態可能需要 gain scheduling 或限制策略。

Questions

調參前先看迴路形狀

如果系統震盪

震盪週期是否對應 plant delay、取樣時間或 actuator 飽和？P/I/D 哪一項最可能在放大它？

如果穩態有偏差

是 I 不足，還是 actuator 已經到限制、sensor 有 offset、setpoint 本身不可達？
